# esa Daily Report Creator Skill

## 概要

Google Calendarの予定を取得し、構造化された日報をesa.ioに自動作成するスキルです。

## セットアップ

このスキルは以下のツールへのアクセスが必要です：

- Google Calendar API (list_gcal_events)
- esa API (esa_create_post, esa_get_teams)

## 使い方

### 基本的な使用方法

```text
ユーザー: 今日の日報を作成して
```

Claudeが以下を自動的に実行します：

1. 今日のカレンダー予定を取得
2. テンプレート構造を読み込み
3. esa.ioに日報を投稿（WIPとして）
   - カレンダー予定を「今日やったこと」に記載
   - 他のセクションは空欄のまま
4. 投稿URLを共有

**ポイント**: 他のセクション（学んだこと、MTG振り返りなど）は後からesaで編集できます。

### 情報を事前に提供する場合

```text
ユーザー: 今日の日報を作成して。今日はReactのuseEffectについて学んだ。
```

提供された情報も含めて日報を作成します。

### カスタマイズ例

#### 異なる日付の日報を作成

```text
ユーザー: 昨日の日報を作成して
```

#### 特定のカテゴリに投稿

```text
ユーザー: report/weekly/に投稿して
```

**注意**: screen_nameは自動的に現在のユーザーから検出されるため、指定する必要はありません。

## デフォルト設定

- **esaチーム**: gaji
- **screen_name**: 自動検出（esa_get_team_membersで現在のユーザーから取得）
- **カテゴリ**: report/daily-report/
- **投稿ステータス**: WIP
- **タイムゾーン**: Asia/Tokyo

## テンプレート構造

```markdown
## 今日やったこと
## 今日学んだこと
## リードとしての動き
## MTG 振り返り
## 次にするべきこと
## その他
```

テンプレートは `/templates/default.md` に定義されています。
このファイルを編集することで、日報の構造をカスタマイズできます。

### テンプレートのカスタマイズ方法

1. `templates/default.md` を編集
2. セクションの追加・削除・順序変更が可能
3. スキルは常にこのテンプレートファイルを参照するため、コードの変更は不要

## ファイル構成

```text
/mnt/skills/user/esa-daily-report/
├── SKILL.md           # メインのスキル定義（Claudeが読む）
├── README.md          # このファイル（人間向けドキュメント）
├── config.json        # 設定ファイル
└── templates/
    └── default.md     # 日報テンプレート
```

## トラブルシューティング

### カレンダー予定が取得できない

- Google Calendarへのアクセス権限を確認してください
- タイムゾーンが正しく設定されているか確認してください

### esa投稿が失敗する

- esaチーム名が正しいか確認してください
- APIアクセス権限があるか確認してください
- カテゴリパスが正しい形式か確認してください（先頭にスラッシュ不要）

### 予定の除外ルール

以下は自動的に除外されます：

- 勤務形態の予定（例：「自宅勤務」）
