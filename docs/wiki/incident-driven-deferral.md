# incident-driven deferral

## 内容

実需 (incident) が未実証の提案 issue は実装に入らず「一旦クローズ + 再起票トリガー明記」で閉じる。トリガーは具体的に書く。どの incident が出たら、どの使用頻度に達したら、2nd consumer が現れたら、の形で再着手条件を close コメントに残す。既存の skill / 内蔵機能でカバーできる提案も新規追加せず却下する。

## 定型手順

1. 提案が inspiration / 仮説段階か、実害の実例があるかを先に判定する
2. 実例がなければ実装せず close し、再起票トリガー (incident の種類・使用頻度・2nd consumer) をコメントに書く
3. 既存 skill / 内蔵機能でカバーできないかを新規追加より先に確認する
4. 前提となる監査・計測が harness 更新で古びていたら、再開時は前提の監査からやり直す
