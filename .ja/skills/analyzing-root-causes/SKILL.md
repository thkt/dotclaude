# 根本原因分析 - 5 Whys

## 対症療法 vs 根本原因

| タイプ   | 例                       | 結果           |
| -------- | ------------------------ | -------------- |
| 対症療法 | setTimeoutでDOM待機      | 後で壊れる     |
| 根本原因 | React refを適切に使用    | 恒久的な修正   |
| 対症療法 | 二重送信防止フラグを追加 | 複雑性増加     |
| 根本原因 | 送信時にボタンを無効化   | シンプルな修正 |

## 5 Whysプロセス

| ステップ | 質問                 | 明らかになること   |
| -------- | -------------------- | ------------------ |
| 1        | なぜこれが発生する？ | 観察可能な事実     |
| 2        | なぜそうなる？       | 実装の詳細         |
| 3        | なぜそうなっている？ | 設計上の決定       |
| 4        | なぜそれが存在する？ | アーキテクチャ制約 |
| 5        | なぜそう設計された？ | 根本原因           |
