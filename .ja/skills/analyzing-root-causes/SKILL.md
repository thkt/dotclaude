# 根本原因分析 - 5 Whys

症状ではなく根本原因に対処して問題を解決。

## 症状 vs 根本原因

| タイプ       | 例                            | 結果             |
| ------------ | ----------------------------- | ---------------- |
| 症状修正     | DOM待ちのためにsetTimeout追加 | 後で壊れる       |
| 根本原因修正 | React refを適切に使用         | 恒久的な解決     |
| 症状修正     | 二重送信防止フラグを追加      | 複雑性が増大     |
| 根本原因修正 | 送信中にボタンを無効化        | シンプルで信頼性 |

## 5 Whysプロセス

1. **Why** 発生する？ → 観察可能な事実
2. **Why** それが起こる？ → 実装詳細
3. **Why** そうなっている？ → 設計決定
4. **Why** それが存在する？ → アーキテクチャ制約
5. **Why** そう設計された？ → 根本原因

## 主要原則

| 原則            | 適用                         |
| --------------- | ---------------------------- |
| 予防 > パッチ   | 最良の修正は問題を防ぐ       |
| シンプル > 複雑 | 根本原因の解決はよりシンプル |
| Whyを問う       | 最初の答えを受け入れない     |
