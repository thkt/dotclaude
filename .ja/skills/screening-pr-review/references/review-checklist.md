# レビューチェックリスト

## ファイル別レビュー

| チェック項目   | 確認内容                                          |
| -------------- | ------------------------------------------------- |
| PR 整合性      | 変更が PR の目的に沿っているか                    |
| コードスタイル | コードベースとの一貫性                            |
| セキュリティ   | インジェクション、XSS、認証バイパス、秘密情報漏洩 |
| 副作用         | 既存機能への意図しない動作変更                    |
| パフォーマンス | 不要な API/DB 呼び出し、メモリリーク、N+1 クエリ  |
| コードスメル   | 重複、深いネスト、巨大関数                        |

## 依存影響

| チェック項目         | 方法                                       |
| -------------------- | ------------------------------------------ |
| インポート依存元     | 変更ファイル/エクスポートの import を Grep |
| インターフェース契約 | 関数シグネチャが変更されていないか確認     |
| 共有状態             | グローバル/モジュールの状態変更を確認      |
| テストカバレッジ     | 変更パスに対して既存テストが有効か         |

## コメントセクション

| セクション | 内容                                 |
| ---------- | ------------------------------------ |
| 要対応     | `[must]`, `[want]` の指摘            |
| 参考情報   | `[imo]`, `[ask]`, `[nits]`, `[info]` |

## テスト評価

「テスト追加」を提案する前に:

1. プロジェクトにテスト基盤があるか確認
2. 類似コードにテストがあるか確認
3. プロジェクト規約がテストを求める場合のみ提案
