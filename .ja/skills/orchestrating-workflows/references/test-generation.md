# テスト生成パターン

## 概要

| パターン       | ユースケース      | テスト状態     |
| -------------- | ----------------- | -------------- |
| Spec駆動       | 機能 (`/code`)    | スキップモード |
| バグ駆動       | バグ修正 (`/fix`) | アクティブ     |
| カバレッジ駆動 | カバレッジ改善    | アクティブ     |

## フレームワークスキップマーカー

| フレームワーク | スキップ構文                                        |
| -------------- | --------------------------------------------------- |
| Jest/Vitest    | `it.skip('test', () => { // TODO: [SKIP] FR-001 })` |
| Mocha          | `it.skip()` または `xit()`                          |
| 不明           | `// TODO: [SKIP]` マーカー                          |

## テスト設計

| 技法               | 用途                     | 例                              |
| ------------------ | ------------------------ | ------------------------------- |
| 同値分割           | 同じ動作の入力をグループ | 年齢: <18, 18-120, >120         |
| 境界値             | エッジをテスト           | 17, 18, 120, 121                |
| デシジョンテーブル | 複数条件                 | isLoggedIn × isPremium → access |

## AAAパターン

```typescript
test("name", () => {
  // Arrange - セットアップ
  // Act - 実行
  // Assert - 検証
});
```

## 命名規則

```typescript
describe("[対象]", () => {
  it("[条件]の場合、[結果]であるべき", () => {});
});
```

## カバレッジ目標

| レベル              | 目標 |
| ------------------- | ---- |
| C0 (ステートメント) | 90%  |
| C1 (ブランチ)       | 80%  |

## テスト優先度

| 優先度     | 内容                                                       |
| ---------- | ---------------------------------------------------------- |
| 必ずテスト | ビジネスロジック、サービス、クリティカルパス、エッジケース |
| 状況次第   | 複雑なユーティリティ、カスタムフック、変換                 |
| スキップ   | シンプルなアクセサ、UIレイアウト、外部ライブラリ動作       |
