# TDDサイクル (RGRC)

## クイックリファレンス

| フェーズ | 目標                       | 最大時間 |
| -------- | -------------------------- | -------- |
| Red      | 期待する理由でテストが失敗 | 2分      |
| Green    | 通過するための最小限コード | 5分      |
| Refactor | 動作変更なしでクリーンに   | 3分      |
| Commit   | 全チェック通過             | 1分      |

## Baby Steps (2分サイクル)

| ステップ | 時間 | アクション                 |
| -------- | ---- | -------------------------- |
| 1        | 30s  | 最小の失敗テストを書く     |
| 2        | 1min | 最小限で通過させる         |
| 3        | 10s  | テスト実行                 |
| 4        | 30s  | 小さなリファクタ（必要時） |
| 5        | 20s  | グリーンならコミット       |

**理由**: バグは常に直近2分の変更にある。

## フェーズ詳細

| フェーズ | キーアクション                   | よくある間違い     |
| -------- | -------------------------------- | ------------------ |
| Red      | 失敗が意図と一致することを確認   | テストが即座に通過 |
| Green    | **最小限**のコードを書く（罪OK） | 過剰実装           |
| Refactor | SOLID, DRY, オッカムを適用       | 動作を変更         |
| Commit   | 全品質チェックを実行             | チェックをスキップ |

## コマンド

```bash
npm test -- --testNamePattern="[test]"        # Red
npm test -- --watch --testNamePattern="[test]" # Green
npm run lint && npm test && npm run type-check # Commit
```

## TDDを使わない場合

- プロトタイプ（使い捨て）
- 外部API統合
- シンプルな単発スクリプト
- UI実験（ビジュアルファースト）
