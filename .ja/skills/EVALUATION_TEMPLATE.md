# スキル評価テンプレート

品質の高い評価シナリオを作成するためのガイドです。

## 使用方法

1. このテンプレートをコピーして `[skill-name]/EVALUATIONS.md` として保存
2. `[skill-name]` を実際のスキル名に置き換え
3. 各シナリオに具体的なクエリと期待動作を指定
4. 最低3つの評価シナリオを作成

---

## テンプレート（以下をコピー）

````markdown
# [skill-name]の評価

## 選択基準

このスキルをトリガーするキーワードとコンテキスト:

- **キーワード**: [日本語と英語のトリガーキーワード]
- **コンテキスト**: [使用コンテキスト、関連コマンド]

## 評価シナリオ（JSON形式）

**フィールドリファレンス**:

- `skills`: 評価対象のスキル名
- `query`: ユーザーリクエスト（リアルなテストのため日本語を使用）
- `files`: 評価時に提供するコンテキストファイル（不要な場合は空）
- `expected_behavior`: 確認すべき3-5の観察可能な動作

### シナリオ1: 基本的な使用

```json
{
  "skills": ["[skill-name]"],
  "query": "[典型的なユーザーリクエスト]",
  "files": [],
  "expected_behavior": [
    "スキルがトリガーされてロードされる",
    "正しいリファレンスファイルがアクセスされる",
    "出力がスキルガイドラインに従う"
  ]
}
```

### シナリオ2: エッジケース

```json
{
  "skills": ["[skill-name]"],
  "query": "[エッジケースリクエスト]",
  "files": [],
  "expected_behavior": [
    "エッジケースを適切に処理",
    "適切なフォールバックまたはガイダンスを提供"
  ]
}
```

### シナリオ3: エラーハンドリング

```json
{
  "skills": ["[skill-name]"],
  "query": "[問題のあるリクエスト]",
  "files": [],
  "expected_behavior": [
    "明確なエラーメッセージを提供",
    "代替案や次のステップを提案"
  ]
}
```

## 手動検証チェックリスト

各シナリオ実行後:

- [ ] スキルが正しくトリガーされた
- [ ] 期待される動作が観察された
- [ ] 出力品質が基準を満たす

## ベースライン比較

### スキルなし

- [スキルなしの制限を列挙]

### スキルあり

- [スキルありの改善点を列挙]
````

## オプションの追加セクション

以下のセクションはオプションであり、スキルの特性に基づいて含めるべきです。

### オプションセクションの決定フロー

```text
開始
  │
  ├─ スキルに列挙可能なタイプ/アクション/モードがある?
  │    YES → "ドメイン固有クイックリファレンス"テーブルを追加
  │    NO  → スキップ
  │
  ├─ SKILL.mdは400行以上?
  │    YES → "プログレッシブ・ディスクロージャー検証"チェックリストを追加
  │    NO  → スキップ
  │
  ├─ スキルはコマンドやエージェントと統合?
  │    YES → "統合ポイント"セクションを追加
  │    NO  → スキップ
  │
  └─ スキルは構造化ファイルを生成?
       YES → "ファイル例"セクションを追加
       NO  → スキップ
```

### ドメイン固有クイックリファレンス（該当する場合）

特定の用語やアクションタイプを持つスキルには、クイックリファレンステーブルを含める:

```markdown
## [ドメイン]リファレンス

| タイプ | 説明 | ユースケース |
|------|------|----------|
| type-a | Xを行う | シナリオY |
| type-b | Zを行う | シナリオW |
```

**使用時**: スキルに列挙可能なタイプ、アクション、モードがある場合（例: creating-hooksにはblock/warnアクションがある）

### プログレッシブ・ディスクロージャー検証（長いスキル用）

500行制限に近づくスキルには、プログレッシブ・ディスクロージャーが実装されていることを検証:

```markdown
## プログレッシブ・ディスクロージャー検証

- [ ] SKILL.mdが概要として機能（<500行）
- [ ] 詳細コンテンツは別のリファレンスファイルに
- [ ] 詳細ドキュメントへの明確なリンク
- [ ] 概要が基本的な使用に十分
```

**使用時**: SKILL.mdが400行以上に近づく場合

### 統合ポイント（システムスキル用）

コマンド/エージェントと統合するスキルには、統合を文書化:

```markdown
## 統合ポイント

### コマンドとの連携
- /command-name: スキルがどう統合されるか

### エージェントとの連携
- agent-name: スキルがエージェントをどうサポートするか
```

**使用時**: スキルが複数のコマンドやエージェントで使用される場合

### ファイル例（ファイル生成スキル用）

特定のファイル形式を作成するスキル用:

```markdown
## 出力形式例

### 例: [説明]
[期待される出力形式を示すコードブロック]
```

**使用時**: スキルが構造化出力ファイルを生成する場合（例: ADR、設定ファイル）
