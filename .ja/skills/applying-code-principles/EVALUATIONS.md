# applying-code-principlesの評価

## 選択基準

このスキルをトリガーするキーワードとコンテキスト:

- **キーワード**: SOLID, DRY, オッカムの剃刀, KISS, ミラーの法則, YAGNI, principle, 原則, simplicity, シンプル, complexity, 複雑, architecture, アーキテクチャ, refactor, リファクタリング, maintainability, 保守性, code quality, コード品質, design pattern, best practice, clean code
- **コンテキスト**: コードレビュー、アーキテクチャ決定、リファクタリング計画、複雑さ評価

## 評価シナリオ

### シナリオ1: 基本的な原則適用

```json
{
  "skills": ["applying-code-principles"],
  "query": "このクラスが大きくなりすぎた。どう整理すべき？",
  "files": ["src/services/UserService.ts"],
  "expected_behavior": [
    "スキルが'クラス'とリファクタリングコンテキストでトリガーされる",
    "単一責任原則（SRP）分析を適用",
    "クラス内の複数の責任を特定",
    "根拠を含む具体的な分割戦略を提案",

}
```

## 手動検証チェックリスト

各シナリオ実行後:

- [ ] スキルがキーワード/コンテキストで正しくトリガーされた
- [ ] 関連する原則が適切に適用された
- [ ] 原則の優先順位が尊重された（オッカムの剃刀 > YAGNI > ...）
- [ ] 具体的で実行可能なアドバイスが提供された
- [ ] 適切な場合にクイック判断質問が参照された
- [ ] 基本概念の過剰説明がない（Claudeは既に知っている）

## ベースライン比較

### スキルなし

- 構造化された原則適用のない一般的なアドバイス
- 原則の競合を見逃す可能性
- クイック判断質問フレームワークがない

### スキルあり

- 体系的な原則ベースの分析
- 優先順位付きの明示的な競合解決
- 実行可能なフレームワーク（クイック判断質問）
- 一貫した用語とアプローチ
