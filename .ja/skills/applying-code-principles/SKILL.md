# コード原則

## 優先順序

| 優先度 | 原則           | 適用場面                         |
| ------ | -------------- | -------------------------------- |
| 1      | 安全第一       | セキュリティ、データ整合性       |
| 2      | YAGNI          | 不要なものは作らない             |
| 3      | オッカムの剃刀 | 最もシンプルな解決               |
| 4      | SOLID          | 複雑なシステム向け               |
| 5      | DRY            | 重複排除（明確性を犠牲にしない） |
| 6      | ミラーの法則   | 7±2の認知限界を尊重              |

## ルール

| 原則   | ルール                                      |
| ------ | ------------------------------------------- |
| DRY    | 3回目の重複で抽象化（3回ルール）            |
| SOLID  | 2番目の実装が現れたときのみインターフェース |
| YAGNI  | 問題が今存在する場合のみ作る                |
| 可読性 | 新しいチームメンバーが1分以内に理解できる   |
