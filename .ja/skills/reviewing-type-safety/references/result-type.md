# Result型エラーハンドリング

**原則**: コードレビューでなく型でエラーハンドリングを強制

## パターン

| 型           | 意味     | 呼び出し側アクション |
| ------------ | -------- | -------------------- |
| `Ok<T>`      | 成功     | 値を使用             |
| `Err<Error>` | 失敗     | **必須ハンドリング** |
| `Err<null>`  | 処理済み | 任意                 |

## 型定義

```typescript
type Result<T, E> = { ok: true; value: T } | { ok: false; error: E };
```

## 適用タイミング

| シナリオ               | アプローチ       |
| ---------------------- | ---------------- |
| API呼び出し            | 常にResult型     |
| フォームバリデーション | エラー付きResult |
| 内部ユーティリティ     | throw/catchでOK  |

## 強制方法

| ツール     | ルール                      |
| ---------- | --------------------------- |
| ESLint     | `eslint-plugin-return-type` |
| TypeScript | `never`で網羅チェック       |
