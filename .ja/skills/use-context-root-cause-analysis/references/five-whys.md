# 5 Whys - 適用例

良い問題ステートメントは具体的で観察可能。「ダッシュボードの読み込みに 5 秒かかる」は分析可能、「アプリが遅い」は不可。

## 例: ダッシュボードの読み込みに 5 秒かかる

| Step | 質問               | 回答                          |
| ---- | ------------------ | ----------------------------- |
| 1    | なぜ 5 秒？        | API `/api/dashboard` が 4.5 秒 |
| 2    | なぜ API 遅い？    | 15 テーブルをクエリ           |
| 3    | なぜ 15 テーブル？ | 全モジュールを 1 ビューで表示 |
| 4    | なぜ全部一度に？   | マウント時に全てレンダリング  |
| 5    | なぜマウント時？   | 遅延読み込みがない            |

解決策: セクション単位の遅延読み込み、オンデマンドでフェッチ。

## 例: フォームが 2 回送信される

| Step | 質問               | 回答                                   |
| ---- | ------------------ | -------------------------------------- |
| 1    | なぜ 2 回？        | リクエスト完了前に 2 回クリック        |
| 2    | なぜクリック可能？ | 送信中にボタン無効化されていない       |
| 3    | なぜ無効化なし？   | `isSubmitting` がハンドラ開始後に設定  |
| 4    | なぜ後？           | React がステート更新をバッチ処理       |
| 5    | なぜバッチ問題？   | 命令的関心事にステートを使っている     |

解決策: `useRef` で即時フラグ、または `disabled={pending}`。
