# ユビキタス言語の抽出

Slackの会話からドメイン用語を抽出し、用語集のドラフトを生成。

## 前提条件

| 変数              | 説明                                             |
| ----------------- | ------------------------------------------------ |
| `SLACK_TOKEN`     | User OAuthトークン (`xoxp-...`)                  |
| `SLACK_WORKSPACE` | ワークスペースのサブドメイン（`.slack.com`の前） |

API呼び出し前に `SLACK_TOKEN` が設定済みであることを確認。

## Phase 1: スコープ定義

AskUserQuestionで確認:

| 質問            | 選択肢 / 入力                                               |
| --------------- | ----------------------------------------------------------- |
| Slackチャンネル | チャンネル名またはID（カンマ区切り）                        |
| 期間            | 1週間 / 1か月 / 3か月 / 6か月                               |
| 検索キーワード  | 抽出フォーカス用のドメインキーワード（任意）                |
| 参考資料        | mdパス、Google Docs/SheetsのURL（カンマ区切り、またはなし） |
| 出力パス        | デフォルト: `docs/domain/glossary.md`                       |
| 既存用語集?     | 既存ファイルのパス（マージモード）、またはなし（新規）      |

## Phase 2: 会話の収集

### チャンネル履歴

```bash
curl -s -H "Authorization: Bearer $SLACK_TOKEN" \
  "https://slack.com/api/conversations.history?channel=$CHANNEL&oldest=$OLDEST_TS&limit=200" \
  | jq 'if .ok then [.messages[] | {text, ts, user}] else "Error: \(.error)" end'
```

### キーワード検索（キーワード指定時）

```bash
curl -s -G -H "Authorization: Bearer $SLACK_TOKEN" \
  --data-urlencode "query=in:#$CHANNEL $KEYWORD" \
  -d "count=100" \
  "https://slack.com/api/search.messages" \
  | jq 'if .ok then [.messages.matches[] | {text, ts, channel: .channel.name, user}] else "Error: \(.error)" end'
```

### スレッド展開

`reply_count > 0`
のメッセージはスレッド返信を取得 — スレッドには明確化や定義が含まれることが多い。

```bash
curl -s -H "Authorization: Bearer $SLACK_TOKEN" \
  "https://slack.com/api/conversations.replies?channel=$CHANNEL&ts=$TS" \
  | jq 'if .ok then [.messages[] | {text, ts, user}] else "Error: \(.error)" end'
```

### ページネーション

`response_metadata.next_cursor`
でページネーション。コンテキストサイズの上限として、チャンネルあたり最大5ページ。

## Phase 2.5: 参考資料の読み込み

参考資料が指定された場合、用語抽出の追加コンテキストとして読み込む。

| ソース             | 読み込み方法                                     |
| ------------------ | ------------------------------------------------ |
| ローカルmdファイル | Readツール                                       |
| Google Docs        | `accessing-google-workspace`スキル（gcloud CLI） |
| Google Sheets      | `accessing-google-workspace`スキル（gcloud CLI） |

参考資料はkikuのDBには埋め込まない。Phase
3の用語抽出時の相互参照コンテキストとして機能 —
LLMが発見した用語を公式ドキュメントと照合するのに役立つ。

## Phase 3: 用語抽出

収集した会話をバッチで分析。各バッチで以下を抽出:

| 抽出対象     | シグナルパターン                                               |
| ------------ | -------------------------------------------------------------- |
| 用語候補     | 複数メッセージにまたがって繰り返される名詞/フレーズ            |
| 定義         | "XはYのこと", "X means Y", "X is Y", "X = Y", "Xって Y のこと" |
| 同義語       | "X (or Y)", "X aka Y", "Xとも言う"                             |
| 訂正         | "XじゃなくてY", "not X but Y" — どちらが正規用語かを示す       |
| コード参照   | バッククォートで囲まれたビジネス用語に対応する識別子           |
| 曖昧シグナル | "Xって何？", "what do you mean by X" — 明確化が必要な用語      |

### 抽出プロンプト構造

各会話バッチに対して構造化出力を生成:

```markdown
### 抽出された用語

| 用語     | 定義                       | コードマッピング                           | 確信度 | エビデンス             | 参照マッピング                                           | 同義語       |
| -------- | -------------------------- | ------------------------------------------ | ------ | ---------------------- | -------------------------------------------------------- | ------------ |
| メンバー | 有料プラン契約者           | User where subscriptionStatus === 'active' | high   | slack:#{channel}/p{ts} | design-doc.md: 「メンバー = 有料プラン契約済みユーザー」 | 有料ユーザー |
| ゲスト   | 未登録・未ログインユーザー | —                                          | medium | slack:#{channel}/p{ts} | —                                                        | —            |
```

参考資料が読み込まれている場合（Phase
2.5）、候補用語を参考資料と照合。`ref_mapping`は用語が参考資料のどこに記載されているかを引用し、公式ドキュメントがSlackでの使用を裏付ける場合に確信度を引き上げる。

### 確信度レベル

| レベル | 基準                                  |
| ------ | ------------------------------------- |
| high   | 会話中に明示的な定義が見つかった      |
| medium | 2つ以上のメッセージの使用文脈から推測 |
| low    | 単一の言及、定義は推測的              |

## Phase 4: コード相互参照（任意）

プロジェクトのコードベースが利用可能な場合、抽出用語との相互参照を実施:

1. 型名、インターフェース、enum、定数から用語候補を検索
2. Slack用語をコード識別子にマッチング
3. 抽出出力の `code_mapping` フィールドを記入

完全一致の識別子検索にはGrepを使用。

## Phase 5: 用語集生成

### 新規用語集

出力パスに以下の形式で書き込み:

```markdown
# ドメイン用語集

> Slackの会話（{channels}、{date_range}）から自動生成。使用前にレビューと改善を実施してください。

| 用語 | 定義 | コードマッピング | 確信度 | ソース | 参照 |
| ---- | ---- | ---------------- | ------ | ------ | ---- |
| ...  | ...  | ...              | ...    | ...    | ...  |

## 同義語

| 正規用語 | 同義語 |
| -------- | ------ |
| ...      | ...    |

## 曖昧な用語

チームの明確化が必要な用語:

| 用語 | 曖昧な理由 | 推奨アクション |
| ---- | ---------- | -------------- |
| ...  | ...        | ...            |
```

### マージモード

既存用語集が指定された場合:

1. 既存用語集を読み込み
2. 既存にない新規用語を特定
3. 既存用語の更新された定義を特定
4. 書き込み前にAskUserQuestionでdiffをユーザーに提示

## 出力の検証可能性

| フィールド | 基準                                        |
| ---------- | ------------------------------------------- |
| 定義       | Slackソースを引用: `slack:#{channel}/p{ts}` |
| 確信度     | 確信度レベルテーブルに準拠                  |
| 曖昧性     | 別セクションに記載、定義を推測しない        |

## エラー処理

| エラー                        | アクション                                         |
| ----------------------------- | -------------------------------------------------- |
| `SLACK_TOKEN` 未設定          | 報告して停止                                       |
| チャンネルにアクセス不可      | チャンネルを報告、権限確認を提案                   |
| 用語が抽出されない            | 「スコープ内にドメイン用語が見つかりません」と報告 |
| メッセージが多すぎる（>1000） | 期間の絞り込みやキーワード追加を提案               |
