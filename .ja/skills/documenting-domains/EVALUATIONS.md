# documenting-domainsの評価

## 選択基準

このスキルをトリガーするキーワードとコンテキスト:

- **キーワード**: domain understanding, glossary, entities, business logic, domain model, ER diagram, use cases, ドメイン理解, 用語集, エンティティ, ビジネスロジック, ドメインモデル, ER図, ユースケース
- **コンテキスト**: ドメインドキュメント、ビジネスモデル分析、/docs:domainコマンド

## 評価シナリオ

### シナリオ1: エンティティ抽出

```json
{
  "skills": ["documenting-domains"],
  "query": "このプロジェクトのエンティティ一覧をドキュメント化して",
  "files": [],
  "expected_behavior": [
    "スキルが'エンティティ'でトリガーされる",
    "TypeScriptインターフェース/クラスを検出",
    "Pythonのdataclass/Pydanticモデルを検出",
    "データベースモデルを検出（Prisma, TypeORM）",
    "属性付きでエンティティをリスト"
  ]
}
```

## 手動検証チェックリスト

各シナリオ実行後:

- [ ] スキルがドメインキーワードで正しくトリガーされた
- [ ] エンティティが正しく抽出された
- [ ] ドメイン用語が正確に解析された
- [ ] 関係が正しく特定された
- [ ] MermaidのER図が有効だった
- [ ] ビジネスロジックが意味のある形でドキュメント化された

## ベースライン比較

### スキルなし

- 手動のドメインドキュメント
- エンティティを見逃す可能性
- 関係のビジュアライゼーションなし
- 用語集が一貫しない

### スキルあり

- 自動エンティティ抽出
- マルチフレームワークサポート（TypeScript, Python, DBモデル）
- MermaidのER図生成
- 自動用語集作成
- 概念関係マッピング
