# Reactパフォーマンス最適化

## メモ化の判断

| Hook          | 使用場面                                | オーバーヘッド |
| ------------- | --------------------------------------- | -------------- |
| `React.memo`  | 高コストコンポーネント、同じpropsが多い | 浅い比較       |
| `useMemo`     | 高コストな計算                          | 依存チェック   |
| `useCallback` | メモ化された子に渡す関数                | 依存チェック   |

## メモ化の基準

| 基準               | メモ化する               | スキップ          |
| ------------------ | ------------------------ | ----------------- |
| 計算コスト         | 重いソート/フィルター    | `count * 2`       |
| 再レンダリング頻度 | 同じpropsで頻繁          | まれ              |
| 子コンポーネント   | メモ化された子が関数受取 | 子なし/メモ化なし |

## パターン

### React.memo

```tsx
const ExpensiveList = React.memo(({ data }: { data: Data[] }) => {
  return (
    <ul>
      {data.map((item) => (
        <li key={item.id}>{item.name}</li>
      ))}
    </ul>
  );
});
```

### useMemo

```tsx
const sortedItems = useMemo(
  () => items.sort((a, b) => b.price - a.price),
  [items],
);
```

### useCallback

```tsx
const handleClick = useCallback(() => {
  console.log("Clicked");
}, []);
```

## リスト仮想化

| リストサイズ | 解決策                     |
| ------------ | -------------------------- |
| <100件       | 通常レンダリング           |
| 100-1000件   | 仮想化を検討               |
| >1000件      | react-window/react-virtual |

## ステート最適化

| 問題                                 | 解決策                 |
| ------------------------------------ | ---------------------- |
| グローバルステートで全再レンダリング | ドメインでステート分割 |
| オブジェクトの部分更新               | プリミティブに分離     |
| Contextで全consumer再レンダリング    | 更新頻度でContext分割  |

## よくある落とし穴

| 落とし穴                   | 修正                      |
| -------------------------- | ------------------------- |
| 空の依存でステートアクセス | 依存にステート追加        |
| 過剰メモ化                 | まず測定                  |
| 依存の欠落                 | exhaustive-depsルール使用 |
