# 文章レビュー

文脈を共有しリンク先を開けるチームメイトに向けて書く。PR が運ぶのは差分の意図とレビュー導線で、コードの what は diff が運ぶ。消しても読み手が判断を誤らない行は書かない。語彙レベルの空句はここでは扱わず、本文言語に対応する空句ファイルで点検する。

## 構造

| チェック       | 質問                                                                     |
| -------------- | ------------------------------------------------------------------------ |
| Why stated     | what でなく変更理由の why が冒頭 1-3 行にあるか                          |
| Test evidence  | 実行コマンド、テストファイル、スクリーンショットなど検証が具体的か       |
| Scope          | 変更が集中しているか、無関係な編集が混ざっていないか                     |
| Reviewer focus | `focus on X` や `skim Y` でレビュー優先度が明確か                        |
| Risk surfaced  | マイグレーション、ロールバック、パフォーマンスのリスクが明示されているか |
