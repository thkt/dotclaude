# ロールバック計画ガイド

## ロールバックトリガー

| トリガー         | 閾値     | アクション         |
| ---------------- | -------- | ------------------ |
| エラー率         | >X%増加  | 即時ロールバック   |
| レスポンスタイム | >Xms劣化 | 段階的ロールバック |
| 重大障害         | いずれか | 即時ロールバック   |

## ロールバック戦略

| 戦略         | 使用時                         | 時間  |
| ------------ | ------------------------------ | ----- |
| Immediate    | 重大障害、データ破損           | <5分  |
| Gradual      | パフォーマンス問題、軽微なバグ | <10分 |
| Roll Forward | ロールバックより修正が速い     | 変動  |

## 即時ロールバック手順

| ステップ | アクション                         | 検証             |
| -------- | ---------------------------------- | ---------------- |
| 1        | 問題検出（監視/ログ）              | トリガー条件確認 |
| 2        | ロールバック承認                   | 決定者サインオフ |
| 3        | `git revert`または前タグをデプロイ | デプロイ成功     |
| 4        | 機能を検証                         | エラー率正常化   |

## 段階的ロールバック手順

| フェーズ | アクション                   | 時間  |
| -------- | ---------------------------- | ----- |
| 1        | フィーチャーフラグを無効化   | <2分  |
| 2        | トラフィックを旧バージョンへ | <5分  |
| 3        | 必要なら完全ロールバック     | <10分 |

## データ考慮事項

| シナリオ               | アクション                   |
| ---------------------- | ---------------------------- |
| DBマイグレーションあり | ロールバックスクリプトを準備 |
| データ形式変更         | 後方互換性を検証             |
| 新データ作成           | 処理戦略を定義               |

## 検証チェックリスト

| チェック         | ターゲット     |
| ---------------- | -------------- |
| 主要機能         | 動作           |
| エラー率         | ≤ ベースライン |
| レスポンスタイム | ≤ ベースライン |
| エラーログ       | 新エラーなし   |

## 判断マトリクス

| 条件                        | 推奨               |
| --------------------------- | ------------------ |
| 修正時間 < ロールバック時間 | Roll Forward       |
| データ損失リスク            | Roll Forward       |
| クリティカルパスに影響      | Immediate Rollback |
| パフォーマンスのみ          | Gradual Rollback   |
