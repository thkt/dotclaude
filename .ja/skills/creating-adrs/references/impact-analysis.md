# 影響分析ガイド

## 分析エリア

| エリア         | 主要な質問                       | リスク指標                  |
| -------------- | -------------------------------- | --------------------------- |
| コードベース   | 影響ファイル数は？破壊的変更は？ | >10ファイル、API変更        |
| 依存関係       | パッケージ更新？バージョン競合？ | メジャーバージョンバンプ    |
| チーム         | 学習コスト？トレーニング必要？   | 新フレームワーク/パラダイム |
| インフラ       | CI/CD変更？環境変数？            | ビルドプロセス変更          |
| パフォーマンス | リソース使用量変化？             | >10%劣化                    |
| セキュリティ   | アクセス制御変更？               | 認証/データ処理             |

## リスク評価

| リスクレベル | 基準                                        |
| ------------ | ------------------------------------------- |
| Low          | <5ファイル、破壊的変更なし、馴染みの技術    |
| Medium       | 5-20ファイル、軽微な破壊的変更、多少の学習  |
| High         | >20ファイル、大きな破壊的変更、新パラダイム |

## コードベース影響チェックリスト

| チェック       | 説明                     |
| -------------- | ------------------------ |
| ファイル数     | 影響ファイルを見積もり   |
| 破壊的変更     | API/インターフェース変更 |
| 互換性レイヤー | 段階的移行に必要？       |
| 移行パス       | 段階的に実行可能？       |

## 依存関係影響チェックリスト

| チェック       | 説明                      |
| -------------- | ------------------------- |
| パッケージ更新 | 新規/削除の依存関係       |
| バージョン競合 | peer dependency問題       |
| ビルドツール   | バンドラー/コンパイラ変更 |

## チーム影響チェックリスト

| チェック     | 説明                 |
| ------------ | -------------------- |
| 学習コスト   | 人あたり時間         |
| トレーニング | 資料が必要？         |
| ドキュメント | 更新が必要           |
| 移行期間     | タイムライン見積もり |

## インフラ影響チェックリスト

| チェック       | 説明               |
| -------------- | ------------------ |
| ビルドプロセス | 変更が必要？       |
| CI/CD          | パイプライン更新？ |
| 環境           | 新しい環境変数？   |
| 監視           | 設定更新？         |
