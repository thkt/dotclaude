# テストカバレッジガイド

## テストタイプ

| タイプ      | 目的                | カバレッジ目標   |
| ----------- | ------------------- | ---------------- |
| Unit        | 関数/メソッドレベル | ≥80%             |
| Integration | モジュール間連携    | 主要パス         |
| E2E         | ユーザーフロー      | クリティカルパス |
| Performance | 負荷/ストレス       | ベンチマーク     |

## 影響評価

| チェック     | 質問                               |
| ------------ | ---------------------------------- |
| 影響テスト   | いくつのテストファイルが更新必要？ |
| 破壊的変更   | 既存テストが失敗する？             |
| モック更新   | どのモックに変更が必要？           |
| テストデータ | フィクスチャに更新が必要？         |

## 新規テスト要件

| 優先度 | テストタイプ | フォーカス                 |
| ------ | ------------ | -------------------------- |
| P0     | Unit         | 新関数、エラーハンドリング |
| P1     | Integration  | APIコントラクト、DB操作    |
| P2     | E2E          | クリティカルユーザーフロー |
| P3     | Performance  | パフォーマンス重視の場合   |

## カバレッジ目標

| メトリクス | 最小 | 推奨 |
| ---------- | ---- | ---- |
| Statement  | 70%  | 80%  |
| Branch     | 60%  | 70%  |
| Function   | 80%  | 90%  |

## テストタイプ別チェックリスト

### Unitテスト

| チェック       | 説明     |
| -------------- | -------- |
| Happy path     | 通常動作 |
| エッジケース   | 境界値   |
| エラーケース   | 例外処理 |
| Null/undefined | 欠損入力 |

### Integrationテスト

| チェック        | 説明                  |
| --------------- | --------------------- |
| モジュール境界  | モジュール間呼び出し  |
| APIコントラクト | リクエスト/レスポンス |
| DB操作          | CRUD操作              |
| 外部サービス    | モック/スタブ動作     |

### E2Eテスト

| チェック         | 説明                        |
| ---------------- | --------------------------- |
| クリティカルパス | ログイン、チェックアウト等  |
| エラーシナリオ   | フォームバリデーション、404 |
| クロスブラウザ   | 該当する場合                |

## 品質ゲート

| ゲート     | 基準                  |
| ---------- | --------------------- |
| PRマージ   | Unitテストパス        |
| Staging    | Integrationテストパス |
| Production | 全テスト + E2Eパス    |
