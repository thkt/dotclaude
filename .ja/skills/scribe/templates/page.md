# wiki ページテンプレート

/scribe が Phase 5 でページを作成 / 昇格するとき、この骨格で書く。1共通項1ページで、ファイル名は kebab-case の `<共通項名>.md`。`<...>` は生成時に置き換える。

## テンプレート

各ページは 内容 → 定型手順 → 参照コード → 根拠 の順で構成する。参照コードには Phase 4 で検証した現行コードの位置を `path` + シンボル名（関数 / 型 / 見出し）で書き、行番号は書かない。コード断片の転記はパターンの形自体が本体の場合のみ数行まで。根拠には元の議論の PR/issue 番号を書く。

```markdown
# <共通項名>

## 内容

何をする/しないかの言語化（1〜3 文）。

## 定型手順

繰り返す手順・チェックリスト（あれば）。

## 参照コード

- `path/to/file` の `シンボル名`（何が読めるか1行）

## 根拠

- #12 何があったか1行
- #34 何があったか1行
```
