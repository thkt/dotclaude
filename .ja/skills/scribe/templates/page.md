# wiki ページテンプレート

/scribe が Phase 6 でページを作成 / 昇格するとき、この骨格で書く。1 共通項 1 ページで、ファイル名は kebab-case の `<共通項名>.md`。`<...>` は生成時に置き換える。

## テンプレート

各ページは内容 → 定型手順 → 参照コード → 由来 → 根拠の順で構成する。参照コードには Phase 4 で検証した現行コードの位置を `path` + シンボル名（関数 / 型 / 見出し）で書き、行番号は書かない。コード断片の転記はパターンの形自体が本体の場合のみ数行まで。由来は任意の節で、Phase 5 の判定で Yes になった DR だけを書き、該当が無ければ節ごと省く。根拠には元の議論の PR / issue 番号を書く。

```markdown
# <共通項名>

## 内容

何をする / しないかの言語化（1 〜 3 文）。

## 定型手順

繰り返す手順 / チェックリスト（あれば）。

## 参照コード

- `path/to/file` の `シンボル名`（何が読めるか 1 行）

## 由来

- `docs/decisions/<NNNN>-<タイトル>.md`（この決定から派生。1 行で何を決めた DR か）

## 根拠

- #12 何があったか 1 行
- #34 何があったか 1 行
```
