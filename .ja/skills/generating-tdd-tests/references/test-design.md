# テスト生成

**原則**: 場当たりでなく体系的に設計

## 技法

| 技法               | いつ      | パターン                         |
| ------------------ | --------- | -------------------------------- |
| 同値分割           | 常に      | 動作グループごとに1値テスト      |
| 境界値分析         | 数値範囲  | [min-1, min, max, max+1]をテスト |
| デシジョンテーブル | 3条件以上 | 全組み合わせをマッピング         |

## カバレッジ目標

| レベル     | 目標 |
| ---------- | ---- |
| C0（命令） | 90%  |
| C1（分岐） | 80%  |

## 優先度

| 高                         | 低                 |
| -------------------------- | ------------------ |
| ビジネスロジック、公開API  | ゲッター/セッター  |
| エラーハンドラ、複雑な条件 | フレームワーク定型 |
