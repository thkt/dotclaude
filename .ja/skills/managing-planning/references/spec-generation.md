# Spec生成

## 入力

一貫性のためにworkspaceから最新SOWを自動検出。

## 必須セクション

| セクション         | 目的                         | IDフォーマット |
| ------------------ | ---------------------------- | -------------- |
| 機能要件           | 実装内容                     | FR-001         |
| データモデル       | TypeScriptインターフェース   | -              |
| 実装詳細           | フェーズの詳細               | -              |
| テストシナリオ     | Given-When-Then              | T-001          |
| 非機能要件         | パフォーマンス、セキュリティ | NFR-001        |
| 依存関係           | 外部/内部                    | -              |
| 実装チェックリスト | フェーズ別                   | -              |

## トレーサビリティ

```text
FR-001  Implements: AC-001
T-001   Validates: FR-001
NFR-001 Validates: AC-002
```

## 信頼度マーカー

| マーカー | 意味     | エビデンス |
| -------- | -------- | ---------- |
| [✓]      | 検証済み | file:line  |
| [→]      | 推論     | 理由を記載 |
| [?]      | 不確実   | 調査が必要 |

## Component API (フロントエンドのみ)

UI関連の場合は含める: Propsテーブル、バリアント、状態、使用例。

## 出力

```text
保存先: .claude/workspace/planning/[same-dir]/spec.md
```

## テンプレート

構造参照: `~/.claude/templates/spec/template.md`

- コピー: セクション構造、ID命名
- コピーしない: 実際のコンテンツ
