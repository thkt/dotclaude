# テスタビリティレビュー

目標: 複雑なモッキングなしでテストしやすいコード。

## テスタビリティ指標

| 指標               | 良い                       | 警告                    |
| ------------------ | -------------------------- | ----------------------- |
| モック複雑性       | シンプルなスタブ           | 深いモックチェーン      |
| テストセットアップ | < 10行                     | > 30行                  |
| 依存関係           | 明示的（パラメータ/props） | 隠れた（インポート）    |
| 副作用             | 分離されている             | ロジックと混在          |
| 状態               | 予測可能                   | グローバル/ミュータブル |

## セクションベースのロード

| セクション         | ファイル                             | フォーカス                   |
| ------------------ | ------------------------------------ | ---------------------------- |
| DI                 | `references/dependency-injection.md` | 注入可能な依存関係           |
| 純粋               | `references/pure-functions.md`       | 副作用分離                   |
| モックフレンドリー | `references/mock-friendly.md`        | インターフェース、ファクトリ |

## クイックチェックリスト

- [ ] 依存関係が注入可能
- [ ] 明確な分離: 純粋 vs 不純
- [ ] Presentationalコンポーネントが純粋
- [ ] 副作用がhooks/containersに分離
- [ ] グローバルなミュータブル状態なし

## 主要原則

| 原則           | 適用                 |
| -------------- | -------------------- |
| DIP（SOLID）   | 抽象に依存           |
| 純粋関数       | 同じ入力 = 同じ出力  |
| 明示的依存関係 | パラメータとして渡す |
| 単一責任       | テストする理由は1つ  |
