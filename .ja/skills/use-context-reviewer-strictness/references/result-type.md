# Result 型によるエラーハンドリング

原則。コードレビューではなく型でエラーハンドリングを強制する。

## パターン

| 型           | 意味         | 呼び出し側のアクション |
| ------------ | ------------ | ---------------------- |
| `Ok<T>`      | 成功         | 値を使用               |
| `Err<Error>` | 失敗         | 必ず処理する           |
| `Err<null>`  | 既に処理済み | 任意                   |

## 型定義

```typescript
type Result<T, E> = { ok: true; value: T } | { ok: false; error: E };
```

## 適用タイミング

| シナリオ               | アプローチ          |
| ---------------------- | ------------------- |
| API 呼び出し           | 常に Result 型      |
| フォームバリデーション | エラー付きの Result |
| 内部ユーティリティ     | throw/catch でも可  |

## 強制

| ツール     | ルール                             |
| ---------- | ---------------------------------- |
| ESLint     | `eslint-plugin-return-type`        |
| TypeScript | `never` による exhaustive チェック |
