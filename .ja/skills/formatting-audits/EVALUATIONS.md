# formatting-auditsの評価

## 選択基準

このスキルをトリガーするキーワードとコンテキスト:

- **キーワード**: SOW review, Spec review, document review, design review, 90 points, scoring, grading, レビュー, 評価, 採点, 品質チェック
- **コンテキスト**: SOW/Spec品質評価、/validateコマンド、設計ドキュメント評価

## 評価シナリオ

### シナリオ1: SOW品質スコアリング

```json
{
  "skills": ["formatting-audits"],
  "query": "このSOWドキュメントの品質を評価して",
  "files": [".claude/workspace/planning/sow.md"],
  "expected_behavior": [
    "スキルが'SOW'と'評価'でトリガーされる",
    "100点満点のスコアリングシステムを適用",
    "評価: 正確性(25)、完全性(25)、明確性(25)、実行可能性(25)",
    "信頼度マーカー（✓/→/?）を使用",
    "90点閾値に対する合否を示す"
  ]
}
```

## スコアリング基準リファレンス

| カテゴリ | 最大点数 | 主なチェック項目 |
| --- | --- | --- |
| 正確性 | 25 | 信頼度マーカー、証拠、事実/推論の区別 |
| 完全性 | 25 | 受け入れ基準、リスク、依存関係 |
| 明確性 | 25 | 可読性、曖昧でない言語、構造 |
| 実行可能性 | 25 | 具体的なタスク、測定可能な成果、タイムライン |

## 手動検証チェックリスト

各シナリオ実行後:

- [ ] スキルがレビューキーワードで正しくトリガーされた
- [ ] 100点満点のスコアリングシステムが適用された
- [ ] 信頼度マーカー（✓/→/?）が使用された
- [ ] 90点合格閾値が参照された
- [ ] 具体的な改善項目がリストされた
- [ ] 実行可能なフィードバックが提供された

## ベースライン比較

### スキルなし

- 主観的なドキュメントレビュー
- スコアリングシステムなし
- 評価基準が一貫しない
- 信頼度マーカーの欠如

### スキルあり

- 客観的な100点満点スコアリング
- 構造化された4カテゴリ評価
- 90点品質ゲート
- マーカー付きの証拠ベース評価
- 実行可能な改善フィードバック
