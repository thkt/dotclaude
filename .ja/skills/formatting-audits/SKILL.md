# レビュー形式 - 設計ドキュメント採点

SOW/Specドキュメント品質評価のための100点満点採点。

## 採点システム

| 項目       | スコア | フォーカス   | 基準                         |
| ---------- | ------ | ------------ | ---------------------------- |
| 正確性     | 0-25   | 事実 vs 推論 | ✓/→/?マーカー、証拠パス      |
| 完全性     | 0-25   | カバレッジ   | テスト可能な基準、リスク     |
| 関連性     | 0-25   | 整合性       | 目標 ↔ ソリューション、YAGNI |
| 実行可能性 | 0-25   | 実装可能性   | 具体的なステップ、実現可能性 |

## 判定基準

| スコア | 判定        | アクション         |
| ------ | ----------- | ------------------ |
| 90-100 | PASS        | 次のフェーズへ進行 |
| 70-89  | CONDITIONAL | 修正後に再レビュー |
| 0-69   | FAIL        | 大幅な修正が必要   |

## 出力形式

```markdown
## 📋 レビュー結果

### 合計スコア: {total}/100 {✅/⚠️/❌}

| 項目       | スコア | マーカー |
| ---------- | ------ | -------- |
| 正確性     | /25    | ✓/→/?    |
| 完全性     | /25    | ✓/→/?    |
| 関連性     | /25    | ✓/→/?    |
| 実行可能性 | /25    | ✓/→/?    |

### 次のアクション

✅ PASS: /code
⚠️ CONDITIONAL: 修正して再レビュー
❌ FAIL: /thinkで再計画
```
