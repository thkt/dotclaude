# SOW/Spec 品質ゲート

バイナリゲート: specはCCが質問せず実装可能か、そうでないかのいずれか。数値スコアなし。

## アウトカム基準

ゴール: 全 FR / AC から一意のテストケースが書けること。エスカレーションなしで
CCがテストを書けないなら NotReady。

判定ルール:「この要件から今すぐ観察可能なアサーションで Given/When/Then を書けるか？」
書けない → Blocker。

## 優先度

| レベル | 意味                                                      | ゲート挙動       |
| ------ | --------------------------------------------------------- | ---------------- |
| P0     | 決定欠落または観察不能な振る舞い。CCが必ずエスカレする    | Block (NotReady) |
| P1     | 曖昧だがCCは推論可能。ドリフトリスクあり                  | Warn             |
| P2     | 品質問題、実装には影響しない                              | Info             |

具体的トリガーは `validating-specs`（チェック単位）と `sow-spec-reviewer`（ドキュメント単位）
に定義。優先度の割り当てルールは、上記の判定を個別findingに対して再適用すること。

## ゲート

| 条件          | 判定     | アクション                         |
| ------------- | -------- | ---------------------------------- |
| P0 = 0        | Ready    | 実装に進む                         |
| P0 ≥ 1        | NotReady | P0 findingsを解消後、再度ゲート実行 |

P1/P2 findingsは可視化のために報告されるが、ゲートをブロックしない。

## Finding フォーマット

各findingは Priority + Fix例 を必須で持つ。Fix例のないfindingは実行不能で、
レビュアーエラーとして拒否する。

| フィールド | 要件                                                               |
| ---------- | ------------------------------------------------------------------ |
| Priority   | P0 / P1 / P2                                                       |
| Location   | ドキュメント:行 またはセクション                                   |
| CC Impact  | CCがこれを読んだときに何が起きるか（エスカレ / ドリフト / 問題なし）|
| Fix        | 具体的な書き換え例。「これを明確化」のような抽象指示は拒否         |

## レガシーフォーマット処理

テンプレート拡張前に書かれたドキュメントは、新規追加された列が欠落することがある。

| 状態                                              | 処理                                        |
| ------------------------------------------------- | ------------------------------------------- |
| 列はテーブルに存在、値が空欄                      | Priority ルール通り finding（Check 8 等）   |
| 列自体がテーブル構造に存在しない                  | 列チェックをスキップ。P2「レガシーフォーマット — 列 X 欠落、移行推奨」を発行 |

テンプレート変更後、既存ドキュメントが全て NotReady になるのを防ぐ。
`validating-specs` と reviewer エージェントの全ての列完全性チェックに適用。

## 委譲

| 関心事                                          | スキル / エージェント       |
| ----------------------------------------------- | --------------------------- |
| AC-FRトレーサビリティ、整合性                   | `validating-specs`          |
