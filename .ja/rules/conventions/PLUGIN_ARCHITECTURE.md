# プラグインアーキテクチャ

Claude Codeプラグインアーキテクチャを維持するためのルール。

## ルール

| ルール           | ガイドライン                                      |
| ---------------- | ------------------------------------------------- |
| モノリシック維持 | marketplace.jsonで `source: "./"` を維持          |
| 相互参照保持     | skills/、rules/、agents/ 間の相互参照を破壊しない |
| 外部共有制限     | 個別プラグインの外部化をしない                    |

## 根拠

プラグインシステムはロード時にキャッシュを行い、境界を越えた `shared/` ディレクトリアクセスを防止する。

| 制約               | 理由                                       |
| ------------------ | ------------------------------------------ |
| 外部共有は不可能   | プラグインは境界外のファイルを参照できない |
| 相互参照は重要     | 390以上の参照が存在。破壊すると障害が発生  |
| モノリシックは安定 | /think, /code, /audit で動作実績あり       |

## 適用タイミング

| シナリオ                         | アクション             |
| -------------------------------- | ---------------------- |
| 新しいプラグイン追加             | `source: "./"` を使用  |
| 外部共有を検討                   | しない - 制限を説明    |
| プラグイン構造のリファクタリング | すべての相互参照を保持 |
| コンポーネント間のファイル移動   | すべての参照を更新     |

## 構造

```text
.claude/
├── .claude-plugin/
│   └── marketplace.json    # すべてのプラグインは source: "./"
├── skills/                 # 教育的コンテンツ
├── rules/                  # 強制ガイドライン
├── agents/                 # 特化エージェント
├── commands/               # ユーザー向けコマンド
└── [上記すべての間の相互参照]
```

## アンチパターン

### Bad: プラグイン分離を試みる

```json
{
  "plugins": [
    {
      "name": "core-rules",
      "source": "git://example.com/rules.git" // 動作しない
    }
  ]
}
```

| 問題                 | 結果                     |
| -------------------- | ------------------------ |
| skills/ への相互参照 | 失敗                     |
| プラグインキャッシュ | 共有リソースが含まれない |
| ワークフロー         | 破壊される               |
