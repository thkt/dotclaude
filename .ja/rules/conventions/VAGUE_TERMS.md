# 曖昧用語

ソフトウェア開発で曖昧になる用語。使用時は具体化必須。

対象: テスト記述、Issue/PR本文、Spec/SOW、コードコメント、技術ドキュメント

## 具体的に定義すべし

そのままでは検証不能。条件と期待結果を明示する。

| 禁止語                       | 問題               | 代替                         |
| ---------------------------- | ------------------ | ---------------------------- |
| 正しい / 正常 / 異常         | 基準が不明         | 条件と期待結果を明示         |
| 通常 / 普通 / 標準 / 一般的  | 誰の通常か不明     | デフォルト値や前提条件を明示 |
| 期待値 / 意図通り / 仕様通り | 参照先が空         | 具体的な値や振る舞いを記述   |
| 処理（名詞/動詞）            | 何をしているか不明 | 変換、検証、保存など具体動詞 |
| データ / 情報                | 何のデータか不明   | ユーザー名、注文リストなど   |
| 問題なく                     | 何を確認したか不明 | 確認した条件を列挙           |

## 比較対象を明示すべし

基準点なしでは意味が定まらない。

| 禁止語              | 問題         | 代替                       |
| ------------------- | ------------ | -------------------------- |
| 大量 / 少量         | 閾値が不明   | 具体数値か閾値を記述       |
| 大きい / 小さい     | 基準が不明   | 具体サイズか比較対象を記述 |
| 最新 / もっとも古い | スコープ不明 | スコープとソート条件を明示 |
