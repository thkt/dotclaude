# Vague Terms

ソフトウェア開発における曖昧な用語を、使用する前に具体化する。対象はテスト記述、Issue / PR 本文、コミットメッセージ、Spec / SOW、コードコメント、技術ドキュメント。

## 具体的に定義する

| 禁止用語                                    | 問題                 | 置換                       |
| ------------------------------------------- | -------------------- | -------------------------- |
| correct / normal / abnormal                 | 基準なし             | 条件と期待結果             |
| usual / ordinary / standard / general       | 誰にとっての標準か   | デフォルト値または前提条件 |
| expected value / as intended / as specified | 参照が空             | 具体的な値または振る舞い   |
| process (noun / verb)                       | どんな操作か         | 変換、検証、永続化など     |
| data / information                          | どんなデータか       | ユーザー名、注文一覧など   |
| without issues                              | 実際に何を確認したか | 確認した条件を列挙         |
| appropriately / properly                    | 何をもって適切か     | 具体的な操作と判断基準     |
| robust / leverage / delve                   | 何がどう優れるのか   | 具体的な性質や操作         |

## 状態比較の基準

| 禁止用語             | 問題     | 置換                         |
| -------------------- | -------- | ---------------------------- |
| large / small amount | 閾値なし | 具体的な数値または閾値       |
| large / small (size) | 基準なし | 具体的なサイズまたは比較対象 |
| latest / oldest      | 範囲不明 | 範囲とソートキー             |
| fast / slow          | 閾値なし | 計測値と閾値                 |

## スコープを明示する

モデルは、ある項目への指示を別の項目へ一般化しない。また、明示されていないスコープを勝手に推論しない。ルール / Spec / プロンプトを書くときは、意図したスコープを明示する。

| 意図       | 書き方                               |
| ---------- | ------------------------------------ |
| 例示リスト | A / B / C など (これらに限らない)    |
| 網羅リスト | A / B / C に限る                     |
| 広く適用   | 最初のセクションだけでなく全体が対象 |
