# Vague Terms

ソフトウェア開発における曖昧な用語。使用前に具体化する。

適用先: テスト記述、Issue/PR 本文、Spec/SOW、コードコメント、技術ドキュメント。

## 具体的に定義する

そのままでは検証不能。条件と期待結果を明示する。

| 禁止用語                                    | 問題                 | 置換                       |
| ------------------------------------------- | -------------------- | -------------------------- |
| correct / normal / abnormal                 | 基準なし             | 条件と期待結果             |
| usual / ordinary / standard / general       | 誰の usual か        | デフォルト値または前提条件 |
| expected value / as intended / as specified | 参照が空             | 具体的な値または振る舞い   |
| process (noun/verb)                         | どんな操作か         | 変換、検証、永続化など     |
| data / information                          | どんなデータか       | ユーザー名、注文一覧など   |
| without issues                              | 実際に何を確認したか | 確認した条件を列挙         |

## 状態比較の基準

参照点なしには無意味。

| 禁止用語             | 問題     | 置換                         |
| -------------------- | -------- | ---------------------------- |
| large / small amount | 閾値なし | 具体的な数値または閾値       |
| large / small (size) | 基準なし | 具体的なサイズまたは比較対象 |
| latest / oldest      | 範囲不明 | 範囲とソート キー            |
