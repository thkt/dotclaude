# ドキュメントルール

Claude Codeエコシステムにおける一貫性のあるドキュメントのガイドライン。

## コア原則

| 原則             | ガイドライン                                  |
| ---------------- | --------------------------------------------- |
| 完全性より明確性 | オッカムの剃刀を適用                          |
| EN/JP同期        | 両版の構造は一致が必須                        |
| 循環参照なし     | スキル: 1レベル、ルール/ドキュメント: 3レベル |
| ASCIIよりMermaid | ASCII図ではなくMermaidダイアグラムを使用      |

## ディレクトリ構造

```text
/rules/
├── core/           # AI動作（hook注入）
├── conventions/    # ドキュメント規約
├── development/    # 実践的手法
└── workflows/      # ワークフローガイド & モジュール化ルール

/skills/            # ナレッジベース（SOLID, DRY等）
```

## 参照ルール

| ルール       | ガイドライン                                    |
| ------------ | ----------------------------------------------- |
| 最大深度     | 3レベル                                         |
| 循環参照     | 禁止（A → B → A）                               |
| 冗長な参照   | CLAUDE.mdで既にロードされるファイルを参照しない |
| 必要時参照   | 現在のコンテキストで必要な場合のみ参照          |
| 相対パス     | 同ディレクトリは`./`、1階層上は`../`            |
| 深いパス     | `../../`は許容だがREADMEに記載                  |
| 深いパス注記 | ファイル再編成後に確認すること                  |

## 更新チェックリスト

| 操作 | ステップ                          |
| ---- | --------------------------------- |
| 追加 | EN+JP作成 → CLAUDE.mdに追加       |
| 変更 | grep参照 → EN+JP更新 → リンク確認 |
| 移動 | 参照検索 → EN+JP移動 → 参照更新   |

## 記述標準

| 要素     | 標準                                           |
| -------- | ---------------------------------------------- |
| ヘッダー | 文頭大文字                                     |
| 例       | 悪い vs 良い の比較                            |
| コメント | AI文書にはテキストラベル、人間出力には絵文字OK |

## 簡潔化ガイドライン

| 原則               | 適用                                                              |
| ------------------ | ----------------------------------------------------------------- |
| DRY                | 同じ情報は1箇所のみ（Metrics + Checklist + Rules に重複させない） |
| 閾値のインライン化 | 閾値はルールに直接含める、別テーブルにしない                      |
| 最小限の例         | パターンを示す、完全な実装は不要                                  |
| 冗長なコメント不要 | 他で説明済みならコードブロック内で繰り返さない                    |
