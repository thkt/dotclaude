# サブエージェント定義

`.claude/agents/` 配下のサブエージェントファイルの規約。

## メモリ

`memory: project` フロントマターでセッションをまたいだプロジェクト固有パターンの保持を有効にする。

### 選定基準

3条件すべてを満たす場合のみ付与。

| 条件             | 説明                                         | 例                       |
| ---------------- | -------------------------------------------- | ------------------------ |
| 頻度             | セッションをまたいで繰り返し呼び出される     | 毎回の audit で呼び出し  |
| プロジェクト依存 | 出力品質がプロジェクト固有の知識に依存する   | 命名規則、許可パターン   |
| 学習効果         | メモリが偽陽性の削減や一貫性の向上に寄与する | 既知の例外の再報告を停止 |

### 除外対象

デフォルトは `memory: project` (30/35 エージェント)。

| エージェント           | 理由                                                       |
| ---------------------- | ---------------------------------------------------------- |
| unit-implementer       | チームエージェント、セッション内のみで動作                 |
| progressive-integrator | チームエージェント、セッション内のみで動作                 |
| devils-advocate-audit  | 他レビュアーの findings を受け取り、メモリがバイアスの原因 |
| devils-advocate-design | 同上、客観性を維持する必要あり                             |
| evidence-verifier      | 先入観なしで findings を検証する必要あり                   |

## レビュアー設計パターン

| 要素           | 規約                                        |
| -------------- | ------------------------------------------- |
| スコープ       | 単一ドメイン、単一責務、約60行              |
| フロントマター | `tools`, `model`, `skills`, `context: fork` |
| 出力           | 構造化 Markdown（`findings` + `summary`）   |

## セキュリティ特性

| 特性     | 値                                       |
| -------- | ---------------------------------------- |
| 保存対象 | 分析パターン、規約、例外リスト           |
| 非保存   | 生のソースコード、シークレット、認証情報 |
| 配置     | `.claude/agent-memory/`                  |

配置先は `.gitignore` に `/.claude/agent-memory/` として登録。
