# Agent Memory Convention

## `memory: project` フロントマター

エージェントにプロジェクト固有のパターンをセッション横断で記憶させる。

## 選定基準

3条件すべてを満たすエージェントのみ付与する。

| 条件             | 説明                                     | 例                         |
| ---------------- | ---------------------------------------- | -------------------------- |
| 頻度             | セッション横断で繰り返し実行される       | audit のたびに毎回呼ばれる |
| プロジェクト依存 | 出力品質がプロジェクト固有の知識に依存   | 命名規則、許容パターン     |
| 学習効果         | メモリで誤検知削減や一貫性向上が見込める | 既知の例外を再報告しない   |

## 現在の付与一覧

| エージェント            | 学習内容                                                |
| ----------------------- | ------------------------------------------------------- |
| security-reviewer       | 許容済みセキュリティパターン、既知の誤検知              |
| code-quality-reviewer   | プロジェクト命名規則、許容される複雑度例外              |
| type-safety-reviewer    | 意図的な any 型、型アサーションの許容パターン           |
| silent-failure-reviewer | 意図的なエラー握りつぶし、graceful degradation パターン |
| commit-generator        | コミットメッセージスタイル、スコープ命名規則            |

## 非付与の理由

| カテゴリ                             | 理由                                                            |
| ------------------------------------ | --------------------------------------------------------------- |
| compound reviewers                   | オーケストレーターのみ、分析は行わない                          |
| devils-advocate                      | 他レビュアーの知見を受ける立場、独自メモリは矛盾を生む          |
| performance / accessibility reviewer | WCAG・Web Vitals 等の普遍的基準を適用、プロジェクト固有性が低い |
| generators（commit 以外）            | 実行頻度が低い、テンプレートで十分                              |

## セキュリティ特性

| 属性       | 値                                       |
| ---------- | ---------------------------------------- |
| 保存対象   | 分析パターン、規則、例外リスト           |
| 保存しない | 生のソースコード、秘密情報、認証情報     |
| 保存先     | `.claude/agent-memory/` (gitignore 推奨) |
