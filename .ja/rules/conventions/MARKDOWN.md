# Markdown Conventions

`.claude/` 配下の Markdown ファイルに対する規約。

## ファイルスコープ

| スコープ   | パス                                                              |
| ---------- | ----------------------------------------------------------------- |
| LLM-facing | `CLAUDE.md`, `agents/**`, `skills/**`, `rules/**`, `workspace/**` |
| Human      | `docs/**`, `README.md`                                            |

`.ja/` の対訳コピーは対応元のスコープに従う。

## 記号

| 記号 | 用途                                | 例                                                                      |
| ---- | ----------------------------------- | ----------------------------------------------------------------------- |
| `/`  | 並列列挙 (AND/OR)                   | `Safety First / Output Verifiability`                                   |
| `.`  | 独立したルールのセパレータ          | `Check scope. Do not skip`                                              |
| `()` | 補足条件のみ                        | `Skip for follow-up (same session)`                                     |
| `→` | 順序 (フォールバックまたはステップ) | `README → package.json → ask user` / `Observe → analyze → conclude` |

## 禁止

| パターン               | 適用範囲   | 理由                                                       |
| ---------------------- | ---------- | ---------------------------------------------------------- |
| `≠`                   | All        | 肯定形に書き換える。`is not X` を使う                      |
| `()` で対比            | All        | `.` で文を分割する                                         |
| `—` (em-dash)         | Prose      | `.` で別文に分割する                                       |
| `ANY` / `ALL` を大文字 | All        | 通常の大小文字を使う                                       |
| `**bold**`             | LLM-facing | トークンの無駄。テーブルやセクションを使う                 |
| Prose 中の絵文字       | LLM-facing | トークンの無駄。ユーザーに見える絵文字は例外               |
| Unicode 装飾           | All        | prose では ASCII。テーブル内では Unicode 記号を許可        |
| 太字先頭の bullet      | All        | `- **Label:** text` パターン。スキャンに役立つ場合のみ使う |

## 疑似見出し

行を `:` で終えてテーブル、リスト、セクションを導入しない。`### Foo` のような見出しに昇格させるか、prose に書き換える。インライン コロン (例: `File: foo.md`、テーブル セル、`- Name: bar`) は許可。

| 違反                                    | 修正                         |
| --------------------------------------- | ---------------------------- |
| `Only excluded:` の後にテーブル         | `### Exclusions` + テーブル  |
| `When unable to verify:` の後にステップ | `#### When unable to verify` |

## テーブル配置

説明 (ルール、コンテキスト、例外) はテーブルの前に置く。テーブル後に段落を続けない。テーブルは列挙のためのもので、prose の分割には使わない。

## Prose 折り返し

段落内に物理改行を入れない。各段落を 1 行で書く。ツールがソフトラップを処理する。

## 参照深度

| 範囲       | 最大階層 |
| ---------- | -------- |
| Skills     | 1        |
| Rules/Docs | 3        |

## 禁止参照

| パターン                 | 理由                                         |
| ------------------------ | -------------------------------------------- |
| 循環 (A → B → A)       | 解決不能な依存を生む                         |
| CLAUDE.md に既に含まれる | グローバルに読み込まれるファイルは再参照不要 |
| 不要な参照               | 現コンテキストに必要な場合のみ参照する       |

## 適用範囲外

- コードブロック / インラインコード
- ADR の日本語 prose
