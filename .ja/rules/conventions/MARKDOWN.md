# Markdown Conventions

`.claude/` 配下の Markdown ファイルに対する規約。

## ファイルスコープ

`.ja/` 配下を正とし、編集は `.ja/` を起点に行ったうえで、同一コミットで英語側ファイルへ反映する。反映先は `.ja/` を除いたパスとし、スコープの判定も同様に `.ja/` を除いたパスで行う。翻訳ではなく同一内容のコピーとして扱い、スクリプトなどの非 Markdown ファイルも含めて同じミラー規律に従う。例外は workflows/build.js のような LLM 向け prompt を内包する workflow script で、散文 (コメント / prompt / メッセージ文字列) は翻訳し、コード構造・識別子・stopped 値・JSON キーは同一に保つ。cp での同期はこの例外ファイルを英語で潰すため使わない。

| スコープ     | パス                                                              |
| ------------ | ----------------------------------------------------------------- |
| LLM-facing   | `CLAUDE.md`, `agents/**`, `skills/**`, `rules/**`, `workspace/**` |
| Human-facing | `docs/**`, `README.md`                                            |

## 記号

| 記号 | 用途                            | 例                                    |
| ---- | ------------------------------- | ------------------------------------- |
| `/`  | AND 並列列挙                    | `Safety First / Output Verifiability` |
| `.`  | 独立したルールのセパレータ      | `Check scope. Do not skip`            |
| `()` | 補足条件のみ                    | `Skip for follow-up (same session)`   |
| `>`  | 優先順位 (左を優先、不可なら右) | `CLI tool > built-in equivalent`      |
| `→`  | ステップ列挙                    | `Observe → analyze → conclude`        |
| `§`  | セクション参照                  | `phase.md § Gate ルール`              |
| `+`  | 構成要素の合成                  | `root causes + Gate decision`         |

## インラインコード

リテラルや識別子であることは、バッククォートで囲う理由にならない。削除したときに誤読されるかどうかで利用を判断する。`<branch>` のような angle-bracket は例外とする。

| 残す                                       | 削る                               |
| ------------------------------------------ | ---------------------------------- |
| 識別子やコマンドが通常テキストと混在する文 | 列名から型が自明な同型セルの列     |
| プレースホルダを含むトークン               | 強調目的のみの囲み                 |
| エスケープ / 記号として識別が必要          | 記号で視覚的に区別されるリスト項目 |

## 禁止

| パターン                   | 適用範囲   | 修正                                                           |
| -------------------------- | ---------- | -------------------------------------------------------------- |
| `≠`                        | All        | 肯定形に書き換えるか `is not X` を使う                         |
| `()` で対比                | All        | `.` で文を分割する                                             |
| `—`                        | All        | `.` で文に分割するか、文脈に応じて `,` / `:` / `-` に置換      |
| 行末 `:` や `Label: value` | All        | 見出しかテーブルに昇格、または 散文化。リテラルのコロンは許可  |
| 強調目的の全大文字         | All        | 通常の大小文字を使う                                           |
| 段落内の物理改行           | All        | 1 段落 1 行で書く。ソフトラップはツールが処理                  |
| テーブル後の段落           | All        | 説明 (ルール、コンテキスト、例外) はテーブル前に置く           |
| `**bold**`                 | LLM-facing | テーブルやセクションを使う。太字先頭の bullet はテーブルに変換 |
| 文中の絵文字               | LLM-facing | 削除。ユーザーに見える絵文字は例外                             |
| Unicode 装飾               | Prose      | ASCII を使用する                                               |

## 参照深度

| 範囲         | 最大階層 |
| ------------ | -------- |
| Skills       | 1        |
| Rules / Docs | 3        |

## セクション語彙

連番手順を持つ skill では、最上位の連番ユニットは常に `## Phase N` とする。`Step` は 1 つの Phase をさらに連番分割する第 2 階層 (単一 Phase のテーブル内の番号列など) にのみ使う。連番でない準備・参照セクションは固有名 (Input / Setup / Output) を保ち、Phase ではない。独立した列挙チェック (観点 / カテゴリ) も Phase ではない。

## 禁止参照

| パターン                 | 理由                                           |
| ------------------------ | ---------------------------------------------- |
| 循環 (A → B → A)         | 解決不能な依存を生む                           |
| CLAUDE.md に既に含まれる | グローバルに読み込まれるファイルは再参照不要   |
| 予防的な参照             | 現コンテキストで読む必要があるものだけ参照する |

## 適用範囲外

- コードブロック / インラインコードの内部
