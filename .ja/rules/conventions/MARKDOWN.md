# Markdown規約

`.claude/` 配下の Markdown ファイルの規約。

## ファイルスコープ

| スコープ | パス                                                              |
| -------- | ----------------------------------------------------------------- |
| LLM向け  | `CLAUDE.md`, `agents/**`, `skills/**`, `rules/**`, `workspace/**` |
| 人間向け | `docs/**`, `README.md`                                            |

`.ja/` のバイリンガル版は対応する英語版のスコープに従う。

## 記号

| 記号 | 用途                            | 例                                                            |
| ---- | ------------------------------- | ------------------------------------------------------------- |
| `/`  | 並列列挙 (AND/OR)               | `Safety First / Output Verifiability`                         |
| `.`  | 独立したルールの区切り          | `スコープ確認。省略しない`                                    |
| `()` | 補足条件のみ                    | `フォローアップは省略 (同一セッション)`                       |
| `→`  | 順序 (フォールバックまたは手順) | `README → package.json → ユーザー確認` / `観察 → 分析 → 結論` |

## 使わない

| パターン             | 適用範囲 | 理由                                             |
| -------------------- | -------- | ------------------------------------------------ |
| `≠`                  | 全て     | 肯定形に書き換え。`〜ではない` を使う            |
| `()` の対比用法      | 全て     | `.` で分離                                       |
| `—` (em-dash)        | 散文     | `.` で独立文に分離                               |
| `ANY` / `ALL` 大文字 | 全て     | 通常の書き方にする                               |
| `**bold**`           | LLM向け  | トークン無駄。テーブル・セクションを使う         |
| 散文中の絵文字       | LLM向け  | トークン消費。ユーザー可視の絵文字は例外         |
| Unicode装飾          | 全て     | 散文は ASCII。テーブル内は Unicode記号可         |
| 太字先頭箇条書き     | 全て     | `- **Label:** text` パターン。スキャン補助に限定 |

## 疑似見出し

テーブル・リスト・セクションを導入する目的で行末に `:` を使わない。正式な見出し記法 (`### Foo`) に昇格するか、散文で書き換える。インラインの `:` は許容 (例: `File: foo.md`、テーブルセル、`- 名前: bar`)。

| 違反                         | 修正                      |
| ---------------------------- | ------------------------- |
| `除外対象:` の後にテーブル   | `### 除外対象` + テーブル |
| `検証不能な場合:` の後に手順 | `#### 検証不能な場合`     |

## テーブル配置

説明 (ルール、文脈、例外) はテーブルの前に書く。テーブルの後に段落を続けない。テーブルは列挙のためのもので、散文を分割するためのものではない。

## 段落の改行

段落内に物理的な改行を入れない。1段落は1行で書く。折り返しはツールに任せる。

## 参照の深度

| スコープ              | 最大レベル |
| --------------------- | ---------- |
| スキル                | 1          |
| ルール / ドキュメント | 3          |

## 参照の禁止事項

| パターン                 | 理由                                         |
| ------------------------ | -------------------------------------------- |
| 循環参照 (A → B → A)     | 依存を解決できなくなる                       |
| CLAUDE.md で既にロード済 | グローバルにロード済みのファイルは再参照不要 |
| 不要な参照               | 現在のコンテキストで必要な場合のみ参照する   |

## 対象外

- コードブロック / インラインコード
- ADR の日本語散文
