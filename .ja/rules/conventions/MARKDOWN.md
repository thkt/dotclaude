# Markdown Conventions

`.claude/` 配下の Markdown ファイルに対する規約。

## ファイルスコープ

| スコープ     | パス                                                              |
| ------------ | ----------------------------------------------------------------- |
| LLM-facing   | `CLAUDE.md`, `agents/**`, `skills/**`, `rules/**`, `workspace/**` |
| Human-facing | `docs/**`, `README.md`                                            |

`.ja/` の対訳コピーは対応元のスコープに従う。

## 記号

| 記号 | 用途                         | 例                                                                  |
| ---- | ---------------------------- | ------------------------------------------------------------------- |
| `/`  | 並列列挙 (AND/OR)            | `Safety First / Output Verifiability`                               |
| `.`  | 独立したルールのセパレータ   | `Check scope. Do not skip`                                          |
| `()` | 補足条件のみ                 | `Skip for follow-up (same session)`                                 |
| `→`  | フォールバックまたはステップ | `README → package.json → ask user` / `Observe → analyze → conclude` |

## インラインコード

バッククォートは「消したら誤読されるか」で判断する。誤読されないなら囲わない。リテラルや識別子であることは、それ自体では囲う理由にならない。`<branch>` のような angle-bracket プレースホルダーはエスケープせずバッククォートで囲う。

| 残す                                                          | 削る                                          |
| ------------------------------------------------------------- | --------------------------------------------- |
| 識別子と通常テキストが混在する文                              | 列内全要素が同じ型のセル                      |
| プレースホルダーを含むトークン (1 件でも含む列は列全体を維持) | 強調目的のみの囲み                            |
| エスケープ・記号として識別が必要                              | 列名で型が自明 (Tool 列にツール名のみ等)      |
| 文中のコマンド全体                                            | 記号で視覚的に区別されるリスト項目 (`[✓]` 等) |

## 禁止

| パターン               | 適用範囲   | 修正                                                           |
| ---------------------- | ---------- | -------------------------------------------------------------- |
| `≠`                    | All        | 肯定形に書き換えるか `is not X` を使う                         |
| `()` で対比            | All        | `.` で文を分割する                                             |
| `—` (em-dash)          | All        | `.` で文に分割するか、文脈に応じて `,` / `:` / `-` に置換      |
| `ANY` / `ALL` を大文字 | All        | 通常の大小文字を使う                                           |
| `**bold**`             | LLM-facing | テーブルやセクションを使う。太字先頭の bullet はテーブルに変換 |
| Prose 中の絵文字       | LLM-facing | 削除する。ユーザーに見える絵文字は例外                         |
| Unicode 装飾           | Prose      | ASCII を使用する                                               |

## 疑似見出し

行末の `:` でテーブル、リスト、セクションを導入しない。インラインの `Label: value` 型コロンも同様。どちらも構造のヒントであり、見出し (`### Foo`) かテーブルに昇格させるか prose に書き換える。リテラルのコロンは許可 (時刻 `14:30`、比率 `2:1`、URL、パス、テーブルセル)。

## テーブル配置

説明 (ルール、コンテキスト、例外) はテーブルの前に置く。テーブル後に段落を続けない。テーブルは列挙のためのもので、prose の分割には使わない。

## Prose 折り返し

段落内に物理改行を入れない。各段落を 1 行で書く。ツールがソフトラップを処理する。

## 参照深度

| 範囲       | 最大階層 |
| ---------- | -------- |
| Skills     | 1        |
| Rules/Docs | 3        |

## 禁止参照

| パターン                 | 理由                                           |
| ------------------------ | ---------------------------------------------- |
| 循環 (A → B → A)         | 解決不能な依存を生む                           |
| CLAUDE.md に既に含まれる | グローバルに読み込まれるファイルは再参照不要   |
| 予防的な参照             | 現コンテキストで読む必要があるものだけ参照する |

## 適用範囲外

- コードブロック / インラインコードの内部
