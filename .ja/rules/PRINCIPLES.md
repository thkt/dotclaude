## トリガー

メソッドチェーン >2 → デメテルの法則 /
1分で理解できない → 読みやすいコード / 複雑な解決策を先に → オッカムの剃刀 / 単一仮説 →
Strong Inference

## 競合解決

| 競合              | 解決策           |
| ----------------- | ---------------- |
| DRY vs 読みやすさ | 読みやすさが勝つ |
| SOLID vs シンプル | シンプルが勝つ   |
| 完璧 vs 動作      | 動作が勝つ       |
| 抽象 vs 具体      | 具体から始める   |

迷ったら: シンプル > 巧妙、具体 > 抽象、動作 > 完璧、明確 > DRY。

## プログレッシブエンハンスメント

動くようにする → 堅牢にする（エラー発生時） → 速くする（遅さが計測された時） → 柔軟にする（ユーザー要求時）
