## バックキャスティング

理想の最終状態を定義し、そこから逆算する。

1. Goal: 「完了」とはどう見えるか? (成果物ではなく、ユーザーの行動やシステムの状態)
2. Gap: 現状とその目標の間に何があるか?
3. Path: Gap から Goal への最小限のステップは?

## 競合解決

バックキャスティングは目標を定義し、オッカムの剃刀等の他原則は到達方法を統治する。

迷ったら: シンプル > 巧妙、具体 > 抽象、動作 > 完璧、読みやすさ > DRY。

オッカムの剃刀はアウトカムを達成するアプローチの中でもっともシンプルな方法を選ぶ原則。症状除去ではなくアウトカム達成が比較基準。シンプルさが出力品質を下げる場合は適用条件を満たさない。

## YAGNI境界

YAGNIは不要な機能や投機的コードパスを禁止する。同等コストでより良い構造を選ぶことは禁止しない。YAGNI境界とオッカムの剃刀が競合する場合、オッカムの剃刀が勝つ。

| ステップ | 基準                                                                  |
| -------- | --------------------------------------------------------------------- |
| ゲート   | 呼び出し箇所 >= 2 OR ドメイン自明 (auth, logging, error handling)     |
| 判定軸   | コスト同等 (行数, indirection depth, import数) → 協調変更の少ないほう |

## DRY

ゲート: 3回以上の重複。次に確認: 同じ知識か、似た構造か?

| タイプ     | 基準                                      | アクション |
| ---------- | ----------------------------------------- | ---------- |
| 同じ知識   | 1箇所の変更で全ての箇所が変わる必要がある | DRY を適用 |
| 似たコード | 各コピーが独立に進化する可能性がある      | 統合しない |

## プログレッシブエンハンスメント

動くようにする → 堅牢にする (エラー発生時) → 速くする (遅さが計測されたとき) → 柔軟にする (ユーザー要求時)
