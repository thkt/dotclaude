## トリガー

メソッドチェーン >2 → デメテルの法則 /
1分で理解できない → 読みやすいコード / 複雑な解決策を先に → オッカムの剃刀 / 単一仮説 →
Strong Inference / 協調変更が必要な呼び出し箇所 >= 2 → YAGNI境界

## 競合解決

迷ったら: シンプル > 巧妙、具体 > 抽象、動作 > 完璧、読みやすさ > DRY。

オッカムの剃刀は同等の結果を出すアプローチの中でもっともシンプルな方法を選ぶ原則。シンプルさが出力品質を下げる場合は適用条件を満たさない。

## YAGNI境界

YAGNIは不要な機能や投機的コードパスを禁止する。同等コストでより良い構造を選ぶことは禁止しない。

| ステップ | 基準                                                                  |
| -------- | --------------------------------------------------------------------- |
| ゲート   | 呼び出し箇所 >= 2 OR ドメイン自明（auth, logging, error handling）    |
| 判定軸   | コスト同等（行数, indirection depth, import数）→ 協調変更の少ないほう |

YAGNI境界とオッカムの剃刀が競合する場合、オッカムの剃刀が勝つ。

## プログレッシブエンハンスメント

動くようにする → 堅牢にする（エラー発生時）→ 速くする（遅さが計測されたとき）→ 柔軟にする（ユーザー要求時）
