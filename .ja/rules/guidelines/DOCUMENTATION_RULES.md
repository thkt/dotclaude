# ドキュメントルール

Claude Codeエコシステムにおける一貫性のあるドキュメントのガイドライン。

## コア原則

1. **完全性より明確性** - オッカムの剃刀を適用
2. **EN/JP同期** - 両版の構造は一致が必須
3. **循環参照なし** - 最大3レベルまで
4. **ASCIIよりMermaid** - ASCII図ではなくMermaidダイアグラムを使用

## ディレクトリ構造

```text
/rules/
├── core/           # AI動作（hook注入）
├── guidelines/     # ドキュメントガイドライン
├── development/    # 実践的手法
└── commands/       # コマンド固有ルール

/skills/            # ナレッジベース（SOLID, DRY等）
```

**配置**: 原則 → guidelines/, 実践 → development/, コマンド → commands/

## 参照ルール

| ルール       | ガイドライン                   |
| ------------ | ------------------------------ |
| 最大深度     | 3レベル                        |
| 循環参照     | 禁止（A → B → A）              |
| セクション名 | `## Related Principles` を使用 |
| コア原則     | CLAUDE.mdから直接参照          |

## 言語同期

- EN: `/path/to/FILE.md`
- JP: `/.ja/path/to/FILE.md`

**ADR例外**: デフォルトで日本語出力、`.ja/` 不要。

## 更新チェックリスト

| 操作 | ステップ                          |
| ---- | --------------------------------- |
| 追加 | EN+JP作成 → CLAUDE.mdに追加       |
| 変更 | grep参照 → EN+JP更新 → リンク確認 |
| 移動 | 参照検索 → EN+JP移動 → 参照更新   |

## 記述標準

- ヘッダー: 文頭大文字
- 例: 悪い vs 良い の比較
- 図表: MermaidまたはテーブルASCIなし
- コメント: AI文書にはテキストラベル、人間出力には絵文字OK
