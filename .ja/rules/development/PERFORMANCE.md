# Performance Guidelines

## コンテキスト管理

| 戦略       | ガイドライン                | 根拠                                 |
| ---------- | --------------------------- | ------------------------------------ |
| MCP tools  | プロジェクトごと ≤10 有効化 | 200k→70k コンテキスト縮小を防止      |
| Plugins    | 必要なものだけ有効化        | 各プラグインがトークンオーバーヘッド |
| `/compact` | コンテキスト >70% で使用    | 会話履歴を保持                       |
| `/fork`    | 並列タスクに使用            | コンテキスト汚染を回避               |

## モデル選択

| タスクタイプ           | 推奨        | 根拠                 |
| ---------------------- | ----------- | -------------------- |
| 複雑なアーキテクチャ   | Opus        | 深い推論が必要       |
| 標準的な開発           | Sonnet      | 速度と品質のバランス |
| 簡単な修正、リファクタ | Haiku       | 高速、低コスト       |
| コードレビュー         | Sonnet/Opus | 品質重視             |

## フロントエンドパフォーマンス

| 指標           | 目標値        | ツール                  |
| -------------- | ------------- | ----------------------- |
| LCP            | <2.5s         | Lighthouse              |
| INP            | <200ms        | Lighthouse              |
| CLS            | <0.1          | Lighthouse              |
| バンドルサイズ | <200KB (初期) | webpack-bundle-analyzer |

## アンチパターン

| パターン                  | 問題点             | 代替案                   |
| ------------------------- | ------------------ | ------------------------ |
| JSX 内インライン関数      | 毎レンダーで再生成 | useCallback              |
| 大きな useEffect 依存配列 | 頻繁な再実行       | effect を分割            |
| 未最適化の画像            | LCP 低下           | next/image, lazy loading |
| ブロッキングスクリプト    | TTI 低下           | async/defer              |
| 大きな初期バンドル        | FCP 低下           | Code splitting           |

## React 最適化

| 技法         | 使用場面                               | 効果       |
| ------------ | -------------------------------------- | ---------- |
| React.memo   | 純粋なプレゼンテーションコンポーネント | Medium     |
| useMemo      | 高コストな計算                         | Medium     |
| useCallback  | メモ化された子へのコールバック         | Low-Medium |
| 仮想化       | >100 アイテムのリスト                  | High       |
| 遅延読み込み | ルート、重いコンポーネント             | High       |

## ビルド最適化

| 戦略           | 実装方法                       |
| -------------- | ------------------------------ |
| Tree shaking   | ES modules, sideEffects: false |
| Code splitting | Dynamic imports, route-based   |
| 圧縮           | Brotli/gzip                    |
| キャッシング   | ファイル名にコンテンツハッシュ |
