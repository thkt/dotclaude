# よく踏む地雷

外部ツール・言語のエッジケース制約。実装前に対象の命名・フォーマット制約を確認する。エラー発生時は制約違反を最初に疑い、横展開チェックする。

| #   | ツール/コンテキスト  | 地雷                                       | 確認方法                               |
| --- | -------------------- | ------------------------------------------ | -------------------------------------- |
| 1   | Git ブランチ/タグ    | `[` `]` `~` `^` `:` `\` `..` スペース禁止  | `git check-ref-format --branch "name"` |
| 2   | npm パッケージ名     | 214文字以下、小文字強制、`@scope/` 形式    | `npm info package-name`                |
| 3   | Docker イメージ/タグ | 小文字のみ、`/` は namespace、`:` は tag   | 公式ドキュメント確認                   |
| 4   | 正規表現エンジン     | PCRE vs ERE vs BRE vs JavaScript 差異      | ツールの `--help` で使用エンジン確認   |
| 5   | macOS ファイル名     | `:` 禁止、デフォルトで大文字小文字区別なし | `touch "test:file"` でエラー確認       |
| 6   | semver バージョン    | `MAJOR.MINOR.PATCH` 必須、先頭 `v` は別物  | `npx semver "version"` で検証          |
| 7   | 環境変数名           | 英数字と `_` のみ、数字始まり禁止          | `export` で既存確認、POSIX準拠         |
| 8   | JSON キー名          | ダブルクォート必須、trailing comma 禁止    | `jq . file.json` でパースエラー確認    |
