# Security

guardrails (静的検出) を補完する。検証の欠落、認可チェックの欠落、ミドルウェアの欠落をカバーする。

## Input

- すべての API エンドポイント入力をクライアント側とサーバー側でスキーマで検証する MUST
- 検索クエリと自由記述フィールドに長さ制限を適用する MUST
- クライアント由来の型情報を DB やその他の信頼境界へ直接渡さない MUST NOT

## Access Control

- リソースアクセス時に所有権を検証する (要求者がターゲットを所有する) MUST
- アクセス制御に URL パラメータや body の数値 ID のみを使わない (IDOR リスク) MUST NOT
- 認証ミドルウェアはエンドポイント単位ではなくルートグループにマウントする MUST
- 未認証ユーザーにリソース存在を露呈しない (403 より 404 を選ぶ) MUST NOT
- 認証エンドポイント (login, register, password-reset) にレート制限を適用する MUST
- 外部 API 呼び出しを誘発するエンドポイントにリクエスト制限を適用する MUST

## Secrets

- 必須 env var が欠落したらエラーを投げる MUST。ハードコード値にフォールバックしてはならない
- セッション ID、トークン、ワンタイムコードには暗号学的に安全な乱数源を使う MUST

## Output

- レスポンスにスタックトレース、内部パス、DB 詳細を含めない MUST NOT。汎用メッセージを返し、詳細はサーバー側にのみ記録する
- 認証失敗の理由を区別しない MUST NOT (例: 「パスワードが違う」と「メールが見つからない」を区別しない)
