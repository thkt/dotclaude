# プログレッシブエンハンスメント

**デフォルトアプローチ**: シンプルに構築 → 段階的に拡張

## コア哲学

- **根本原因**: 「どう修正するか？」ではなく「なぜ？」
- **予防 > パッチ**: 最良の解決策は問題を予防
- **シンプル > 複雑**: エレガンス = 正しい問題を解決

## CSS-First原則

JavaScriptを書く前に確認: 「CSSで解決できる？」

| 優先度 | 解決策 | 例                                 |
| ------ | ------ | ---------------------------------- |
| 1      | HTML   | セマンティック要素、ネイティブ入力 |
| 2      | CSS    | Grid, Flexbox, :has(), transitions |
| 3      | JS     | 本当に必要な場合のみ               |

## アウトカムファースト開発

**コア原則**: アーキテクチャではなく、アウトカムを出荷

| フェーズ       | アクション             | トリガー             |
| -------------- | ---------------------- | -------------------- |
| 動くようにする | 直接の問題を解決       | 常に最初             |
| 堅牢にする     | エラーハンドリング追加 | エラー発生時         |
| 速くする       | 最適化                 | 遅さが測定されたとき |
| 柔軟にする     | オプション追加         | ユーザー要求時       |

## 重要なポイント

The Best Code is No Code At All.
