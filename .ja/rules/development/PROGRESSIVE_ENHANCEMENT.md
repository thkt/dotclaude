# プログレッシブエンハンスメント - アウトカムファースト開発

**デフォルトアプローチ**: シンプルに構築 → 段階的に拡張

## コア哲学

- **根本原因**: 「どう修正するか？」ではなく「なぜ？」
- **予防 > パッチ**: 最良の解決策は問題を予防
- **シンプル > 複雑**: エレガンス = 正しい問題を解決

## 優先度

1. **HTML** - 構造
2. **CSS** - ビジュアル/レイアウト
3. **JavaScript** - 必要な場合のみ

## CSS優先ソリューション

- **レイアウト**: Grid/Flexbox
- **配置**: Transform（リフローなし）
- **表示/非表示**: visibility/opacity
- **レスポンシブ**: Media/Containerクエリ
- **状態**: :target, :checked, :has()

## 例

```css
/* ✅ CSS Gridオーバーレイ */
.container { display: grid; }
.panel { grid-column: 1 / -1; grid-row: 1 / -1; }
```

## 適用対象

レイアウト、配置、表示/非表示、レスポンシブ、アニメーション、視覚状態

## 覚えておく

「最高のコードはコードなし」 - CSSで解決できるなら、JSをスキップ

## アウトカムファースト開発

**コア原則**: アーキテクチャではなく、アウトカムを出荷

### 実装フェーズ

1. **動くようにする** - 直接の問題を解決
2. **堅牢にする** - エラーが発生したときにエラーハンドリングを追加
3. **速くする** - 遅さが測定されたときに最適化
4. **柔軟にする** - ユーザーが要求したときにオプションを追加

### 判断フレームワーク

コードを追加する前に尋ねる:

- これは今存在する実際の問題を解決している？
- 実際に本番環境で失敗した？
- ユーザーからこれについて苦情があった？
- 問題の測定された証拠がある？

「いいえ」なら → まだ追加しない

### プログレッシブエンハンスメントのマインドセット

- ハッピーパスから始める
- 現実に応じてのみ複雑さを追加
- すべてのコード行は負債
- YAGNIがデフォルトの立場

## 重要なポイント

「最高のコードは、まだ存在する必要のないコード」
