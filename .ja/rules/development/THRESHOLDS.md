# Code Thresholds

認知限界 (作業記憶、1 画面集中) と確立されたメトリクス (McCabe complexity) に基づく目標。ハードリミットは guardrails フックを介して oxlint で強制する。

| 目標         | 推奨値 | 根拠                                |
| ------------ | ------ | ----------------------------------- |
| 関数行数     | ≤30   | 1 画面の可読性                      |
| ファイル行数 | ≤400  | モジュールレベルの認知上限          |
| ネスト深度   | ≤3    | 作業記憶内での分岐追跡              |
| 関数引数     | ≤3    | 引数順の暗記限界                    |
| 循環的複雑度 | ≤10   | McCabe 1976: パス爆発なくテスト可能 |

## Coverage

| レベル | 目標  | 焦点                              |
| ------ | ----- | --------------------------------- |
| C0     | ≥90% | 全行実行 (未テストコードを検出)   |
| C1     | ≥80% | 全分岐通過 (未テストの判断を検出) |

例外: 自動生成コード、データ定義、テストファイル、移行中のレガシー。
