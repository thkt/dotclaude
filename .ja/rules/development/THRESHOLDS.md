# Code Thresholds

認知限界（ワーキングメモリ、1画面集中）と確立された指標（McCabe 複雑度）に基づく推奨値。ハードリミットは oxlint（guardrails hook 経由）で強制。

| 対象                     | 推奨 | 根拠                                  |
| ------------------------ | ---- | ------------------------------------- |
| 関数行数                 | ≤30  | 1画面で読める                         |
| ファイル行数             | ≤400 | モジュール単位の認知負荷上限          |
| ネスト深度               | ≤3   | ワーキングメモリでの分岐追跡限界      |
| 関数引数                 | ≤3   | 引数順序の記憶限界                    |
| サイクロマティック複雑度 | ≤10  | McCabe 1976: パス爆発なしにテスト可能 |

## カバレッジ

| レベル | 目標 | 対象                           |
| ------ | ---- | ------------------------------ |
| C0     | ≥90% | 全行実行（未テストコード検出） |
| C1     | ≥80% | 全分岐実行（未テスト判定検出） |

例外: 自動生成コード、データ定義ファイル、テストファイル、移行中のレガシー
