# コード構造

## コードベース一貫性原則

優先順：1 > 2 > 3（衝突時は上位が勝つ）

| 優先 | 原則               | 内容                                                                             |
| ---- | ------------------ | -------------------------------------------------------------------------------- |
| 1    | 既存パターンに従う | 構造・命名・エラー処理・状態管理のパターンを揃える。変更はユーザー指示のときだけ |
| 2    | ないならフラット   | AIが構造を発明しない。分割判断はユーザーに委ねる。指示があれば戦略の手札から選択 |
| 3    | コロケーション     | 分割するときは一緒に変わるものを近くに                                           |

## デフォルト値

フレームワーク規約がある場合はそちらが優先。

| 項目             | デフォルト     |
| ---------------- | -------------- |
| ディレクトリ深さ | 3階層まで      |
| 横断モジュール   | `shared/`      |
| テスト配置       | ソースと同階層 |

## 構造戦略の手札

既存構造がないとき、What/Howで選択。階層ごとに独立適用する（例：プロジェクト
全体はパイプライン、ステージ内はフィーチャー）。

| 戦略         | ハマるとき                         |
| ------------ | ---------------------------------- |
| フラット     | 小規模、単機能                     |
| パイプライン | データが一方向に流れる（CLI, ETL） |
| レイヤー     | リクエスト-レスポンス型（API）     |
| フィーチャー | 独立した機能群（UI, SaaS）         |
| ドメイン     | 境界づけられたコンテキストが複数   |

```
小規模                  → フラット
概念1つ + 線形          → パイプライン
概念1つ + ツリー        → レイヤー
概念複数 + ツリー       → フィーチャー
概念複数 + 複雑         → ドメイン
```

## テスタブルレイヤリング

Container/PresentationalのContainer側をさらに分離する。

| レイヤー     | 責務                     | テスト方法        |
| ------------ | ------------------------ | ----------------- |
| Presentation | props → JSX              | props を渡す      |
| Container    | hook 呼び出し + 組み立て | integration / E2E |
| Logic        | 純粋な変換・判定・導出   | 入力 → 出力       |

### 切り出し基準

入力と出力だけでテストケースが書けるならLogicに切り出す。

| 条件                     | Action           |
| ------------------------ | ---------------- |
| 分岐・条件が 2+ ある変換 | Logic に切り出す |
| 同じ変換を 2+ 箇所で使用 | Logic に切り出す |
| 単純な props マッピング  | Container に残す |

### Logic ファイル命名

概念名で命名する。`utils.ts` は使わない。

```
# React
features/order/
├── OrderDetail.tsx         # Presentation
├── useOrderDetail.ts       # Container
├── pricing.ts              # Logic（概念名）
├── pricing.test.ts
├── orderStatus.ts          # Logic（概念名）
└── orderStatus.test.ts

# Rust
src/
├── tools/mod.rs            # Orchestration
├── chunker.rs              # Logic（概念名）
└── redact.rs               # Logic（概念名）
```

## ファイル内整理

関数定義順は呼び出し順に揃える。実装時に適用し、TIDYINGSでは並べ替えしない。
