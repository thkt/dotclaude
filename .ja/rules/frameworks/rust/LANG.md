# Rust

## モジュール構造

| ルール       | 詳細                                                                                                                  |
| ------------ | --------------------------------------------------------------------------------------------------------------------- |
| ファイル配置 | `sub.rs` + `sub/child.rs` スタイルを使う。`sub/mod.rs` は避ける                                                       |
| 可視性       | 必要最小限。`pub(super)` > `pub(crate)` > `pub` の順で優先                                                            |
| impl 分割    | 1つの struct の `impl` ブロックを複数ファイルに分割可。分割メソッドは `pub(super)` で宣言モジュールに可視性を限定する |

### バイナリCLIレイアウト

```
src/
├── main.rs           # サブコマンド振り分け
├── lib.rs            # crate root、接続
├── config.rs         # 設定型
├── <domain>.rs       # ドメインモジュール
├── <domain>/
│   └── tests.rs      # インラインが長くなったら分割
├── storage.rs        # 永続化層
├── storage/
│   ├── <concern>.rs
│   └── tests.rs
└── tools.rs          # 外部ツールラッパー
    └── <tool>.rs
tests/
└── cli_integration.rs
```

## 命名規約

### Casing (RFC 430 / C-CASE)

| 項目                                               | 規約                                                      |
| -------------------------------------------------- | --------------------------------------------------------- |
| Crate / Module / Function / Method / Local / Macro | `snake_case`                                              |
| Type / Trait / Enum variant                        | `UpperCamelCase`                                          |
| Constant / Static                                  | `SCREAMING_SNAKE_CASE`                                    |
| Type parameter                                     | 簡潔な `UpperCamelCase` — 曖昧でなければ 1 文字(`T`, `E`) |
| Lifetime                                           | 短い小文字(`'a`, `'src`)                                  |

`UpperCamelCase` の頭字語は 1 単語扱い: `Uuid` not `UUID`、`Stdin` not `StdIn`。クレート名は `-rs` / `-rust` サフィックスを付けない。

### 変換プレフィックス (C-CONV)

| プレフィックス | コスト | 意味                                   |
| -------------- | ------ | -------------------------------------- |
| `as_`          | 安価   | Borrowed → borrowed(ビュー)            |
| `to_`          | 高価   | Borrowed → borrowed / borrowed → owned |
| `into_`        | 可変   | Owned → owned(消費)                    |

### その他の規約

| ルール         | 詳細                                                                          |
| -------------- | ----------------------------------------------------------------------------- |
| Getter         | `get_` プレフィックス不要。`self.name()` / `self.name_mut()`                  |
| Iterator       | `iter` / `iter_mut` / `into_iter`                                             |
| エラー型       | `UpperCamelCase` の動詞-目的語-Error 順: `ParseIntError`(not `IntParseError`) |
| コンストラクタ | `new` または `with_<detail>`。変換コンストラクタは `from_<other>`             |

### 避けるべき

| AI がやりがち                                                       | 正しい                           |
| ------------------------------------------------------------------- | -------------------------------- |
| `get_name()`                                                        | `name()`                         |
| `UserID`, `HTTPClient`                                              | `UserId`, `HttpClient`           |
| `fn as_string(&self) -> String`(割当するのに `as_` は安価を暗示)    | `fn to_string(&self) -> String`  |
| `fn to_bytes(self) -> Vec<u8>`(消費するのに `to_` は借用保持を暗示) | `fn into_bytes(self) -> Vec<u8>` |

## テスト

### ファイル配置

| パターン                                       | 使い分け                                             |
| ---------------------------------------------- | ---------------------------------------------------- |
| `#[cfg(test)] mod tests { use super::*; ... }` | デフォルト — テストをコード近くに                    |
| `#[cfg(test)] mod tests;` → `tests.rs`         | テストコードがモジュールを埋没させるほど長くなったら |
| `tests/cli_integration.rs`                     | バイナリ CLI の振る舞い(実バイナリを起動)            |

### 命名

| ルール       | 詳細                                                           |
| ------------ | -------------------------------------------------------------- |
| T-NNN 識別子 | 各テストに `// T-NNN: function_name` を `#[test]` 直前に付ける |
| 関数名       | 自己説明的な snake_case                                        |

### ヘルパー

ヘルパーは primary handle と RAII guard(`TempDir` 等)の両方を返すように中央化する。役割で命名し、クレート名で命名しない。

| ヘルパーパターン                          | 目的                                                                       |
| ----------------------------------------- | -------------------------------------------------------------------------- |
| `test_db() -> (Db, TempDir)`              | 新鮮で隔離された DB を一時ディレクトリに                                   |
| `test_<crate>() -> (<MainType>, TempDir)` | クレートの主型をテストダブルで構築 — 実ネットワーク / ファイルシステムなし |
| `setup_test_files(files)`                 | 一時ディレクトリにフィクスチャファイル書き込み、ハンドル返却               |

### 隔離

| ルール           | 詳細                                                                                                                                         |
| ---------------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| 一時ディレクトリ | `tempfile::tempdir()` を使い、`TempDir` を `_dir` にバインドしてテスト終了まで生存させる                                                     |
| モック           | クレート提供のモック / 失敗ダブル(例: `MockEmbedder`, `FailingEmbedder`)を使う — 単体テストで実ネットワーク / モデルエンドポイントを呼ばない |

### アサーション

アサーション不一致だけでは問題が分からない場合、失敗メッセージを追加する。

```rust
assert!(stdout.starts_with(expected_prefix), "expected prefix {expected_prefix:?}, got: {stdout}");
```

### `test-support` フィーチャー

テストヘルパー(モック、フィクスチャ)を `[features] test-support = []` で公開し、下流クレートが `features = ["test-support"]` を `[dev-dependencies]` で指定できるようにする。`#[cfg(test)]` は外部テストから隠される。

## Use 宣言

曖昧さを避け、リファクタリングコストを下げるため、明示的なパスプレフィックスを使う。クレートごとにグルーピングスタイルを統一(`use std::{a, b}` vs 別行)。`rustfmt` がアルファベット順を強制する。

| プレフィックス   | スコープ           |
| ---------------- | ------------------ |
| `crate::`        | クレートルートから |
| `super::`        | 親モジュール       |
| `self::`         | 現在のモジュール   |
| `::crate_name::` | 外部クレート       |

## イディオム

行数を減らすため、イディオマティックな Rust を優先する。`unwrap()` はテストや不変条件が構築で証明されている箇所では許容。`(1.95+)` タグの機能は `Cargo.toml` で `rust-version = "1.95"` が必要。クレートが実際に使う最新機能に MSRV を揃える。

| 場面                                | 優先                                                     | より                                          |
| ----------------------------------- | -------------------------------------------------------- | --------------------------------------------- |
| エラー伝播                          | `?`                                                      | 本番コードでの `unwrap()`                     |
| `Option` フォールバック             | `unwrap_or(default)` / `unwrap_or_else(\|\| ...)`        | `match`                                       |
| `Result` フォールバック             | `unwrap_or(default)` / `unwrap_or_else(\|e\| ...)`       | `match`                                       |
| エラー型変換                        | `map_err(\|e\| ...)`                                     | `match Err(e) =>`                             |
| 破棄 + 変換                         | `.filter_map()`                                          | `.filter().map()`                             |
| Boolean 短絡                        | `.any()` / `.all()`                                      | 明示ループ                                    |
| 単一バリアントチェック              | `if let` / `let else`                                    | `match`                                       |
| Boolean パターンチェック            | `matches!(val, Pat)`                                     | bool を返す `match`                           |
| 単一引数クロージャ                  | point-free(`f`)                                          | `\|x\| f(x)`                                  |
| `Option<String>` → `Option<&str>`   | `opt.as_deref()`                                         | `opt.as_ref().map(\|s\| s.as_str())`          |
| HashMap insert-or-modify            | `map.entry(k).or_insert_with(...)`                       | `if !map.contains_key(k) { map.insert(...) }` |
| パターン + match 内の条件           | `match v { Some(x) if let Ok(y) = f(x) => ... }` (1.95+) | ネストした `match` / `if let` ブロック        |
| Push + 可変参照利用                 | `let r = v.push_mut(x);` (1.95+)                         | `v.push(x); v.last_mut().unwrap()`            |
| Integer to bool                     | `bool::try_from(n)?` (1.95+)                             | `n != 0`(意図不明瞭)                          |
| プラットフォーム / フィーチャー分岐 | `cfg_select! { unix => ..., _ => ... }` (1.95+)          | `cfg-if` クレート依存                         |

### Derive 選択

型ごとに最小セット。過剰 derive は表現をリークしコンパイル時間を膨張させる。

| Derive                      | いつ                                                               |
| --------------------------- | ------------------------------------------------------------------ |
| `Debug`                     | 常時 — `?` エラー出力、テスト失敗、`dbg!` に必要                   |
| `Clone`                     | ユーザーが複製を必要とする                                         |
| `Copy`                      | 意味的コストなしの POD ≤ 16 bytes。`Clone` 必須                    |
| `Default`                   | 自明なゼロ値が存在する                                             |
| `PartialEq` / `Eq`          | 等価性が well-defined                                              |
| `Hash`                      | map キーとして使う(`Eq` と整合必須)                                |
| `Serialize` / `Deserialize` | 型がシリアライゼーション境界を越える。内部専用の型には絶対付けない |

### 手動 impl(derive 不可)

| トレイト                 | いつ                                                                           |
| ------------------------ | ------------------------------------------------------------------------------ |
| `Display`                | ユーザー向け型。`thiserror::Error` の `#[error("...")]` と連携                 |
| `From<T>` / `TryFrom<T>` | 他の型からの自然な変換 — 場当たり的な `new` / `with_*` より優先(C-CONV-TRAITS) |

## エラーハンドリング

### ライブラリ vs アプリケーション

| コンテキスト                | クレート                                   | パターン                                      |
| --------------------------- | ------------------------------------------ | --------------------------------------------- |
| ライブラリ                  | `thiserror`                                | 呼び出し側はバリアントをパターンマッチ        |
| アプリケーション / CLI 入口 | `anyhow`                                   | 呼び出し側は伝播のみ。`.context()` でトレース |
| 混在                        | `lib.rs` で thiserror、`main.rs` で anyhow | lib は型付きエラー、bin は context 付きラップ |

### thiserror 規約

| ルール               | 詳細                                                                                                                                          |
| -------------------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| 属性                 | enum に `#[derive(Error, Debug)]`                                                                                                             |
| メッセージ           | `#[error("... {field} ...")]`                                                                                                                 |
| ラップ + 自動変換    | 内部エラーに `#[from]` — `?` 有効化。`#[source]` を暗黙に含むので両方付けない                                                                 |
| チェーン保持         | ラップしない場合は `#[source]`(追加コンテキストフィールド)                                                                                    |
| バリアント粒度       | 原因ごとに 1 バリアント。`String` にフラット化しない                                                                                          |
| 前方互換性           | `pub` エラー enum に `#[non_exhaustive]` — バリアント追加が非破壊的                                                                           |
| クレートレベル alias | クレートルートで `pub type Result<T> = std::result::Result<T, CrateError>;`。エラー型が違う時は `std::result::Result<T, OtherError>` で上書き |

### anyhow 規約

| ルール             | 詳細                                 |
| ------------------ | ------------------------------------ |
| 返り値型           | `anyhow::Result<T>`                  |
| コンテキスト層追加 | `.context("failed to load config")?` |
| 早期終了           | `bail!("reason")`                    |
| アサーション       | `ensure!(cond, "msg")`               |

### 避けるべき

| AI がやりがち                                          | 正しい                                              |
| ------------------------------------------------------ | --------------------------------------------------- |
| ライブラリの公開 API で `-> Result<T, Box<dyn Error>>` | `thiserror` enum を定義して呼び出し側がマッチ可能に |
| 全モジュールをカバーするグローバル `AppError` enum     | モジュールごとのエラー型、`#[from]` でラップ        |
| `.map_err(\|e\| e.to_string())?`                       | `#[from]` または `.context()` — チェーン保持        |
| ライブラリ内の `anyhow!`                               | `anyhow` はアプリケーション層に限定                 |
| 同じフィールドに `#[from]` と `#[source]` を両方       | `#[from]` 単独 — すでに `#[source]` を暗黙に含む    |

## Unsafe (2024 Edition)

すべての `unsafe {}` ブロックに `// SAFETY: ...` コメントを付け、不変条件が成立する理由を説明する。公開 `unsafe fn` は呼び出し側の契約を rustdoc の `# Safety` セクションに明記する。

| ルール                                              | 詳細                                                                                                           |
| --------------------------------------------------- | -------------------------------------------------------------------------------------------------------------- |
| `unsafe fn` の本体                                  | unsafe 操作を `unsafe {}` で囲む — `unsafe_op_in_unsafe_fn` は warn-by-default                                 |
| `extern` ブロック                                   | `unsafe extern "C" { ... }` — ブロックに `unsafe` キーワードが必要                                             |
| `#[no_mangle]`, `#[export_name]`, `#[link_section]` | `unsafe #[no_mangle]` 等が必要                                                                                 |
| `static mut`                                        | `static mut` への参照は deny-by-default (2024 `static_mut_refs` lint)。`Mutex`, `RwLock`, atomics に置き換える |
| `std::env::set_var` / `remove_var`                  | `unsafe` になった — `unsafe {}` で囲む                                                                         |

## 所有権 & スマートポインタ

### ポインタ選択

| ポインタ     | 所有             | スレッド安全 | いつ                                 |
| ------------ | ---------------- | ------------ | ------------------------------------ |
| `Box<T>`     | 単一             | N/A          | Heap alloc、再帰型、`Box<dyn Trait>` |
| `Rc<T>`      | 共有(count)      | No           | 単一スレッドの Graph / DAG           |
| `Arc<T>`     | 共有(count)      | Yes          | スレッド間共有                       |
| `Cow<'a, B>` | borrowed / owned | N/A          | 主に不変で時々変更                   |

### 内部可変性

| パターン                     | スコープ                                             |
| ---------------------------- | ---------------------------------------------------- |
| `Rc<RefCell<T>>`             | 単一スレッド共有可変                                 |
| `Arc<Mutex<T>>`              | 複数スレッド、クリティカルセクションに `.await` なし |
| `Arc<RwLock<T>>`             | 複数スレッド、読み取り多数                           |
| `Arc<tokio::sync::Mutex<T>>` | ロックを `.await` 越しに保持                         |

### パラメータ型

コンパイル通る最も狭い型をデフォルト。

| 欲しい             | パラメータ                                             |
| ------------------ | ------------------------------------------------------ |
| 読み取りのみ       | `&str`, `&[T]`, `&T`                                   |
| 変更               | `&mut T`                                               |
| 消費 / 保存        | `String`, `Vec<T>`, `T`                                |
| 柔軟な読み取り専用 | `impl AsRef<str>` — `&str`, `&String`, `String` を受理 |
| 柔軟な所有権取得   | `impl Into<String>` — `&str`(変換), `String` を受理    |

### 避けるべき

| AI がやりがち                               | 正しい                         |
| ------------------------------------------- | ------------------------------ |
| `fn f(s: &String)`                          | `fn f(s: &str)`                |
| `fn f(v: &Vec<T>)`                          | `fn f(v: &[T])`                |
| `Arc<Mutex<T>>` で `.await` 越しにロック    | `Arc<tokio::sync::Mutex<T>>`   |
| borrow checker を満たすため至る所で `Clone` | ライフタイム調整か所有権再構築 |

## Async / 並行

`tokio::spawn` は `'static` 必須。ローカル変数を spawn 境界越しに借用できない。スコープ境界付き並列には `async-scoped::TokioScope::scope_and_block` を使う(`std::thread::scope` と類似)。

| シナリオ                               | アプローチ                                                                       |
| -------------------------------------- | -------------------------------------------------------------------------------- |
| 並列非同期タスク(I/O-bound)            | `move` クロージャで `tokio::spawn`                                               |
| 並列タスク + 共有ローカル変数          | `Arc<T>` で包み spawn 前にクローン、または `async-scoped` でスコープ境界付き並列 |
| 非同期コンテキスト内のブロッキング I/O | `tokio::task::spawn_blocking` — blocking スレッドプールに移動                    |
| CPU-bound 並列                         | `rayon`(データ並列)または `std::thread::spawn`(場当たり)                         |

### パターン

| パターン                   | いつ                                                                |
| -------------------------- | ------------------------------------------------------------------- |
| `OnceLock<T>` (std, 1.70+) | `get_or_init()` で遅延一度初期化 — 関数ローカルの遅延構築用         |
| `LazyLock<T>` (std, 1.80+) | インラインクロージャ init の `static` — `lazy_static!` を置き換える |
| `const fn`                 | コンパイル時計算(`Duration`、ビットマスク等)                        |

### 避けるべき

| AI がやりがち                                | 正しい                                                                   |
| -------------------------------------------- | ------------------------------------------------------------------------ |
| `lazy_static!`                               | `LazyLock`(グローバル `static`)または `OnceLock`(関数ローカル)、両方 std |
| ブロッキング呼び出しを `tokio::spawn` で囲む | `tokio::task::spawn_blocking`                                            |
| `std::sync::Mutex` を `.await` 越しに保持    | `tokio::sync::Mutex`                                                     |

## 多態性

| 選択                    | いつ                                                                                               |
| ----------------------- | -------------------------------------------------------------------------------------------------- |
| `enum`                  | バリアントがコンパイル時に閉じた集合。ホットループでは静的ディスパッチが速い — 計測してから最適化  |
| `dyn Trait`             | 外部呼び出し側が新型を追加する必要、または公開 crate API に型が出る                                |
| `impl Trait` (generics) | vtable なしの静的ディスパッチ — オーバーヘッド回避だが monomorphization 膨張、選ぶ前にプロファイル |

### 避けるべき

| AI がやりがち                                  | 正しい                                               |
| ---------------------------------------------- | ---------------------------------------------------- |
| 公開返り値位置での `-> impl Trait`             | 名前付き型 — `impl Trait` 返却は将来広げる破壊的変更 |
| 閉じたバリアント集合に `Box<dyn Trait>`        | `enum` — ディスパッチ安価、網羅的 `match`            |
| 呼び出し側が拡張可能にすべき型を `enum` で潰す | 下流クレートが新バリアント追加必要なら `dyn Trait`   |

## API 設計パターン

### Newtype

プリミティブを 1 フィールド struct で包み、表現が同じでも意味的に異なる値を区別する。

```rust
pub struct UserId(pub u64);
pub struct ProductId(pub u64);
```

実行時コストゼロ。型間混同をコンパイル時に防ぐ。

### Type state

型パラメータに状態をエンコードし、不正な遷移をコンパイルエラーにする。

| ルール       | 詳細                                                              |
| ------------ | ----------------------------------------------------------------- |
| 状態マーカー | ゼロサイズ struct(`struct Connected;`)または `PhantomData<State>` |
| 遷移         | `self` を消費し新しい型を返す                                     |
| 操作         | 有効な状態にのみメソッドを実装                                    |

### Builder

struct が ≥3 の optional フィールドを持つ、または構築前に検証が必要な場合に使う。

| ルール | 詳細                                                                |
| ------ | ------------------------------------------------------------------- |
| Entry  | `Foo::builder()` が `FooBuilder` を返す                             |
| Setter | `fn with_x(&mut self, x: X) -> &mut Self`(可変参照、エルゴノミクス) |
| Build  | `fn build(self) -> Foo` — 検証が失敗しうる場合は `Result<Foo, _>`   |

1–2 フィールドなら struct リテラルか通常の `new` を優先。

### Sealed trait

将来のメソッド追加が非破壊的である必要がある公開トレイトに、外部クレートからの実装を禁じる。

```rust
mod private { pub trait Sealed {} }
pub trait MyTrait: private::Sealed { /* ... */ }
```

### ファクトリパラメータによる依存性注入

```rust
pub fn from_env() -> Result<Self> { Self::from_env_with(std::env::var) }
pub fn from_env_with<F>(get: F) -> Result<Self>
where F: Fn(&str) -> Result<String, VarError> { /* ... */ }
```

本番用の `from_env()`、テスト用のモック lookup 付き `from_env_with()`。

### 避けるべき

| AI がやりがち                                  | 正しい                                                   |
| ---------------------------------------------- | -------------------------------------------------------- |
| `fn f(force: bool, dry_run: bool)`             | enum または struct(`Mode::Force` / `{ force, dry_run }`) |
| 状態を実行時フラグで持つ(`is_connected: bool`) | Type state パターン                                      |
| すべての識別子に `String`                      | Newtype(`UserId`, `OrderId`)                             |
| 公開トレイトへの外部トレイト実装を自由に受容   | 進化を気にするなら Sealed パターン                       |

## クレート分割

以下のいずれかに該当する場合、モジュールを別クレートに分割する。

| ケース         | 詳細                                                                                      |
| -------------- | ----------------------------------------------------------------------------------------- |
| コンパイル時間 | 大きくて変更頻度の低いコードを隔離し、差分ビルドでスキップ                                |
| 共有ロジック   | 複数ターゲット向けにコンパイル(例: ネイティブバイナリ + WASM)                             |
| Proc macro     | 手続き的マクロは別クレート必須 — `rustc` がロードするネイティブプラグインとしてコンパイル |
| Unsafe 隔離    | FFI / `unsafe extern` を子クレートに隔離、外側クレートで `#![forbid(unsafe_code)]`        |

## ドキュメント (rustdoc)

### コメントスタイル

| 構文  | スコープ                                            |
| ----- | --------------------------------------------------- |
| `///` | アイテム(関数、struct、enum、trait、メソッド)       |
| `//!` | モジュールまたは crate(`lib.rs` / `module.rs` 先頭) |

### セクション

1 エントリしかない場合でも複数形。

| ヘッダー     | いつ                                                        |
| ------------ | ----------------------------------------------------------- |
| `# Examples` | すべての `pub` アイテム — コードブロックは自動で doctest 化 |
| `# Errors`   | `Result` を返す関数                                         |
| `# Panics`   | panic しうる関数 — 前提条件を明記                           |
| `# Safety`   | `unsafe fn` / `unsafe trait` — 呼び出し側の契約を明記       |

### 規約

| ルール               | 詳細                                                                  |
| -------------------- | --------------------------------------------------------------------- |
| Summary              | 最初の行: 1 文、ピリオドで終わる                                      |
| 例のエラー処理       | `?` を使い、`.unwrap()` は使わない                                    |
| 隠し setup           | doctest の setup 行に `#` プレフィックス — コンパイルされるが隠される |
| 型参照               | ジェネリクス含む完全名: `Option<T>`                                   |
| イントラドックリンク | ``[`Foo`]`` で自動解決                                                |

### 強制

`#![warn(missing_docs)]` はライブラリ crate ルートのみ(バイナリエントリはアイテム単位の docs 不要)。

### 避けるべき

| AI がやりがち                 | 正しい                                          |
| ----------------------------- | ----------------------------------------------- |
| `# Examples` での `.unwrap()` | fallible な `fn main() -> Result<(), E>` と `?` |
| `# Example`(単数形)           | `# Examples`(複数形、常時)                      |
| バイナリで `missing_docs`     | ライブラリ crate のみ                           |

## ロギング

`tracing`(構造化、async-aware)を `log` より優先。

| レベル   | いつ                                        |
| -------- | ------------------------------------------- |
| `error!` | 操作失敗、ユーザー可視の影響                |
| `warn!`  | 回復可能な異常(リトライ、フォールバック)    |
| `info!`  | ライフサイクルイベント(起動、設定ロード)    |
| `debug!` | 開発時のリクエスト / アイテム単位のトレース |
| `trace!` | 細粒度フロー(稀)                            |

| ルール                        | 詳細                                                                               |
| ----------------------------- | ---------------------------------------------------------------------------------- |
| 構造化フィールド              | `info!(count = items.len(), "indexed batch")` — フィールドを先、メッセージを後     |
| スパン                        | リクエスト / タスクエントリーポイントに `#[tracing::instrument]`                   |
| サブスクライバ                | `main` で一度だけ `tracing_subscriber` を `EnvFilter::from_default_env()` で初期化 |
| `println!` / `eprintln!` 禁止 | CLI stdout 出力がプログラムの実際の結果である場合を除く                            |

### 避けるべき

| AI がやりがち                                       | 正しい                                       |
| --------------------------------------------------- | -------------------------------------------- |
| ライブラリコードに散在する `println!("debug: {x}")` | `tracing::debug!(x = ?x)`                    |
| `info!("loaded {}", count)`                         | `info!(count, "loaded")`(構造化)             |
| ライブラリコードで `tracing_subscriber` 初期化      | アプリケーションエントリポイントのみで初期化 |
