# React / TypeScript フロントエンド

## 状態管理

| ルール       | 詳細                                                              |
| ------------ | ----------------------------------------------------------------- |
| ローカル優先 | 状態管理ライブラリの前に useState/useReducer                      |
| 読み書き分離 | 読取フックと書込フックを分けて不要な再レンダーを防止              |
| 派生状態     | レンダー内か useMemo で計算。ストアに追加エントリを作らない       |
| 同期優先     | フェッチがストアの責務でない限り、非同期 atoms/selectors を避ける |

## メモ化

計測ベースのみ。先回りのReact.memo, useCallback, useMemoは禁止。

## コンポーネント設計

| ルール            | 詳細                                       |
| ----------------- | ------------------------------------------ |
| 共有UI            | `components/ui/` に配置                    |
| Propsバケツリレー | 最大2階層、超えたら Context か composition |

## スタイリング

| ルール                      | 詳細                                                                   |
| --------------------------- | ---------------------------------------------------------------------- |
| ユーティリティ/トークン優先 | カスタムCSSの前にPJのCSSフレームワークのユーティリティを使う           |
| 条件付きクラス              | クラスマージユーティリティ（clsx, cva, cx 等）を使用。文字列結合は不可 |
| レスポンシブ                | モバイルファーストのブレークポイント                                   |

## エラーハンドリング

| レイヤー       | パターン                                          |
| -------------- | ------------------------------------------------- |
| コンポーネント | Error Boundary でルートレベルをラップ             |
| 非同期         | Result型（neverthrow, option-t 等）+ 早期リターン |
| ユーザー通知   | エラー時は Toast/ダイアログ。握りつぶし禁止       |

## アクセシビリティ

| ルール           | 基準                                       |
| ---------------- | ------------------------------------------ |
| HTML             | セマンティック要素を優先                   |
| ARIA             | セマンティックHTMLで不十分な場合のみ       |
| キーボード       | 全インタラクティブ要素がフォーカス・操作可 |
| コントラスト     | WCAG 2.2 AA 以上                           |
| タッチターゲット | 44px 以上                                  |

## テスト

| ツール                | スコープ                                              |
| --------------------- | ----------------------------------------------------- |
| Vitest                | ユニット + インテグレーション                         |
| React Testing Library | ユーザー視点のアサーション（実装詳細に依存しない）    |
| Storybook             | UIコンポーネントのビジュアル + インタラクションテスト |
