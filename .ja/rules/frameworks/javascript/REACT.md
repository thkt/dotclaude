# React

## 状態管理

| ルール       | 詳細                                                                 |
| ------------ | -------------------------------------------------------------------- |
| ローカル優先 | 状態管理ライブラリより前に useState/useReducer                       |
| 読み書き分離 | 不要な再レンダリングを防ぐため読み込み hook と書き込み hook を分ける |
| 派生状態     | render か useMemo で計算する。store エントリを増やさない             |
| 同期優先     | フェッチが store の責務でない限り async atom/selector を避ける       |

### スコープ選択

| スコープ   | ツール        | 例                   |
| ---------- | ------------- | -------------------- |
| ローカル   | useState      | フォーム入力、トグル |
| 共有       | Context       | テーマ、認証状態     |
| グローバル | Zustand/Redux | アプリ全域キャッシュ |

## メモ化

計測に基づく場合のみ。先回りの React.memo, useCallback, useMemo は使わない。

## コンポーネント設計

| ルール         | 詳細                                           |
| -------------- | ---------------------------------------------- |
| 共通 UI        | `components/ui/` に置く                        |
| Props drilling | 最大 2 階層。それ以上は Context か composition |

## コンポーネント パターン

### Container/Presentational

| 役割           | 責務                   | スタイル                                 |
| -------------- | ---------------------- | ---------------------------------------- |
| Container      | データ、状態、ロジック | レイアウト (grid, flex, spacing, sizing) |
| Presentational | props のみ、UI         | 装飾 (色、ボーダー、タイポグラフィ)      |

データ取得と表示が交わる場合に適用する。1 回限りのコンポーネントには適用しない (YAGNI)。

### アンチパターン

| アンチパターン                              | 修正                             |
| ------------------------------------------- | -------------------------------- |
| Presentational が `useState`/`fetch` を持つ | 状態を Container に持ち上げる    |
| Container が装飾スタイルを持つ              | スタイルを Presentational に移す |

## スタイリング

| ルール             | 詳細                                                                      |
| ------------------ | ------------------------------------------------------------------------- |
| ユーティリティ優先 | カスタム CSS の前にプロジェクトの CSS フレームワーク ユーティリティを使う |
| 条件付きクラス     | クラスマージ ユーティリティ (clsx, cva, cx 等) を使う。文字列連結は避ける |
| Responsive         | モバイル ファースト ブレークポイント                                      |

## エラーハンドリング

| レイヤー     | パターン                                                        |
| ------------ | --------------------------------------------------------------- |
| Component    | route レベルを Error Boundary でラップ                          |
| Async        | 早期 return 付きの Result 型 (neverthrow, option-t, または同等) |
| ユーザー通知 | エラー時に toast/dialog。サイレント握りつぶしはしない           |

## アクセシビリティ

| ルール       | 基準                                      |
| ------------ | ----------------------------------------- |
| HTML         | セマンティック要素を優先                  |
| ARIA         | セマンティック HTML が不足する場合のみ    |
| キーボード   | 全インタラクティブ要素が focus + 操作可能 |
| コントラスト | WCAG 2.2 AA 最低                          |
| タッチ標的   | 44px 最低                                 |

## テスト

| ツール                | スコープ                                        |
| --------------------- | ----------------------------------------------- |
| Vitest                | ユニット + 統合                                 |
| React Testing Library | ユーザー視点のアサーション (実装詳細を扱わない) |
| Storybook             | UI コンポーネントの視覚 + 操作テスト            |
