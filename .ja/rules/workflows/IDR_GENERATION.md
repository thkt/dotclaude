# IDR (Implementation Decision Record) 生成

開発ライフサイクル全体で実装決定を追跡。

## コマンドアクション

| コマンド    | IDRアクション        |
| ----------- | -------------------- |
| `/code`     | 決定事項とともに作成 |
| `/audit`    | レビュー結果を追記   |
| `/polish`   | 簡素化内容を追記     |
| `/validate` | SOW ACとの照合       |

**場所**: `~/.claude/workspace/planning/[feature]/idr.md`

## IDR検出

| シナリオ | 検出方法                                       | パス                                       |
| -------- | ---------------------------------------------- | ------------------------------------------ |
| SOWあり  | `~/.claude/workspace/planning/**/sow.md`を検索 | `[SOWディレクトリ]/idr.md`                 |
| SOWなし  | コンテキストからフィーチャー名を推測           | `~/.claude/workspace/idr/[feature]/idr.md` |

## IDR生成 (/code)

| セクション     | 内容                           |
| -------------- | ------------------------------ |
| 変更ファイル   | `git diff --name-status HEAD`  |
| 主要な決定     | 実装の根拠                     |
| 備考           | トレードオフ、却下した代替案   |
| レビュー注意点 | コードレビューでの確認ポイント |

テンプレート: `~/.claude/templates/idr/template.md`

## IDR更新セクション

| コマンド  | セクション内容                               |
| --------- | -------------------------------------------- |
| /audit    | サマリー（重大度別件数）、課題テーブル、備考 |
| /polish   | 削除テーブル、簡素化の説明                   |
| /validate | AC検証テーブル、ギャップ、サインオフ         |

## SOW連携

| 方向      | リンク                                   |
| --------- | ---------------------------------------- |
| IDR → SOW | メタデータに `SOW: ./sow.md`             |
| SOW → IDR | Implementation Recordsに `IDR: ./idr.md` |

## エラーハンドリング

| シナリオ          | アクション                              |
| ----------------- | --------------------------------------- |
| SOWが見つからない | workspace/idr/にスタンドアロンIDRを作成 |
| IDRが見つからない | 新規作成(/code)またはスキップ(その他)   |
| SOW更新失敗       | 警告ログ、SOWリンクなしで続行           |
| Git利用不可       | 変更ファイルをスキップ、手動入力を使用  |
