# ワークフローガイド

## 標準ワークフロー

| パターン | ワークフロー                                                                              | 使用場面                         |
| -------- | ----------------------------------------------------------------------------------------- | -------------------------------- |
| 簡易修正 | `/fix`                                                                                    | 小さなバグ、安定したコードベース |
| 調査     | `/research` → `/fix`                                                                      | 原因不明                         |
| 機能開発 | `/feature`（または: `/research` → `/think` → `/code` → `/test` → `/audit` → `/validate`） | 新機能、要件が不安定             |
| シンプル | `/code` → `/test`                                                                         | 明確な実装                       |

## コマンド選択

| 理解度 | 複雑さ | 結果              |
| ------ | ------ | ----------------- |
| ≥95%   | 簡単   | `/fix` or `/code` |
| 70-94% | 任意   | `/think`          |
| <70%   | 任意   | `/research`       |
| 任意   | 緊急   | `/fix`（緊急）    |
| 任意   | 不明確 | `/research`       |

## Team-First 原則

デフォルト: Team（TeamCreate + TaskList で進捗可視化）

| コマンド       | モード | 備考                     |
| -------------- | ------ | ------------------------ |
| `/feature`     | Team   | 既存チーム構成           |
| `/audit`       | Auto   | スコープで判断           |
| `/think`       | Team   | 事前判断が困難           |
| `/code`        | Auto   | スコープで判断           |
| `/fix`         | Auto   | 下記 Solo 条件で判断     |
| `/research`    | Solo   |                          |
| ユーティリティ | Solo   | /commit, /branch, /pr 等 |

Auto: Solo 条件すべてに該当 → Solo、それ以外 → Team

| Solo 条件                | 例                        |
| ------------------------ | ------------------------- |
| 対象ファイル 1-2 個      | typo 修正、単一関数の変更 |
| 単一フェーズで完結       | 調査不要、テスト生成不要  |
| エージェント間の協調不要 | 他エージェントの出力不要  |
