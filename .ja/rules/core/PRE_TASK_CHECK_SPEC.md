## Done Definition テンプレート

### 機能追加

```markdown
- [ ] 指定された機能が動作する
- [ ] リグレッションなし | テスト追加
- [ ] 現状で十分な場合は変更不要
```

### 修正

```markdown
- [ ] 問題が再現しなくなる
- [ ] 根本原因が解決されている
- [ ] 現状で十分な場合は変更不要
```

### リファクタリング

```markdown
- [ ] 動作が変わっていない（テスト通過）
- [ ] 品質が向上している（測定可能）
- [ ] 現状で十分な場合は変更不要
```

### 調査（バグ/問題）

```markdown
- [ ] 再現手順を確認した
- [ ] 根本原因を特定した（症状だけでなく）
- [ ] 正常ケースを理解した（差分検出用）
- [ ] バグでなければ変更不要
```

## 「変更不要」検証ゲート

**「変更不要」と報告する前に:**

1. **証拠**: 読んだファイル:行番号を明示
2. **説明**: なぜ現状が目標を満たすか
3. **確認**: AskUserQuestion でユーザーに確認

```markdown
📋 変更不要の分析:

- 読んだ: src/auth/login.ts:45-67
- 現状: エラーケースXは既に処理されている
- 結論: 既存実装が要件を満たす
  → クローズ前にユーザーに確認
```
