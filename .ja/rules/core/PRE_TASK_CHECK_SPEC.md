理解度チェックの表示フォーマットとテンプレート。

```markdown

🧠 理解度: [██████░░░░] XX%

✅ 明確な要素:

- [✓] [検証済み項目]
- [→] [推論項目]

❓ 不明確な要素:

- [?] [確認が必要]

🎯 完了定義:

- [ ] [完了基準]

💡 推奨: /command - [理由]

⚡ 実現可能性: 🟢/🟡/🔴
```

進める前に `AskUserQuestion` を使用して確認。

## Dry-run / 実行計画

実行せずに変更の影響をプレビュー。

**ガイドライン**:

- 簡潔に: 最大3-5項目
- 影響に焦点: 何が変わるか、何が壊れる可能性があるか
- リスク評価: 潜在的な問題の正直な評価

### パターン A: 完全表示（複雑な変更）

Dry-run 条件を満たす場合（3ファイル以上、コア設定、認証/セキュリティ）:

```markdown
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🔍 影響シミュレーション（Dry-run）

• 変更ファイル: src/auth/login.ts, src/components/LoginForm.tsx
• 影響コンポーネント: AuthService, UserSession
• リスクレベル: 🟡 中（認証フロー変更）
• 注意点: 既存セッションは再ログインが必要になる可能性

─────────────────────────────────────────────────

📝 実行計画

1. src/auth/login.ts を編集 - OAuth2フローを追加
2. src/components/LoginForm.tsx を編集 - プロバイダーボタンを追加
3. テストを実行して検証

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### パターン B: 計画のみ（単純な変更）

Dry-run をスキップする場合（単純読取、ドキュメントのみ、単一ファイル10行未満）:

```markdown
📝 実行計画

1. src/utils/helper.ts を編集 - タイポ修正
```
