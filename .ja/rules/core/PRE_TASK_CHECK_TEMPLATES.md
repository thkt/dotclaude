# PRE_TASK_CHECK 詳細仕様

理解度チェックと実行計画のための技術仕様。基本ルールはPRE_TASK_CHECK_RULES.mdを参照。

---

## 分析方法

ユーザーリクエストを分析して以下を決定:

- 利用可能な情報に基づく理解度パーセンテージ
- 明確な要素と不明確な要素
- 以下の優先基準に基づくコマンド選択:
  1. **タスクスコープ**: 単一ファイル = /fix、複数ファイル = /code、調査 = /research
  2. **変更タイプ**: バグ修正 = /fix、新機能 = /think → /code
  3. **信頼度レベル**: <70% = まず/research、70-90% = /think、>90% = 直接/code
- 実行の実現可能性

**出力検証可能性要件**（AI動作原則 #4 による）:

理解度チェックの各要素について評価:

- **信頼度レベル**: 高（検証済み）、中（推論）、低（仮定）
- **証拠根拠**: 直接観察、論理的推論、または仮定
- **知識のギャップ**: 未知または不確実な点を明示的に認識

**信頼度レベルのマーカー**:

- ✓ = 高信頼度（ファイル/コンテキストから直接検証）
- → = 中信頼度（妥当な推論）
- ? = 低信頼度（確認が必要な仮定）

**プログレスバーの構築**:

- 100%を表すのに合計10ブロックを使用
- 各ブロック（█ または ░）は10%を表す
- パーセンテージに基づいて左から右に埋める
- █ = 埋まっている（信頼度レベル達成）
- ░ = 空（信頼度レベル未達成）

**例**:

```text
100% → ██████████ 100%  (10個埋まっている)
 90% → █████████░  90%  (9個埋まり、1個空)
 70% → ███████░░░  70%  (7個埋まり、3個空)
 50% → █████░░░░░  50%  (5個埋まり、5個空)
 30% → ███░░░░░░░  30%  (3個埋まり、7個空)
  0% → ░░░░░░░░░░   0%  (10個空)
```

**完了定義セクション**: 実装/修正タスク（/code、/fix コマンド）でのみ表示。このセクションはTodoWriteとは別 - 完了定義は目標（何を達成するか）を表し、TodoWriteは実行ステップ（どう達成するか）を追跡する。

## 影響シミュレーション（Dry-run）

**表示タイミング**: Y確認後、実行計画の前、ファイル操作または複雑な変更の場合。

予想される変更とその影響の簡潔なシミュレーションを提供。「Dry-run」アプローチに基づく - 実際の実行なしに変更をシミュレートして効果をプレビュー。

**表示フォーマット**:

```markdown
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🔍 影響シミュレーション（Dry-run）

予想される変更:
• 変更するファイル: [主要ファイル2-5個をリスト]
• 影響を受けるコンポーネント: [影響を受けるモジュール/コンポーネントをリスト]
• リスクレベル: 🟢 低 / 🟡 中 / 🔴 高
• 注意点: [重要な考慮事項]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**ガイドライン**:

- **長さ制限**:
  - 箇条書き: 3-5項目（最大5を強制）
  - 箇条書きごとの文字数: 最大100文字
  - 影響シミュレーション全体: 最大300文字
- **影響に焦点**: 何が変わるか、何が壊れる可能性があるか
- **リスク評価**: 潜在的な問題の正直な評価
- **実行可能**: 特別な注意が必要な領域を強調

**例**:

```markdown
🔍 影響シミュレーション（Dry-run）

予想される変更:
• 変更するファイル: src/auth/login.ts, src/components/LoginForm.tsx
• 影響を受けるコンポーネント: AuthService, UserSession, LoginPage
• リスクレベル: 🟡 中（認証フロー変更）
• 注意点: 既存のユーザーセッションは再ログインが必要な場合あり、新フローのテストを更新
```

**スキップ条件**（以下がすべてFALSEの場合のみ影響シミュレーションを適用）:

- **単純なファイル読み取り**: 変更なしのReadツール
- **ドキュメント更新**: \*.mdファイルのみが影響、コード変更なし
- **明らかに低リスクな変更**: 単一ファイル、10行未満の変更、依存関係への影響なし

以下のいずれかが該当する場合、常に影響シミュレーションを表示:

- コア設定の変更（CLAUDE.md、PRE_TASK_CHECK.md、settings.json）
- 3つ以上のファイルを同時に変更
- 認証、セキュリティ、またはデータ整合性ロジックに影響

## 統合ワークフロー

1. **理解度チェック**: 理解度を評価＆アプローチを提案
2. **ユーザー確認**: `AskUserQuestion` を使用 ← **停止ポイント**
3. **影響シミュレーション**（確認済みかつ複雑/リスクのある変更の場合）: dry-runプレビューを表示
4. **実行計画**（確認済みかつファイル/コマンド操作の場合）: 具体的なアクションを表示
5. **計画確認**: `AskUserQuestion` を使用 ← **停止ポイント**
6. **実行**: 承認されたアクションを実行
