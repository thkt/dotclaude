# PRE_TASK_CHECK テンプレート

理解度チェックと実行計画のための技術仕様。

## 分析方法

ユーザーリクエストを分析して決定:

- 理解度パーセンテージ
- 明確/不明確な要素
- コマンド選択: 単一ファイル → /fix、複数 → /code、調査 → /research
- 信頼度: <70% → /research、70-90% → /think、>90% → /code

**信頼度マーカー**: ✓（検証済み）、→（推論）、?（仮定）

## 表示フォーマット

```markdown

🧠 理解度: [██████░░░░] XX%

✅ 明確な要素:

- [✓] [検証済み項目]
- [→] [推論項目]

❓ 不明確な要素:

- [?] [確認が必要]

- [ ] [完了基準]

💡 推奨: /command - [理由]

⚡ 実現可能性: 🟢/🟡/🔴
```

進める前に `AskUserQuestion` を使用して確認。

## 影響シミュレーション

**タイミング**: ユーザー確認後、実行前、複雑な変更の場合。

```markdown
🔍 影響シミュレーション

• 変更ファイル: [2-5ファイル]
• 影響コンポーネント: [モジュール]
• リスクレベル: 🟢 低 / 🟡 中 / 🔴 高
• 注意点: [考慮事項]
```

**スキップ**: 単純読取、ドキュメントのみ更新、単一ファイル10行未満。

**常に表示**: コア設定変更、3ファイル以上、認証/セキュリティロジック。

## 実行計画

```markdown
📝 実行計画

1. [アクション]
2. [アクション]
```

## ワークフロー

1. 理解度チェック → AskUserQuestion（停止）
2. 影響シミュレーション（複雑な場合）
3. 実行計画 → AskUserQuestion（停止）
4. 実行
