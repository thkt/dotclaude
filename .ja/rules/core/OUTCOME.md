# OUTCOME

CLAUDE.md の Foundation Outcome-driven をリポジトリ単位で具体化するルール定義。実体は各リポジトリの `.claude/OUTCOME.md` に置き、CLAUDE.md と独立して読めるようリポルートには置かない。additive な層で、既存 CLAUDE.md の migration は強制しない。

## なぜ必要か

リポジトリ単位のアウトカム状態は曖昧なまま放置されやすく、後段の判断 (スコープ、優先度、何を捨てるか) を全部汚染する。OUTCOME.md はそれを見える状態に変え、書く行為そのものが discovery moment になる。

## アウトカムとは何か

アウトカムは在り方の状態。アウトプットが「やったこと」(機能をリリースした、資料を作成した) であるのに対し、アウトカムは「その結果、主体がどう振る舞っているか」という状態。「エラー率 < 0.1%」のようなメトリクスは、その状態を裏付ける Indicator であってアウトカムそのものではない。

## アウトカムテスト

アウトカム文の候補を以下のチェックで判定する。1 つでも fail なら書き直す。

| チェック             | pass 条件                                               |
| -------------------- | ------------------------------------------------------- |
| 主体が明示されている | 主体 (人間/AI エージェント/システム) の状態を述べている |
| 実装非依存           | 実装を一から書き換えても文が成立する                    |
| 完了状態             | 完了状態を述べている。進行中の活動ではない              |
| 観察可能             | ソースを読まずに外部観察者が状態を検証できる            |

## 中身

Behavior は ~/.claude/rules/conventions/VAGUE_TERMS.md を厳格に守る。プロジェクトの存在意義と Indicators は任意で、理想方向の曖昧表現はこの 2 つのみ許容する。

| セクション    | 内容                                                                          |
| ------------- | ----------------------------------------------------------------------------- |
| Outcome state | 冒頭文。完了状態の Behavior (必須 ≥1)。Indicators (Time / Error rate / Value) |
| Non-goals     | 明示的にスコープ外とするもの                                                  |
| Constraints   | 動かせない技術、法的、組織的制約                                              |

## 更新トリガー

| トリガー                   | 動作                                             |
| -------------------------- | ------------------------------------------------ |
| Outcome state が達成された | 埋めた Behavior と Indicators を次フェーズに更新 |
| Non-goals が変わった       | Non-goals を更新                                 |
| Constraints が変わった     | Constraints を更新                               |
| 半年経過                   | 全 section を見直す                              |
