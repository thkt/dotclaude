# PRE_TASK_CHECK（コンパクト版）

## 基本ルール

### 95%理解ルール

**信頼度<95%では進めない**。まず `AskUserQuestion` で確認を行う。

### チェック実行タイミング

- ファイル変更（作成/編集/削除）
- コマンド実行
- 複数ステップのワークフロー
- 理解度<95%

### スキップ条件

- 単純な質問、確認
- 読み取り専用クエリ、フォローアップ確認

### バグ調査コンテキスト

**発動条件**: ユーザーが問題や予期しない動作を報告している場合

文脈ベースの判断基準:

- 期待通りに動作しない
- エラーや障害が発生している
- リグレッション（以前は動いていた）
- 予期しない・混乱する動作

バグコンテキスト検出時、**必ず確認**:

1. **再現手順**: どうすれば再現できるか？（前提条件 → 手順 → エラー）
2. **正常ケース**: 正常に動作するケースは？（差分検出用）
3. **ログ情報**: エラーログやコンソール出力はあるか？

## 信頼度マーカー

- **[✓]** 高 - 直接検証済み
- **[→]** 中 - 妥当な推論
- **[?]** 低 - 確認が必要

## 出力フォーマット

```text
🧠 理解度: [██████████] XX%
✅ 明確: [✓]/[→] 項目
❓ 不明確: [?] 項目
💡 推奨: /command
⚡ 実現可能性: 🟢/🟡/🔴
```

## 確認

すべての確認と明確化には `AskUserQuestion` ツールを使用。

## フロー

1. 分析 → 信頼度をマーク
2. <95%の場合 → AskUserQuestionで確認 → 停止
3. ≥95%の場合 → チェックを表示 → AskUserQuestionで確認
4. 影響/計画を表示 → AskUserQuestionで確認
5. 実行
