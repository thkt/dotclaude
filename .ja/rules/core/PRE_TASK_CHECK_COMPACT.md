# PRE_TASK_CHECK（コンパクト版）

## 基本ルール

### 95%理解ルール

**信頼度<95%では進めない**。まず `AskUserQuestion` で確認を行う。

### チェック実行タイミング

- ファイル変更（作成/編集/削除）
- コマンド実行
- 複数ステップのワークフロー
- 理解度<95%

### スキップ条件

- 単純な質問、確認
- 読み取り専用クエリ、フォローアップ確認

## 信頼度マーカー

- **[✓]** 高 - 直接検証済み
- **[→]** 中 - 妥当な推論
- **[?]** 低 - 確認が必要

## 出力フォーマット

```text
🧠 理解度: [██████████] XX%
✅ 明確: [✓]/[→] 項目
❓ 不明確: [?] 項目
💡 推奨: /command
⚡ 実現可能性: 🟢/🟡/🔴
```

## 確認

すべての確認と明確化には `AskUserQuestion` ツールを使用。

## フロー

1. 分析 → 信頼度をマーク
2. <95%の場合 → AskUserQuestionで確認 → 停止
3. ≥95%の場合 → チェックを表示 → AskUserQuestionで確認
4. 影響/計画を表示 → AskUserQuestionで確認
5. 実行
