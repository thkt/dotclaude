# 完了定義と判断フレームワーク

このモジュールは`/code`コマンドの完了基準と判断サポートを定義します。

→ **共通基準**: [@~/.claude/rules/development/COMPLETION_CRITERIA.md](~/.claude/rules/development/COMPLETION_CRITERIA.md)

## /code固有の基準

実装完了の条件:

- [x] 全RGRCサイクル完了
- [x] 機能が仕様通りに動作
- [x] エッジケースを処理
- [x] 品質ゲートに合格（共通基準を参照）

重要なメトリクスで信頼度 < 0.8 の場合は改善を継続。

## 判断フレームワーク

### 実装信頼度が低い場合

```markdown
## 低信頼度検出 (< 0.7)
### 問題: [実装アプローチが不確実]

オプション:
1. **さらに調査** (/research)
   - 時間: +30分
   - 信頼度向上: +0.3

2. **まずプロトタイプ**
   - 時間: +15分
   - 信頼度向上: +0.2

3. **ドキュメントを参照**
   - 時間: +10分
   - 信頼度向上: +0.15

推奨: 複雑な機能にはオプション1
```

### 信頼度ベースの判断

エビデンスに基づいて実装を選択:

- **高信頼度 (>0.8)**: 実装を進める
- **中程度 (0.5-0.8)**: 防御的チェックを追加
- **低 (<0.5)**: 実装前に調査

## 実装パターン

### 信頼度によるパターン選択

```markdown
## 利用可能なパターン（コンテキストに基づいて選択）

### 高信頼度パターン (>0.9)
1. **Factoryパターン** - オブジェクト作成
   - 条件: 類似オブジェクトが複数
   - 信頼度: 0.95
   - 例: src/factories/

2. **Repositoryパターン** - データアクセス
   - 条件: データベース操作
   - 信頼度: 0.92
   - 例: src/repositories/

### 中信頼度パターン (0.7-0.9)
1. **Observerパターン** - イベント処理
   - 条件: 疎結合が必要
   - 信頼度: 0.85
   - 検討: 組み込みEventEmitter

### 実験的パターン (<0.7)
1. **新しいアーキテクチャパターン**
   - 信頼度: 0.6
   - 推奨: まずプロトタイプ
```

## 結果に基づく次のステップ

- **高信頼度 (>0.9)** → `/test`またはレビューの準備完了
- **中程度 (0.7-0.9)** → 追加テストを検討
- **低 (<0.7)** → `/research`または計画が必要
- **品質問題** → 進む前に修正
- **全てグリーン** → PR/コミットの準備完了
