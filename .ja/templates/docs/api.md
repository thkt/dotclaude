# APIテンプレート

## 構造

```markdown
# <project_name> - API仕様

## ベースURL

\`<base_url>\`

## 認証

| 方式   | ヘッダー | 説明          |
| ------ | -------- | ------------- |
| <type> | <header> | <description> |

## エンドポイント

### <resource>

#### <METHOD> <path>

**説明**: <description>

**リクエスト**:
\`\`\`json
{
"<field>": "<type>"
}
\`\`\`

**レスポンス**:
\`\`\`json
{
"<field>": "<type>"
}
\`\`\`

**ステータスコード**:
| コード | 説明 |
|-------|------|
| 200 | 成功 |
| 400 | 不正なリクエスト |
| 401 | 認証エラー |

## エラー形式

\`\`\`json
{
"error": {
"code": "<error_code>",
"message": "<message>"
}
}
\`\`\`

## 型定義

| 型名        | フィールド | 説明          |
| ----------- | ---------- | ------------- |
| <type_name> | <fields>   | <description> |
```

## ガイドライン

| セクション     | 説明                                    |
| -------------- | --------------------------------------- |
| ベースURL      | APIのベースエンドポイント               |
| 認証           | 認証方式とヘッダー                      |
| エンドポイント | リソース別、リクエスト/レスポンス例付き |
| エラー形式     | 標準エラーレスポンス構造                |
| 型定義         | 共有データ型とスキーマ                  |
