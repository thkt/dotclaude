# ドキュメントテンプレート

## 構造

```markdown
## 概要

[追加・変更が必要なドキュメント]

## タイプ

- [ ] 追加（新規ドキュメント）
- [ ] 更新（既存の改善）
- [ ] 修正（typo、誤り、古い情報）

## 場所

[ドキュメントの場所 - ファイルパスまたはセクション]

## 詳細

[具体的な変更内容]
```

## ガイドライン

| フィールド | 説明                               |
| ---------- | ---------------------------------- |
| 概要       | どのドキュメント変更が必要か       |
| タイプ     | 追加、更新、修正のいずれか         |
| 場所       | 変更対象のファイルまたはセクション |
| 詳細       | 具体的に何を書く/変更するか        |
