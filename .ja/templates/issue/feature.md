# 機能テンプレート

## 構造

```markdown
## 概要

[What: 何を作るか — 1-2文で] [Why: なぜ必要か — ユーザー課題 or ビジネス理由]

## 受け入れ基準

- [ ] [具体例付きの振る舞い: 「○○すると△△になる」]
- [ ] [具体例付きの振る舞い]
- [ ] [具体例付きの振る舞い]

## スコープ

- **スコープ内**: [この Issue で対応すること]
- **スコープ外**: [この Issue では対応しないこと — 明示的に除外]

## 制約（任意）

- [技術制約: 使用ライブラリ、パフォーマンス要件など]
- [禁止事項: やってはいけないアプローチ]
- [依存: 先に完了すべき Issue や前提条件]

## 備考（任意）

[参考リンク、スクリーンショット、関連 Issue など]
```

## ガイドライン

| フィールド   | 説明                                                                 |
| ------------ | -------------------------------------------------------------------- |
| 概要         | What + Why を分けて書く。Why がないと AI が目的を誤解する            |
| 受け入れ基準 | 「○○すると△△になる」形式の具体例。実装手段ではなく観察可能な振る舞い |
| スコープ     | スコープ外は必須。書かないと AI がスコープ外まで実装する             |
| 制約         | 「やるな」を明示。AI のハルシネーション防止に最も効く                |
| 備考         | 判断材料になるリンクや画像があれば貼る                               |
