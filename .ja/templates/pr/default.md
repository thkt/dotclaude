# PR テンプレート

## 構造

```markdown
## What & Why

[この PR で何をするか — 1-2文]
[なぜ — どの課題を解決し、何を可能にするか]

## 変更内容

- [変更 1: 何をして、なぜそうしたか]
- [変更 2: 何をして、なぜそうしたか]

## スコープ（任意）

- **含まない**: [この PR で意図的にやらないこと]

## 設計判断

- [なぜこのアプローチを選んだか]

## テスト方法

1. [手順]
2. [期待結果]

## 関連

- Closes #[issue]
```

## ガイドライン

| フィールド | OK                                                 | NG                                       |
| ---------- | -------------------------------------------------- | ---------------------------------------- |
| What & Why | 「CSV出力を追加し、オフライン分析を可能に」         | 「CSV出力機能を追加」（Whyなし）          |
| 変更内容   | 「ExportButton追加 — メニューより1クリックで操作」  | 「ファイルを追加」（理由なし）            |
| スコープ   | 「認証トークン更新は含まない（別PR）」              | （大きいPRで省略 — レビュアーが境界を推測）|
| 設計判断   | 「大規模データのOOM回避のためストリーミングを採用」  | （省略 — レビュアーがなぜか推測する）     |
| テスト方法 | 「エクスポート → .csvが3行でDLされることを確認」    | 「機能をテスト」（曖昧）                  |
