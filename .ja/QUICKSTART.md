# クイックスタート（5分）

## 1. 基本コマンド

| コマンド    | 使用場面           |
| ----------- | ------------------ |
| `/fix`      | 小さなバグ、即修正 |
| `/code`     | TDDで新機能開発    |
| `/research` | 実装前の調査       |
| `/audit`    | コード品質レビュー |
| `/commit`   | コミットメッセージ |

## 2. 判断フロー

```text
すぐ直せる？ → /fix
まず理解が必要？ → /research → /fix
機能開発？ → /code → /audit
```

## 3. セッション例

```bash
# 即修正
> /fix ログインボタンが動かない

# 機能開発
> /research このコードベースの認証はどう動いてる？
> /code ログアウト機能を追加
> /audit
> /commit
```

## 4. 基本原則

- **一度に一つ**: 各コマンドの完了を待つ
- **ワークフローを信頼**: コマンドは自然に連鎖
- **不明時は質問**: Claudeが確認してくれる
