# /preview - PR スクリーニングレビュー

## 入力

- PR 参照: `$1`（URL、番号、または空 → current branch から自動検出）

## 実行

| Step | アクション                                                                                               |
| ---- | -------------------------------------------------------------------------------------------------------- |
| 1    | PR 特定: `gh pr view $1 --json number,title,body,labels,files,url`（フォールバック: `$1` 省略）          |
| 2    | PR 未検出または作業ツリーが dirty (`git status --porcelain`) の場合中止                                  |
| 3    | PR チェックアウト: `gh pr checkout $PR`                                                                  |
| 4    | コンテキスト並列取得（diff、コメント、インラインコメント）— skill 参照                                   |
| 5    | 各変更ファイルをローカルで読み取り                                                                       |
| 6    | skill に従いレビュー: 概要 → ファイル別 → 依存影響 → 指摘                                                |
| 7    | 構造化されたスクリーニングレポートを出力                                                                 |

## 出力

```markdown
## PR スクリーニングレポート

### 概要
{この PR の背景と目的を2-3文で}

### 変更サマリ
| ファイル | 変更内容 |
| -------- | -------- |

### 依存影響
{変更の影響を受けるファイル、回帰リスク評価}

---

### 要対応
{`[must]` と `[want]` の指摘（file:line 付き）}

### 参考情報
{`[imo]`, `[ask]`, `[nits]`, `[info]` の項目（file:line 付き）}

---

### レビューコメント提案
{ファイル別にグループ化されたレビュアー向けコメント（ラベル付き）}
```

## ルール

| ルール         | 詳細                                                                                   |
| -------------- | -------------------------------------------------------------------------------------- |
| 自動投稿禁止   | PR にコメントを自動投稿しない                                                          |
| dirty 中止     | 未コミット変更がある場合は警告して中止                                                 |
| 速度優先       | スクリーニングであり、完全な監査ではない                                               |
| 指摘前に検証   | [ask]/[want]以上の指摘を出す前に、実際のユーザー操作経路で問題が発現するか確認すること |
