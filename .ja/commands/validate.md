# /validate - SOW基準チェッカー

手動検証のためにSOW受け入れ基準を表示。

## 入力

- 機能名: `$1`（任意）
- `$1`が空の場合 → 最新SOWを使用

## 実行

1. Globで SOWを検索（`$HOME/.claude/workspace/planning/*/sow.md`）
2. 受け入れ基準セクションを抽出
3. 各ACを検証
4. チェックリストとして表示

## AC検証

| チェック | 質問                             |
| -------- | -------------------------------- |
| 実装済み | コードはこのACを実装しているか？ |
| テスト済 | テストはこのACを検証しているか？ |
| 文書化   | 動作は文書化されているか？       |
| レビュー | /auditでレビューされたか？       |

## 合格/不合格基準

| スコア | ステータス | アクション           |
| ------ | ---------- | -------------------- |
| 100%   | PASS       | リリース可能         |
| 90-99% | WARN       | ギャップを確認       |
| <90%   | FAIL       | リリース前に対処必須 |

## SOWステータス更新

| スコア | ステータス更新                     |
| ------ | ---------------------------------- |
| 100%   | `Status:` → `completed` に更新     |
| <100%  | 現在のステータスを維持（変更なし） |

## Todo完了

| 条件  | アクション                          |
| ----- | ----------------------------------- |
| 100%  | `TaskUpdate` 残りすべて → completed |
| <100% | スキップ                            |

## 出力

```markdown

| AC ID  | 説明   | 確認       | Status |
| ------ | ------ | ---------- | ------ |
| AC-001 | [説明] | [検証内容] | [ ]    |
| AC-002 | [説明] | [検証内容] | [ ]    |
```
