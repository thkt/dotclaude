# /sow - SOWジェネレーター

## 目的

計画と分析のためにsow.mdのみ（単一成果物）を生成。

## 入力解決

```text
/sow 実行
    │
    ├─ 引数提供？ ─YES──→ 引数をタスク説明として使用
    │
    └─ 引数なし
           │
           ├─ リサーチコンテキスト存在？ ─YES──→ コンテキストからトピック抽出、発見を組み込む
           │
           └─ なし ──→ ユーザーに質問:
                       「何を計画しますか？タスク説明を入力してください。」
```

### リサーチコンテキスト検出

```bash
!`ls -t .claude/workspace/research/*-context.md 2>/dev/null | head -1 || ls -t ~/.claude/workspace/research/*-context.md 2>/dev/null | head -1 || echo "(no research context)"`
```

見つかった場合:

- コンテキストファイルからトピック抽出
- リサーチ発見をSOWに組み込む
- 表示: `📄 リサーチコンテキスト使用: [filename]`

## テンプレート参照

**構造とセクション順序のみ**に使用:
[@../../templates/sow/workflow-improvement.md]

**重要**:

- ✅ コピー: セクション構造、表フォーマット、ID命名（I-001、AC-001、R-001）
- ❌ コピーしない: 実際のコンテンツ、具体的な値
- ユーザーの機能説明に基づいて新鮮なコンテンツを生成

## 信頼度マーカー

全体を通じて数値形式 `[C: X.X]` を使用:

| 範囲         | 意味     | 必要な証拠                    |
| ------------ | -------- | ----------------------------- |
| [C: 0.9+]    | 検証済み | file:line、コマンド出力、ログ |
| [C: 0.7-0.9] | 推論     | 推論根拠を記載                |
| [C: <0.7]    | 不確実   | 調査が必要                    |

**YAMLフロントマター**: ドキュメントレベルのスコアに `confidence.overall` を含める。

## コードベース分析（オプション）

グリーンフィールドでないプロジェクトには、Planエージェントを呼び出す:

```typescript
Task({
  subagent_type: "Plan",
  model: "haiku",
  description: "機能コンテキストのためにコードベースを分析",
  prompt: `機能: "${featureDescription}"
調査: 既存パターン、影響モジュール、技術スタック。
マーカー付きで返す: [C: 0.9+] 検証済み、[C: 0.7-0.9] 推論、[C: <0.7] 不確実。`,
});
```

## 必須セクション

テンプレート構造に従う:

1. **エグゼクティブサマリー** - ハイレベル概要 [C: 70%]
2. **問題分析** - 現状 [C: ≥90%]、信頼度別の課題
3. **前提条件と要件** - 事実 [C: ≥90%]、仮定 [C: 70%]、不明 [C: <50%]
4. **ソリューション設計** - アプローチ、代替案、推奨
5. **テスト計画** - Unit/Integration/E2Eと優先度
6. **受け入れ基準** - フェーズ別、信頼度マーカー付き
7. **実装計画** - ステップ付きフェーズ、**Progress Matrix含む**
8. **成功メトリクス** - 測定可能な成果
9. **リスクと緩和策** - 信頼度レベル別
10. **検証チェックリスト** - 実装前チェック
11. **参照** - 関連ドキュメント

## Progress Matrix（PDD統合） - オプション

実装計画セクションに含める。Progress-Driven Development追跡を有効化。

### フォーマット

```markdown
### Progress Matrix

| 機能  | spec | design | impl | test | review | 進捗 |
| ----- | :--: | :----: | :--: | :--: | :----: | :--: |
| 機能A |  ⬜  |   ⬜   |  ⬜  |  ⬜  |   ⬜   |  0%  |
| 機能B |  ⬜  |   ⬜   |  ⬜  |  ⬜  |   ⬜   |  0%  |

**凡例**: ⬜ なし | 🔄 開始 | 📝 ドラフト | 👀 レビュー | ✅ 完了
```

### ステップ（5列）

| ステップ | 説明               | 完了基準                       |
| -------- | ------------------ | ------------------------------ |
| spec     | 要件定義           | SOW/Specセクション完了         |
| design   | アーキテクチャ決定 | ソリューション設計承認         |
| impl     | コード実装         | コア機能動作                   |
| test     | テスト合格         | Unit/Integrationテストグリーン |
| review   | 品質検証           | /audit + /validate合格         |

### 進捗計算

- 各ステップ = 20%（合計5ステップ）
- ステータス重み: ⬜=0%、🔄=25%、📝=50%、👀=75%、✅=100%
- 機能進捗 = （Σステップ重み）/ 5

### 使い方

1. **初期**: すべての機能がすべての列で⬜から開始
2. **/code中**: 作業進行に応じてステータス更新
3. **/validateでレビュー**: マトリクスが実際の状態を反映しているか確認

## 出力

保存先: `.claude/workspace/planning/[timestamp]-[feature]/sow.md`

```bash
# 出力場所を検出
```

## 例

```bash
# 明示的な引数付き
/sow "OAuthでユーザー認証を追加"

# /research後（コンテキストを自動検出）
/research "ユーザー認証オプション"
/sow  # リサーチコンテキストを自動使用

# コンテキストなし、引数なし → 入力を要求
/sow
# → 「何を計画しますか？タスク説明を入力してください。」
```
