# /pr - プルリクエスト説明生成

現在のブランチとベースブランチを比較し、すべての変更を分析して包括的なPR説明を生成します。

**実装**: このコマンドは最適なパフォーマンスとコンテキスト効率のため、専門の `pr-generator` サブエージェントに委譲します。

## 動作概要

このコマンドを実行すると:

1. Task ツールで `pr-generator` サブエージェントを起動
2. サブエージェントがベースブランチを動的に検出（main/master/develop）
3. git diff、コミット履歴、ファイル変更を分析
4. 包括的なPR説明を生成
5. 複数のテンプレート案を返却

## 使い方

### 基本的な使い方

```bash
/pr
```

現在のブランチの変更からPR説明を生成します。

### Issue参照付き

```bash
/pr "#456"
```

特定のIssueにリンクします。

### カスタムコンテキスト付き

```bash
/pr "このPRはチームミーティングで議論された新しい認証フローを実装します"
```

追加のコンテキストを説明に組み込みます。

## PR説明の構成

### 必須セクション

1. **概要**: ハイレベルな概要
2. **動機**: なぜこの変更が必要か
3. **変更内容**: 詳細な内訳
4. **テスト**: 検証手順
5. **関連**: リンクされたIssue/PR

### オプションセクション

- **スクリーンショット**: UI変更の場合
- **破壊的変更**: API変更の場合
- **パフォーマンス影響**: 最適化の場合
- **移行ガイド**: 破壊的変更の場合

## 出力形式

コマンドは以下を提供します:

- **ブランチ分析**: 現在/ベースブランチ、コミット数、ファイル数、変更行数
- **変更概要**: タイプ、影響コンポーネント、破壊的変更、テストカバレッジ
- **推奨テンプレート**: 分析に基づいた包括的なPR説明
- **代替フォーマット**: 詳細版、簡潔版、カスタム版
- **使用方法**: 説明を使ってPRを作成する方法

## ワークフローとの統合

以下とシームレスに連携:

- `/branch` - まずブランチを作成
- `/commit` - コミットを作成
- `/pr` - PR説明を生成
- `/audit` - PR後のコードレビュー

## 技術詳細

### サブエージェントのメリット

- **90%のコンテキスト削減**: git操作のみ、コードベース読み込みなし
- **2-3倍の高速実行**: git分析に最適化された軽量エージェント
- **専門ロジック**: PR説明生成に特化
- **並列実行**: 他の操作と同時実行可能

### 使用するGit操作

サブエージェントはgitコマンドのみを実行:

- `git symbolic-ref` - ベースブランチ検出
- `git diff` - ブランチ比較
- `git log` - コミット分析
- `git status` - 現在の状態確認
- ファイルシステムアクセスやコード解析なし

### ベースブランチ検出

サブエージェントは自動的にベースブランチを検出:

1. 試行: `git symbolic-ref refs/remotes/origin/HEAD`
2. フォールバック: `main` → `master` → `develop`
3. 検証なしには仮定しない

## 関連コマンド

- `/branch` - ブランチ名を生成
- `/commit` - コミットメッセージを生成
- `/audit` - コードレビュー

## ベストプラクティス

1. **コミット後にPR作成**: すべての変更がコミットされていることを確認
2. **コンテキストを含める**: 動機と目標を提供
3. **テスト手順を追加**: レビュアーの検証を支援
4. **Issueをリンク**: 関連するIssueに接続
5. **提出前にレビュー**: 生成された説明を確認

## コンテキスト効率

このコマンドは最小限のコンテキスト使用に最適化:

- ✅ コードベースファイルの読み込みなし
- ✅ gitメタデータのみを分析
- ✅ 高速実行（10秒未満）
- ✅ 他のタスクと並列実行可能

## スマート機能

### 自動検出

- コミット/ブランチからのIssue番号
- 変更タイプ（feature/fix/refactor）
- 破壊的変更

- 影響コンポーネント

### パターン認識

- API変更
- UI更新
- データベース変更
- 設定変更
- 依存関係更新
