# /kousei - 日本語校正

textlintで日本語テキストを校正し、自動修正する。

## 入力

- `$1`: ファイルパス / `#123`（Issue）/ `#pr123`（PR）/ テキスト
- 空の場合 → 直近で編集した `.md` ファイルを対象

## 言語サポート

| 言語   | textlint（機械的） | 構造レビュー（AI） |
| ------ | ------------------- | ------------------- |
| 日本語 | 実行                | 実行                |
| 英語   | スキップ            | 実行                |

判定：ひらがな・カタカナ・漢字が50文字以上 → 日本語。それ以外 → 英語（構造レビューのみ）。

## 実行手順

| Step | Action                                               |
| ---- | ---------------------------------------------------- |
| 1    | 対象を判定                                           |
| 2    | 修正前の内容を取得                                   |
| 3    | `bun x textlint --fix` を実行（日本語のみ）          |
| 4    | 構造レビュー（AIによるチェック、下記参照、全言語対応）|
| 5    | textlint の diff + 構造フィードバックを表示          |
| 6    | Issue/PR の場合 → 修正内容を提示し、更新を確認       |

### 対象の判定

| 入力パターン           | 処理                                                  |
| ---------------------- | ----------------------------------------------------- |
| `*.md` ファイルパス    | そのファイルに `textlint --fix` を実行                |
| `#N`（Issue番号）      | `gh issue view N --json body` → fix → `gh issue edit` |
| `#prN`（PR番号）       | `gh pr view N --json body` → fix → `gh pr edit`       |
| それ以外のテキスト     | 一時ファイルに書き出して fix → 結果を表示             |
| 空                     | `git diff --name-only` で直近の .md を取得            |

### textlint 実行

```bash
cd ~/.claude/textlint && bun x textlint --fix --config .textlintrc.json "$file"
```

### 構造レビュー（ワークスロップ防止）

textlint修正後、以下の構造品質チェックを行う：

| チェック項目       | 確認すること                                                          |
| ------------------ | --------------------------------------------------------------------- |
| 筆者の判断         | 筆者自身の結論が1〜3行で冒頭に書かれているか？AI出力が主役、判断が脇役になっていないか。 |
| 半分にできるか     | この文書は半分にできるか？できないなら、何が重要か分かっていない可能性がある。 |
| 事実と意見の区別   | 事実・推測・提案にラベルが貼られ、明確に分離されているか？             |
| アクションの明確さ | 読み手に求める行動が具体的か？（「ご確認ください」ではなく「Xを判断してください」） |
| 読み手の認知負荷   | この文書は読み手の負荷を下げているか、仕事を押し付けていないか？       |

改善が必要な項目のみ報告する。構造に問題がなければスキップ。

フォーマット：

```
構造レビュー：
- [チェック項目]: [具体的な改善提案]
```

### Issue/PR 更新フロー

1. bodyを取得 → 一時 `.md` ファイルに書き出し
2. `textlint --fix` を実行
3. diffを表示
4. ユーザーに確認：「この修正を反映しますか？」
5. 承認 → `gh issue edit N --body "..."` or `gh pr edit N --body "..."`

## 表示フォーマット

修正ありの場合：

```
校正完了：N 件修正しました

[修正内容の diff]
```

修正なしの場合：

```
校正完了：指摘なし
```
