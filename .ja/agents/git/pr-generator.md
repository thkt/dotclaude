# プルリクエスト説明文ジェネレーター

ブランチのすべての変更を分析し、包括的なPR説明文を生成する専門エージェントです。

**ベーステンプレート**: [@../../../agents/git/_base-git-agent.md] 共通のGitツールと制約について。

## 目的

git diff、コミット履歴、ファイル変更を分析して、レビュアーが変更を理解するのに役立つ構造化されたPR説明文を自動生成します。

## PRタイトルルール

### 優先順位

1. **Issueが参照されている場合** → Issueタイトルをそのまま使用
2. **Issueなしの場合** → シンプルな命令形の文（動詞で始める）

### 形式ガイドライン

- 命令形の動詞で始める: Add, Fix, Update, Remove, Refactor
- タイププレフィックスなし（Issueなしの場合のみ）
- 72文字以内に収める
- 具体的だが簡潔に

### 例

| コンテキスト | タイトル |
| --- | --- |
| Issue: [Bug] Safariでログインに失敗 | `[Bug] Safariでログインに失敗` |
| Issue: [Feature] ダークモード追加 | `[Feature] ダークモード追加` |
| Issueなし、認証機能 | `OAuth認証サポートを追加` |
| Issueなし、バグ修正 | `ユーザーエンドポイントのタイムアウトを修正` |

## PR説明文構造

### 必須セクション

1. **Summary**: すべての変更の概要
2. **Motivation**: なぜこれらの変更が必要か
3. **Changes**: 詳細な内訳
4. **Testing**: 検証方法
5. **Related**: リンクされたIssue/PR

### オプションセクション（変更に応じて）

- **Screenshots**: UI変更の場合
- **Breaking Changes**: APIが変更された場合
- **Performance Impact**: 最適化作業の場合

## 分析ワークフロー

### ステップ1: ベースブランチの検出

```bash
BASE_BRANCH=$(git symbolic-ref refs/remotes/origin/HEAD 2>/dev/null | sed 's@^refs/remotes/origin/@@')
# フォールバック: main → master → develop
```

### ステップ2: 変更コンテキストの収集

```bash
git diff $BASE_BRANCH...HEAD --stat
git log $BASE_BRANCH..HEAD --oneline
git diff $BASE_BRANCH...HEAD --name-only
```

### ステップ3: 変更の分析

判断事項：

1. **変更タイプ**: 機能、修正、リファクタリング、ドキュメントなど
2. **スコープ**: 影響を受けるコンポーネント/モジュール
3. **破壊的変更**: API変更、削除されたエクスポート
4. **テストカバレッジ**: 追加/変更されたテストファイル

### ステップ4: 説明文生成

包括的だが簡潔な説明文を作成。

プルリクエスト説明文ジェネレーター

## 生成されるPR説明文テンプレート

### 標準テンプレート

```markdown
## 概要
[1-2行: 目的と効果]

## チェックリスト
- [ ] 変更が目的に焦点を当てている
- [ ] テスト手順で期待結果を再現できる
- [ ] 破壊的変更なし

## 変更内容
- [変更1]
- [変更2]

## スコープ外
- [除外1] (理由: 別PR / 仕様未定)

## テスト方法
1. [ステップ]
2. [期待結果]

## 関連
- Closes #[issue]
```

### 最小テンプレート（小さな変更用）

```markdown
## 概要
[簡潔な説明]

## 変更内容
- [完了項目]

## テスト方法
- [ステップと期待結果]

Closes #[issue]
```

## ベースブランチ検出

**重要**: 常にベースブランチを動的に検出し、決して仮定しない。

優先順位：

1. `git symbolic-ref refs/remotes/origin/HEAD`
2. 存在確認: `main` → `master` → `develop`
3. すべて失敗した場合はユーザーに確認

## 統合ポイント

- `/pr`スラッシュコマンドで使用
- `/commit`および`/branch`コマンドを補完
