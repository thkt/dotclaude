# レビュー共通パターン

全レビューエージェントが参照する共通パターン。

## 1. 信頼度マーカーシステム

| マーカー | 信頼度 | 用途                      |
| -------- | ------ | ------------------------- |
| [✓]      | ≥95%   | 検証済み（file:line必須） |
| [→]      | 70-94% | 推論（根拠を記述）        |
| [?]      | <70%   | 仮説（確認が必要）        |

**エビデンス要件**: すべての発見事項にはfile:line、コードスニペット、影響を含める。

## 2. 標準レビュープロセス

1. **コンテキスト発見**: プロジェクト規約、既存パターンの特定
2. **評価**: スキル知識の適用、パターン検出、信頼度評価
3. **フィルタリング**: 閾値未満（70%）を除外、エビデンスを記録

## 3. 統合パターン

エージェント間で発見事項が重複する場合：

- **主担当**: ドメインが中心となるエージェント
- **副担当**: 関連する発見事項を相互参照
- **重複なし**: 同一の問題を複数エージェントが報告しない

## 4. ブラウザ/MCP検証（オプション）

対象: accessibility-reviewer, performance-reviewer

| 使用する場合           | スキップする場合 |
| ---------------------- | ---------------- |
| 複雑なインタラクション | 静的HTML/CSS     |
| カスタムARIA           | 開発サーバーなし |
| パフォーマンス計測     | コード分析で十分 |

フォールバック: MCP利用不可の場合、[→]マーカーでコードのみの分析。

## 5. JP/EN翻訳の取り扱い

`.ja/`以下のファイルは日本語翻訳。

| レビュー対象   | スキップ対象     |
| -------------- | ---------------- |
| 構造の一致     | 内容の比較       |
| YAMLの一致     | 翻訳の違い       |
| リンクの有効性 | ローカライズ形式 |
