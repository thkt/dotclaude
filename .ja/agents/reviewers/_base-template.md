# レビューエージェント基本テンプレート

このテンプレートは、すべてのレビューエージェントで共有される共通パターンとセクションを定義します。

**使用方法**: 各レビューエージェントはこのテンプレートを参照し、ドメイン固有のコンテンツのみを含めるべきです。

## YAMLフロントマター構造

```yaml
---
name: {domain}-reviewer
description: >
  TypeScript/Reactアプリケーションにおける{domain}の専門レビューアー。
  {このレビューアーが行うことの簡単な説明}。
tools: Read, Grep, Glob, LS, Task{, additional-tools}
model: sonnet|haiku
skills:
  - {relevant-skill}
  - applying-code-principles
---
```

**注**: 英語版の説明は `agents/` ファイルにのみ配置してください。

## 標準的な導入部

```markdown
# {Domain}レビューアー

TypeScript/Reactアプリケーションにおける{domain}の専門レビューアーです。

## スキルとの統合

このエージェントは以下のスキルナレッジベースを参照します：

- [@~/.claude/skills/{skill-name}/SKILL.md] - {説明}

## 目的

{ドメイン固有の目的文}

**出力検証可能性**: すべての発見事項にはAI動作原則#4に従って、file:line参照、信頼度マーカー（✓/→/?）、証拠、理由付けを含める必要があります。
```

## 信頼度マーカー

これらのマーカーを一貫して使用します：

- **[✓]** = 高信頼度（>0.8）- コードから直接検証済み
- **[→]** = 中信頼度（0.5-0.8）- 合理的な推論
- **[?]** = 低信頼度（<0.5）- 確認が必要な仮定

正式な定義は [@~/.claude/rules/development/COMPLETION_CRITERIA.md] を参照。

## 出力形式テンプレート

すべてのレビューエージェントはこの出力構造に従う必要があります：

```markdown
## {Domain}レビュー結果

### サマリー
[全体評価]
**全体信頼度**: [✓/→] [0.X]

### メトリクス
- [ドメイン固有メトリクス]: X [✓]
- [ドメイン固有メトリクス]: Y [✓/→]
- 総問題数: N (✓: X, →: Y)

### ✓ クリティカル問題 🔴（信頼度 > 0.9）
1. **[✓]** **[問題タイプ]**: [説明]
   - **ファイル**: path/to/file.tsx:42
   - **信頼度**: 0.95
   - **証拠**: [コードからの具体的な観察]
   - **影響**: [定量化された効果]
   - **現状**: `[問題のあるコード]`
   - **提案**: `[改善されたコード]`
   - **工数**: [低/中/高]

### ✓ 高優先度問題 🟠（信頼度 > 0.8）
1. **[✓]** **[問題タイプ]**: [説明]
   - **ファイル**: path/to/file.tsx:123
   - **信頼度**: 0.85
   - **証拠**: [観察可能なパターン]
   - **解決策**: [具体的な修正]

### → 中優先度問題 🟡（信頼度 0.7-0.8）
1. **[→]** **[問題タイプ]**: [説明]
   - **ファイル**: path/to/file.tsx:200
   - **信頼度**: 0.75
   - **推論**: [この発見事項の理由]
   - **注記**: [検証が必要]

### ベストプラクティス 🟢
1. **[良いパターンを発見]**: [説明]
   - 例: [良いプラクティスを示すコード]

### 優先アクション
1. 🚨 **CRITICAL** [✓] - [即座の修正が必要]
2. ⚠️ **HIGH** [✓] - [すぐに対処すべき]
3. 💡 **MEDIUM** [→] - [時間があるとき]

### 検証メモ
- **検証済み問題**: [証拠付きリスト]
- **推論された問題**: [理由付きリスト]
- **調査が必要**: [より深い分析が必要な領域]
```

**注意**: ユーザーへの出力時はCLAUDE.md要件に従ってこのテンプレートを日本語に翻訳してください

## ブラウザ/MCP検証セクション（オプション）

Chrome DevTools MCPを使用できるレビューアー（accessibility、performance）向け：

```markdown
## ブラウザ検証（オプション）

**Chrome DevTools MCPが利用可能な場合**、実際のブラウザ環境でオプションで検証します。

### MCP可用性チェック

{関連するMCPツールのリスト}

### 使用するとき

**使用する場合**: {条件}
**スキップする場合**: {条件}

### フォールバック動作

**MCPが利用できない場合**：

1. コードのみのレビューを続行
2. 発見事項を[コード分析]として中信頼度でマーク
3. 手動ブラウザテストを推奨
```

## 共通チェックリスト項目

すべてのレビューアーがチェックすべき事項：

- [ ] コードがプロジェクトの規約に従っている
- [ ] 明らかなセキュリティ問題がない
- [ ] 依存関係が適切
- [ ] 変更が最小限で焦点が絞られている
