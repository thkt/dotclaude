# Claude コマンドリファレンス

体系的なソフトウェア開発をサポートするカスタムコマンド集。

## 利用可能なコマンド

### コア開発

| コマンド    | 目的                                 |
| ----------- | ------------------------------------ |
| `/think`    | SOW作成とバリデーション              |
| `/research` | 実装なしの調査                       |
| `/code`     | TDD/RGRC実装 + IDR生成               |
| `/test`     | 包括的テスト                         |
| `/audit`    | エージェントによるレビュー + IDR更新 |
| `/polish`   | AI生成スロップ除去 + IDR更新         |
| `/validate` | SOW適合性検証 + IDR照合              |
| `/plans`    | 計画ドキュメント（SOW/Spec）一覧     |
| `/spec`     | Spec生成（実装詳細）                 |
| `/sow`      | SOW進捗表示                          |

### クイックアクション

| コマンド  | 目的                              |
| --------- | --------------------------------- |
| `/fix`    | 迅速なバグ修正（think→code→test） |
| `/rabbit` | CodeRabbit AI外部レビュー         |

### 自動化

| コマンド      | 目的                        |
| ------------- | --------------------------- |
| `/auto-test`  | 条件付きfixを伴う自動テスト |
| `/full-cycle` | 完全な開発サイクル          |

### ブラウザ＆ドキュメント

| コマンド  | 目的                          |
| --------- | ----------------------------- |
| `/e2e`    | ブラウザ操作からE2Eテスト     |
| `/adr`    | Architecture Decision Record  |
| `/rulify` | ADRからプロジェクトルール生成 |

### Git操作

| コマンド  | 目的                     |
| --------- | ------------------------ |
| `/branch` | ブランチ名提案           |
| `/commit` | Conventional Commits生成 |
| `/pr`     | PR説明文                 |
| `/issue`  | GitHub Issues            |

## 標準ワークフロー

```text
機能開発: /research → /think → /code → /test → /audit → /validate
バグ修正: /research → /fix
シンプル: /code → /test
```

**Plan Mode**: `Shift+Tab` でアーキテクチャ決定後に実装。

## IDR (Implementation Decision Record)

開発ライフサイクル全体で実装判断を追跡する自動ドキュメント。

| コマンド    | IDRアクション      |
| ----------- | ------------------ |
| `/code`     | 判断を記録して作成 |
| `/audit`    | レビュー結果を追記 |
| `/polish`   | 簡素化内容を追記   |
| `/validate` | SOW ACと照合       |

**配置場所**: `~/.claude/workspace/planning/[feature]/idr.md`

## アーキテクチャ

| レイヤー         | 役割                                 | 例                                        |
| ---------------- | ------------------------------------ | ----------------------------------------- |
| **スキル**       | ユーザーワークフローとナレッジベース | `/code`, `/audit`, `generating-tdd-tests` |
| **エージェント** | 専門的分析                           | `audit-orchestrator`                      |

**注**: v2.1.3 以降、スラッシュコマンドとスキルは統合されました。スキルは `/skill-name` で呼び出し可能で、キーワードに基づいて自動トリガーもされます。

## コマンド選択

| 状況                 | コマンド                | 理由                 |
| -------------------- | ----------------------- | -------------------- |
| 小さなバグ、修正明確 | `/fix`                  | 迅速な反復           |
| 原因不明             | `/research` → `/fix`    | 先に調査             |
| 新機能               | `/think` → `/code`      | 計画→実装            |
| 複雑なアーキテクチャ | Plan Mode (`Shift+Tab`) | 承認を先に           |
| コードレビュー必要   | `/audit`                | エージェントレビュー |
| コミット前           | `/polish`               | AIスロップ除去       |

## 関連ファイル
