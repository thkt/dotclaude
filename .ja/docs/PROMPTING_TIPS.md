# プロンプトテクニック集

Claude Code を効果的に使うためのプロンプトテクニック集。

## やり直し・改善

| プロンプト                                                   | 効果                               | タイミング         |
| ------------------------------------------------------------ | ---------------------------------- | ------------------ |
| 「今の知識全部使って、エレガントな解決策をゼロから実装して」 | 蓄積した文脈で最善の実装を引き出す | 中途半端な実装の後 |

## レビュー・検証

| プロンプト                                   | 効果                     | タイミング |
| -------------------------------------------- | ------------------------ | ---------- |
| 「変更をグリルして、テスト通るまでPR作るな」 | Claudeをレビュアーにする | PR作成前   |

## 計算リソース

| プロンプト                    | 効果                       | タイミング   |
| ----------------------------- | -------------------------- | ------------ |
| 「use subagents」を末尾に追加 | サブエージェントで並列処理 | 大きなタスク |

## デバッグ

| プロンプト               | 効果                       | タイミング |
| ------------------------ | -------------------------- | ---------- |
| 「CIの失敗テスト直して」 | マイクロマネジメントしない | CI失敗時   |

## ワークフロー

| テクニック                              | 効果               |
| --------------------------------------- | ------------------ |
| Plan Modeで計画 → 別Claudeでレビュー    | 計画品質の向上     |
| うまくいかなくなったら即Plan Modeに戻る | 無理に押し進めない |
| 修正のたびに「CLAUDE.md更新して」       | 繰り返しミスを防ぐ |
