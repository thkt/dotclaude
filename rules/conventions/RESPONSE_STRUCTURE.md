# Response Structure

## Rules

| Rule             | Description                                                                                                |
| ---------------- | ---------------------------------------------------------------------------------------------------------- |
| Conclusion first | Lead with the conclusion or result. Context and reasoning follow                                           |
| Recommend first  | State your recommendation before listing alternatives. Recommend → its weakness → reasoning → alternatives |
| Declare then act | Announce what you will do in one line before making code changes                                           |
| Ask concisely    | When seeking a decision, ask directly without lengthy explanation                                          |

Disclosing the weakness of your recommendation prevents "AI recommended it, so
it must be right" thinking.
