# 完了基準（共通）

コマンド間で共有される完了基準。

## 信頼度メトリクス

信頼度レベルは実装の確実性を示します：

| レベル | 範囲 | アクション |
| --- | --- | --- |
| 高 | >0.8 | 通常通り進行 |
| 中 | 0.5-0.8 | 防御的チェックを追加 |
| 低 | <0.5 | 実装前に調査 |

**目標**: 完了には全体信頼度 ≥0.9 が必要。

## 品質ゲート

すべての実装で必須：

### 必須（パス必須）

- ✅ **全テスト合格** - 終了コード: 0
- ✅ **Lintエラーなし** - 0エラー（警告は5個未満なら許容）
- ✅ **型エラーなし** - TypeScriptパス
- ✅ **リグレッションなし** - 関連機能が動作

### 推奨

- ⭐ **カバレッジ ≥80%** - 新規コード対象
- ⭐ **ドキュメント更新** - 動作が変更された場合
- ⭐ **コードベースとの一貫性** - 既存パターンに従う

## 出力形式

レポートには信頼度マーカーを使用：

```markdown
- ✅ 仕様通りに機能 [C: 0.93]
- ⚠️ エッジケース処理 [C: 0.75]
- ❌ エラー処理が不完全 [C: 0.40]
```

## 適用原則

実装を導いた原則を文書化：

- **オッカムの剃刀**: 最もシンプルな解決策を選択
- **TIDYINGS**: 関連コードのみを修正
- **プログレッシブエンハンスメント**: UIはCSS優先
- **DRY**: パターンの重複を特定

## エスカレーション

信頼度 < 0.7 の場合：

| 現在のコマンド | エスカレーションパス |
| --- | --- |
| `/fix` | → `/research` または `/think` |
| `/code` | → `/research` で深い理解を |
