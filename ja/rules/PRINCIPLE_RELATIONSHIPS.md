# 原則関係マトリックス

すべての原則間の相互参照を統合し、重複を削減します。

## 参照原則

### コア参照原則

| 原則 | コア原則（同レベル） | 適用プラクティス |
|-----------|------------------------------|-------------------|
| **オッカムの剃刀** | SOLID（SRPのシンプルさ）<br>DRY（重複の排除） | プログレッシブエンハンスメント<br>TDD/RGRC（ベイビーステップ）<br>可読性の高いコード<br>漏れのある抽象化（完璧より簡潔） |
| **SOLID** | オッカムの剃刀（SRPのシンプルさ）<br>DRY（重複の防止）<br>ミラーの法則（SRPの認知的制約） | TDD/RGRC（テストを通じた設計）<br>デメテルの法則 |
| **DRY** | SOLID（DIPによる抽象化）<br>オッカムの剃刀（単一ソース） | TDD/RGRC（テストが重複を明らかにする）<br>Tidyings（段階的な削除） |
| **ミラーの法則** | オッカムの剃刀（シンプルさの定量化）<br>SOLID（SRPの認知的整合性）<br>DRY（認知負荷の軽減） | 可読性の高いコード（認知的基盤）<br>コンテナ/プレゼンテーショナル（責務の制限） |
| **YAGNI** | オッカムの剃刀（最もシンプルな解決策）<br>DRY（真実の単一ソース） | プログレッシブエンハンスメント（最小限の構築）<br>TDD/RGRC（YAGNIを自然に強制） |

### 開発プラクティス

| プラクティス | コア原則 | 関連プラクティス |
|----------|----------------|-------------------|
| **TDD/RGRC** | オッカムの剃刀（ベイビーステップのシンプルさ） | テスト生成（体系的な設計） |
| **プログレッシブエンハンスメント** | オッカムの剃刀（シンプルさ優先） | 漏れのある抽象化（段階的構築）<br>デメテルの法則（段階的結合）<br>可読性の高いコード（段階的改善） |
| **可読性の高いコード** | ミラーの法則（認知的制約）<br>オッカムの剃刀（シンプルさが可読性を向上） | デメテルの法則（よりシンプルなインターフェース）<br>コンテナ/プレゼンテーショナル（明確な分離） |
| **デメテルの法則** | SOLID（原則を補完）<br>オッカムの剃刀（複雑さの軽減） | コンテナ/プレゼンテーショナル（自然な適用）<br>可読性の高いコード（可読性の向上） |
| **漏れのある抽象化** | オッカムの剃刀（完璧よりシンプル）<br>SOLID（DIPが漏れを考慮） | プログレッシブエンハンスメント（段階的構築）<br>デメテルの法則（境界の管理） |
| **コンテナ/プレゼンテーショナル** | ミラーの法則（責務の制限） | デメテルの法則（propsのみのパターン）<br>可読性の高いコード（明確な分離） |

| **AI支援開発** | オッカムの剃刀（AI提案のシンプル化）<br>DRY（AIが重複を発見）<br>SOLID（人間の判断） | TDD/RGRC（AIテスト足場）<br>可読性の高いコード（AI出力の洗練）<br>プログレッシブエンハンスメント（AIでシンプルに開始） |

## 使用方法

各ファイルで完全な「関連原則」セクションを繰り返す代わりに、このマトリックスを参照してください：

```markdown
## 関連原則

参照: [@./PRINCIPLE_RELATIONSHIPS.md](./PRINCIPLE_RELATIONSHIPS.md#参照原則)
- 特定の注記（必要な場合）
```
