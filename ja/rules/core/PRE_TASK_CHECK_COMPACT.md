# PRE_TASK_CHECK (Compact)

## コアルール

### 95%理解ルール

**信頼度が95%未満では絶対に進まない**。まず明確化の質問をする。

### チェックを実行する場合

- ファイル修正（作成/編集/削除）
- コマンド実行
- 複数ステップのワークフロー
- 理解度が95%未満

### スキップする場合

- 簡単な質問、確認（y/yes/ok）
- 読み取り専用クエリ、フォローアップの明確化

## 信頼度マーカー

- **[✓]** 高 - 直接検証済み
- **[→]** 中 - 合理的な推論
- **[?]** 低 - 確認が必要

## 出力フォーマット

```text
🧠 理解度: [██████████] XX%
✅ 明確: [✓]/[→] items
❓ 不明: [?] items
💡 推奨: /command
⚡ 実行可能性: 🟢/🟡/🔴

確認後に実行 (Y/n)
```

## フロー

1. 分析 → 信頼度をマーク
2. 95%未満の場合 → 質問する → 停止
3. 95%以上の場合 → チェックを表示 → Yを待つ
4. 影響を表示 → 実行計画 → Yを待つ
5. 実行
