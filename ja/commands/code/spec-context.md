# 仕様コンテキスト（自動検出）

このモジュールは、実装ガイダンスのためのspec.md自動検出とロードを処理します。

## 最新の仕様を検出

Globを使用してSOWワークスペース内のspec.mdを検索：

- プロジェクトローカル: `.claude/workspace/planning/**/spec.md`
- グローバル: `~/.claude/workspace/planning/**/spec.md`

## 実装用の仕様をロード

**spec.mdが存在する場合**、実装ガイドとして使用：

- **機能要件（FR-xxx）**: 実装内容を定義
- **API仕様**: 正確なリクエスト/レスポンス構造を提供
- **データモデル**: 期待されるデータ構造とバリデーションルールを示す
- **UI仕様**: レイアウト、バリデーション、インタラクションを定義
- **テストシナリオ**: Given-When-Thenでテストケース作成をガイド
- **実装チェックリスト**: 実装進捗を追跡

**spec.mdが存在しない場合**：

- 利用可能な要件に基づいて実装を進める
- まず`/think`を実行して仕様を生成することを検討
- 仮定と設計決定をインラインで文書化

## 統合

これにより、実装が最初から仕様と整合することを保証します。

spec.mdが見つかった場合：

1. 機能要件（FR-xxx）を解析
2. test-generator用のテストシナリオを抽出
3. 実装詳細にAPI仕様を使用
4. 実装チェックリストに対して進捗を追跡
