# 完了定義と決定フレームワーク

このモジュールは `/code` コマンドの完了基準と決定サポートを定義します。

→ **共通基準**: [@~/.claude/ja/rules/development/COMPLETION_CRITERIA.md](~/.claude/ja/rules/development/COMPLETION_CRITERIA.md)

## /code 固有の基準

実装完了の条件：

- ✅ すべてのRGRCサイクル完了
- ✅ 仕様通りに機能動作
- ✅ エッジケース対応済み
- ✅ 品質ゲート合格（共通基準参照）

重要なメトリクスで信頼度 < 0.8 の場合、改善を継続してください。

## 決定フレームワーク

### 実装信頼度が低い場合

```markdown
## 低信頼度検出 (< 0.7)
### 問題: [実装アプローチについて不確実]

選択肢:
1. **追加調査** (/research)
   - 時間: +30分
   - 信頼度向上: +0.3

2. **プロトタイプ優先**
   - 時間: +15分
   - 信頼度向上: +0.2

3. **ドキュメント参照**
   - 時間: +10分
   - 信頼度向上: +0.15

推奨: 複雑な機能には選択肢1
```

### 信頼度ベースの決定

証拠に基づいて実装選択を行う：

- **高信頼度 (>0.8)**: 実装を進める
- **中信頼度 (0.5-0.8)**: 防御的チェックを追加
- **低信頼度 (<0.5)**: 実装前に調査

## 実装パターン

### 信頼度によるパターン選択

```markdown
## 利用可能なパターン（コンテキストに基づいて選択）

### 高信頼度パターン (>0.9)
1. **Factory Pattern** - オブジェクト生成
   - 使用時: 類似オブジェクトが複数
   - 信頼度: 0.95
   - 例: src/factories/

2. **Repository Pattern** - データアクセス
   - 使用時: データベース操作
   - 信頼度: 0.92
   - 例: src/repositories/

### 中信頼度パターン (0.7-0.9)
1. **Observer Pattern** - イベント処理
   - 使用時: 疎結合が必要
   - 信頼度: 0.85
   - 検討: 組み込みEventEmitter

### 実験的パターン (<0.7)
1. **新しいアーキテクチャパターン**
   - 信頼度: 0.6
   - 推奨: まずプロトタイプ
```

## 結果に基づく次のステップ

- **高信頼度 (>0.9)** → `/test`またはレビュー準備完了
- **中信頼度 (0.7-0.9)** → 追加テストを検討
- **低信頼度 (<0.7)** → `/research`または計画が必要
- **品質問題** → 進行前に修正
- **すべてグリーン** → PR/commit準備完了
