# /branch - Gitブランチ名ジェネレーター

現在のGit変更を分析し、慣習的なパターンに従った適切なブランチ名を提案します。

**実装方法**: このコマンドは、最適なパフォーマンスとコンテキスト効率のために、専門の `branch-generator` サブエージェントに委譲します。

## 仕組み

このコマンドを実行すると：

1. Taskツール経由で `branch-generator` サブエージェントを起動
2. サブエージェントがgit diffとstatusを分析（コードベースコンテキスト不要）
3. 慣習的なブランチ名を生成
4. 複数の命名の代替案を返す

## 使用方法

### 基本的な使い方

```bash
/branch
```

現在の変更を分析してブランチ名を提案します。

### コンテキスト付き

```bash
/branch "OAuthでユーザー認証を追加"
```

説明を提案に組み込みます。

### チケット付き

```bash
/branch "PROJ-456"
```

ブランチ名にチケット番号を含めます。

## ブランチ命名規則

### タイププレフィックス

| プレフィックス | 使用場面 |
|---------------|----------|
| `feature/` | 新機能 |
| `fix/` | バグ修正 |
| `hotfix/` | 緊急修正 |
| `refactor/` | コード改善 |
| `docs/` | ドキュメント |
| `test/` | テスト追加/修正 |
| `chore/` | メンテナンスタスク |
| `perf/` | パフォーマンス改善 |
| `style/` | フォーマット/スタイリング |

### 形式

```text
<type>/<scope>-<description>
<type>/<ticket>-<description>
<type>/<description>
```

### 良い例

```bash
✅ feature/auth-add-oauth-support
✅ fix/api-resolve-timeout-issue
✅ docs/readme-update-install-steps
✅ feature/PROJ-123-user-search
```

### 悪い例

```bash
❌ new-feature (タイププレフィックスなし)
❌ feature/ADD_USER (大文字)
❌ fix/bug (曖昧すぎる)
❌ update_code (間違ったセパレータ)
```

## 出力形式

このコマンドは以下を提供：

- **現在の状態**: 現在のブランチ、変更ファイル数
- **分析**: 変更タイプ、主要スコープ、主要な変更
- **推奨名**: 分析に基づいて最も適切
- **代替案**: スコープ付き、説明的、簡潔版
- **使用手順**: ブランチを作成または名前変更する方法

## ワークフローとの統合

以下とシームレスに連携：

- `/commit` - 最初にブランチを作成、次にコミット
- `/pr` - ブランチ作成後にPR説明を生成
- `/think` - ブランチ作成前の計画

## 技術詳細

### サブエージェントの利点

- **90%のコンテキスト削減**: Git操作のみ、コードベースの読み込みなし
- **2-3倍高速な実行**: Git分析に最適化された軽量エージェント
- **専門ロジック**: ブランチ名生成専用
- **並列実行**: 他の操作と同時実行可能

### 使用するGit操作

サブエージェントはgitコマンドのみを実行：

- `git branch --show-current` - 現在のブランチを確認
- `git status` - ファイルステータスを確認
- `git diff` - 変更を分析
- ファイルシステムアクセスやコード解析なし

## 関連コマンド

- `/commit` - コミットメッセージを生成
- `/pr` - PR説明を作成
- `/research` - ブランチ作成前の調査

## ベストプラクティス

1. **早期にブランチ作成**: 変更を加える前に
2. **明確な命名**: 変更について具体的に
3. **慣習に従う**: プロジェクトのパターンに従う
4. **チケットを含める**: 該当する場合はIssueにリンク
5. **簡潔に保つ**: 50文字以下

## コンテキスト効率

このコマンドは最小限のコンテキスト使用に最適化：

- ✅ コードベースファイルは読み込まない
- ✅ Gitメタデータのみを分析
- ✅ 高速実行（5秒未満）
- ✅ 他のタスクと並列実行可能
