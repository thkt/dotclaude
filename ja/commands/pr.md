# /pr - Pull Request説明ジェネレーター

現在のブランチのすべての変更をベースブランチと比較して分析し、包括的なPR説明文を生成します。

**実装方法**: このコマンドは、最適なパフォーマンスとコンテキスト効率のために、専門の `pr-generator` サブエージェントに委譲します。

## 仕組み

このコマンドを実行すると：

1. Taskツール経由で `pr-generator` サブエージェントを起動
2. サブエージェントがベースブランチを動的に検出（main/master/develop）
3. git diff、コミット履歴、ファイル変更を分析
4. 包括的なPR説明文を生成
5. 複数のテンプレート代替案を返す

### 基本的な使い方

```bash
/pr
```

現在の変更を分析してPR説明文を提案します。

### Issue参照付き

```bash
/pr "#456"
```

特定のIssueにPRをリンクします。

### カスタムコンテキスト付き

```bash
/pr "このPRはチームミーティングで議論された新しい認証フローを実装"
```

追加のコンテキストを説明に組み込みます。

## PR説明文構造

### 必須セクション

1. **サマリー**: すべての変更の高レベル概要
2. **動機**: なぜこれらの変更が必要か
3. **変更内容**: 詳細な内訳
4. **テスト**: 確認方法
5. **関連**: リンクされたIssue/PR

### オプションセクション（変更に基づく）

- **スクリーンショット**: UI変更の場合
- **破壊的変更**: APIが変更された場合
- **パフォーマンス影響**: 最適化作業の場合
- **マイグレーションガイド**: 破壊的変更の場合

## 出力形式

このコマンドは以下を提供：

- **ブランチ分析**: 現在/ベースブランチ、コミット数、ファイル数、行数変更
- **変更サマリー**: タイプ、影響を受けるコンポーネント、破壊的変更、テストカバレッジ
- **推奨テンプレート**: 包括的なPR説明文
- **代替フォーマット**: 詳細版、簡潔版、カスタム版
- **使用手順**: 説明付きPRの作成方法

## ワークフローとの統合

以下とシームレスに連携：

- `/branch` - 最初にブランチを作成
- `/commit` - コミットを作成
- `/pr` - PR説明文を生成
- `/audit` - PR後のコードレビュー

## 技術詳細

### サブエージェントの利点

- **90%のコンテキスト削減**: Git操作のみ、コードベースの読み込みなし
- **2-3倍高速な実行**: Git分析に最適化された軽量エージェント
- **専門ロジック**: PR説明文生成専用
- **並列実行**: 他の操作と同時実行可能

### 使用するGit操作

サブエージェントはgitコマンドのみを実行：

- `git symbolic-ref` - ベースブランチを検出
- `git diff` - ブランチを比較
- `git log` - コミットを分析
- `git status` - 現在の状態を確認
- ファイルシステムアクセスやコード解析なし

### ベースブランチの検出

サブエージェントは自動的にベースブランチを検出：

1. 試行: `git symbolic-ref refs/remotes/origin/HEAD`
2. フォールバック: `main` → `master` → `develop`
3. 検証なしでは決して仮定しない

## 関連コマンド

- `/branch` - ブランチ名を生成
- `/commit` - コミットメッセージを生成
- `/audit` - コードレビュー

## ベストプラクティス

1. **コミット後にPRを作成**: すべての変更がコミットされていることを確認
2. **コンテキストを含める**: 動機と目標を提供
3. **テスト手順を追加**: レビュアーが検証するのを支援
4. **Issueをリンク**: 関連するIssueに接続
5. **送信前にレビュー**: 生成された説明を確認

## コンテキスト効率

このコマンドは最小限のコンテキスト使用に最適化：

- ✅ コードベースファイルは読み込まない
- ✅ Gitメタデータのみを分析
- ✅ 高速実行（10秒未満）
- ✅ 他のタスクと並列実行可能

## スマート機能

### 自動検出

- コミット/ブランチからのIssue番号
- 変更タイプ（feature/fix/refactor）
- 破壊的変更

### パターン認識

- API変更
- UI更新
