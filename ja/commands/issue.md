# /issue - GitHub Issue ジェネレーター

明確なタイトルと包括的な本文を持つ、適切に構造化されたGitHub Issueを生成します。

**実装**: このコマンドは、最適なパフォーマンスとコンテキスト効率のために、専門の`issue-generator`サブエージェントに委譲します。

## 仕組み

呼び出されると、このコマンドは：

1. Taskツール経由で`issue-generator`サブエージェントを起動
2. サブエージェントが提供された説明を分析
3. 構造化されたissueタイトルと本文を生成
4. オプションで`gh issue create`経由でissueを作成

## 使用方法

### 基本的な使用法

```bash
/issue "モバイルでログインボタンが動作しない"
```

説明からissueを生成します。

### タイプ指定

```bash
/issue bug "空入力でAPIが500を返す"
/issue feature "ダークモードサポートを追加"
```

issueタイプを明示的に指定します。

### 直接作成

```bash
/issue --create "データベース接続タイムアウト"
```

`gh` CLI経由で即座にissueを作成します。

## Issue構造

### タイトル形式

```text
[タイプ] 簡潔で具体的な説明
```

例:

- `[Bug] Safariモバイルでログイン失敗`
- `[Feature] CSV出力機能を追加`
- `[Docs] API認証ガイドを更新`

### 本文形式

```markdown
## 説明
[明確な問題文または機能要求]

## 再現手順（バグの場合）
1. [手順1]
2. [手順2]
3. [期待 vs 実際]

## 受け入れ基準
- [ ] [基準1]
- [ ] [基準2]

## 追加コンテキスト
[スクリーンショット、ログ、関連issue]
```

## 出力形式

コマンドは以下を提供します：

- **Issueタイトル**: タイププレフィックス付きフォーマット
- **Issue本文**: 構造化されたmarkdown
- **ラベル**: 内容に基づく推奨ラベル
- **ghコマンド**: 即座に実行可能なコマンド

## ワークフローとの統合

以下とシームレスに連携：

- `/branch` - issueからブランチを作成
- `/commit` - コミットでissueを参照
- `/pr` - PRをissueにリンク

## 関連コマンド

- `/branch` - ブランチ名を生成
- `/commit` - コミットメッセージを生成
- `/pr` - PR説明を作成

## コンテキスト効率

このコマンドは最小限のコンテキスト使用に最適化されています：

- ✅ コードベースファイルを読み込まない
- ✅ 説明のみを分析
- ✅ 高速実行（<5秒）
- ✅ 他のタスクと並列実行可能
