# 検証（Phase 3）

修正が堅牢で完全であることを確認するための品質チェックを実行。

## 目的

バグが修正されたと見なす前に、並列品質チェックで修正を検証。

## 品質チェック（並列実行）

最速の検証のためにこれらのチェックを同時に実行:

```bash
# 並列で全て実行（サポートされている場合）
npm test -- --findRelatedTests &
npm run lint -- --fix &
npm run type-check &
wait
```

### 個別コマンド

並列実行が利用できない場合は順次実行:

```bash
# 1. 関連ファイルのみテスト（高速）
npm test -- --findRelatedTests src/utils/pricing.ts

# 2. 自動修正付きリント
npm run lint -- --fix

# 3. 型チェック
npm run type-check

# または環境に応じた同等のもの:
# - yarn test / pnpm test / bun test
# - eslint --fix / tsc --noEmit
```

## 検証チェックリスト

### 1. テスト合格

- ✅ 回帰テストが合格（バグ修正）
- ✅ 既存のすべてのテストがまだ合格（回帰なし）
- ✅ 関連テストが合格（影響を受けた領域）

**コマンド**:

```bash
# 全テスト実行
npm test

# または影響を受けたテストのみ実行（高速）
npm test -- --findRelatedTests [変更ファイル]
```

**期待される結果**:

```text
PASS src/utils/pricing.test.ts
  ✓ 割引が合計を超える場合、負の値ではなく0を返すべき
  ✓ 基本的な割引を正しく計算
  ✓ ゼロ割引を処理

Test Suites: 1 passed, 1 total
Tests:       3 passed, 3 total
```

### 2. リント合格

- ✅ 新しいリントエラーなし
- ✅ 自動修正可能な問題は解決
- ⚠️ 警告は5未満なら許容

**コマンド**:

```bash
npm run lint -- --fix
```

**期待される結果**:

```text
✨ 0 errors, 2 warnings
```

**エラーがある場合のアクション**:

- すべてのエラーを即座に修正
- 時間があれば警告に対処
- リントエラーがある状態でコミットしない

### 3. 型チェック合格

- ✅ 変更ファイルに型エラーなし
- ✅ 関連ファイルに型エラーなし
- ✅ TypeScriptコンパイル成功

**コマンド**:

```bash
npm run type-check
# または: tsc --noEmit
```

**期待される結果**:

```text
✨ No type errors
```

### 4. 回帰が検出されない

修正が以下を壊していないことを確認:

- 関連機能
- エッジケース
- 統合ポイント
- APIコントラクト

**手動スポットチェック**:

```bash
# 開発サーバーを起動して手動テスト
npm run dev

# 確認:
# - 機能が期待通りに動作
# - 関連機能が影響を受けていない
# - コンソールエラーなし
```

## 品質ゲートサマリー

| チェック | ステータス | アクション |
| --- | --- | --- |
| テスト | ✅ 全て合格 | 続行 |
| テスト | ❌ 一部失敗 | 即座に修正 |
| リント | ✅ 0エラー | 続行 |
| リント | ❌ エラーあり | 全エラー修正 |
| リント | ⚠️ 警告5未満 | 許容 |
| 型 | ✅ エラーなし | 続行 |
| 型 | ❌ エラーあり | 即座に修正 |
| 手動 | ✅ 動作 | 続行 |
| 手動 | ❌ 問題あり | デバッグ＆修正 |

## 検証が失敗した場合

### テスト失敗

**根本原因調査**:

1. どのテストが失敗?
   - 回帰テスト → 修正が正しくない
   - 他のテスト → 回帰を導入

2. なぜ失敗?
   - ロジックエラー → 実装を見直す
   - テストの前提 → テストを更新または修正

**アクション**:

- Phase 2（実装）に戻る
- 修正を見直す
- 検証を再実行

### リント失敗

**アクション**:

```bash
# 可能なものを自動修正
npm run lint -- --fix

# 残りの問題を手動で修正
# 具体的なエラーはリント出力を確認
```

### 型チェック失敗

**アクション**:

```bash
# 具体的なエラーを確認
npm run type-check

# 型エラーを修正
# 必要に応じて型または実装を更新
```

## Fast Track vs Thorough

### Fast Track（信頼度 >0.9）

```bash
# 最小限のチェック
npm test -- --findRelatedTests [ファイル]
npm run lint -- --fix [ファイル]
```

### Thorough（信頼度 <0.9）

```bash
# 完全なテストスイート
npm test

# 完全なリント
npm run lint -- --fix

# 型チェック
npm run type-check

# 手動テスト
npm run dev
```

## 統合ポイント

- **前**: Phase 2（実装）
- **次**: Phase 3.5（テスト生成）または完了
- **失敗時**: Phase 2 に戻る
