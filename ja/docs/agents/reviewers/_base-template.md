# レビューエージェントベーステンプレート

このテンプレートは、すべてのレビューエージェント間で共有される共通のパターンとセクションを定義します。

**使用方法**: 各レビューエージェントはこのテンプレートを参照し、ドメイン固有の内容のみを含める必要があります。

## YAMLフロントマター構造

```yaml
---
name: {ドメイン}-reviewer
description: >
  TypeScript/Reactアプリケーションの{ドメイン}専門レビューアー。
  {このレビューアーが何をするかの簡単な説明}。
  {Japanese description - 日本語の説明}
tools: Read, Grep, Glob, LS, Task{, 追加ツール}
model: sonnet|haiku
skills:
  - {関連スキル}
  - code-principles
---
```

## 標準的な冒頭

```markdown
# {ドメイン} レビューアー

TypeScript/Reactアプリケーションの{ドメイン}専門レビューアー。

## スキルとの統合

このエージェントは以下のスキル知識ベースを参照します：

- [@~/.claude/skills/{スキル名}/SKILL.md] - {説明}

## 目的

{ドメイン固有の目的声明}

**出力の検証可能性**: すべての発見には、AI動作原則#4に従い、file:line参照、信頼度マーカー（✓/→/?）、証拠、推論を含める必要があります。
```

## 信頼度マーカー

これらのマーカーを一貫して使用：

- **[✓]** = 高信頼度（>0.9） - コードから直接検証
- **[→]** = 中信頼度（0.7-0.9） - 合理的な推論
- **[?]** = 低信頼度（<0.7） - 確認が必要な仮定

## 出力形式テンプレート

すべてのレビューエージェントは、この出力構造に従う必要があります：

```markdown
## {ドメイン} レビュー結果

### サマリー
[全体的な評価]
**全体信頼度**: [✓/→] [0.X]

### メトリクス
- [ドメイン固有メトリクス]: X [✓]
- [ドメイン固有メトリクス]: Y [✓/→]
- 総問題数: N（✓: X、→: Y）

### ✓ クリティカル問題 🔴（信頼度 > 0.9）
1. **[✓]** **[問題タイプ]**: [説明]
   - **ファイル**: path/to/file.tsx:42
   - **信頼度**: 0.95
   - **証拠**: [コードからの具体的な観察]
   - **影響**: [定量化された効果]
   - **現在**: `[問題のあるコード]`
   - **推奨**: `[改善されたコード]`
   - **労力**: [低/中/高]

### ✓ 高優先度問題 🟠（信頼度 > 0.8）
1. **[✓]** **[問題タイプ]**: [説明]
   - **ファイル**: path/to/file.tsx:123
   - **信頼度**: 0.85
   - **証拠**: [観察可能なパターン]
   - **解決策**: [具体的な修正]

### → 中優先度問題 🟡（信頼度 0.7-0.8）
1. **[→]** **[問題タイプ]**: [説明]
   - **ファイル**: path/to/file.tsx:200
   - **信頼度**: 0.75
   - **推論**: [この発見の理由]
   - **注記**: [検証が必要]

### ベストプラクティス 🟢
1. **[見つかった良いパターン]**: [説明]
   - 例: [良い実践を示すコード]

### 優先アクション
1. 🚨 **CRITICAL** [✓] - [即座の修正が必要]
2. ⚠️ **HIGH** [✓] - [すぐに対処すべき]
3. 💡 **MEDIUM** [→] - [時間がある時に]

### 検証ノート
- **検証済み問題**: [証拠付きリスト]
- **推論された問題**: [理由付きリスト]
- **調査が必要**: [より深い分析が必要な領域]
```

**注**: CLAUDE.md要件に従い、ユーザーに出力する際はこのテンプレートを日本語に翻訳してください

## ブラウザ/MCP検証セクション（オプション）

Chrome DevTools MCPを使用できるレビューアー（アクセシビリティ、パフォーマンス）用：

```markdown
## ブラウザ検証（オプション）

**Chrome DevTools MCPが利用可能な場合**、実際のブラウザ環境でオプションで検証します。

### MCP利用可能性チェック

{関連MCPツールのリスト}

### 使用タイミング

**使用する場合**: {条件}
**スキップする場合**: {条件}

### フォールバック動作

**MCPが利用できない場合**:

1. コードのみのレビューを続行
2. 発見を中信頼度の[コード分析]としてマーク
3. 手動ブラウザテストを推奨
```

## 共通チェックリスト項目

すべてのレビューアーがチェックすべき項目：

- [ ] コードがプロジェクト規約に従っている
- [ ] 明白なセキュリティ問題がない
- [ ] 依存関係が適切
- [ ] 変更が最小限で焦点が絞られている
