# TDD/RGRCサイクル実装詳細

このモジュールは、ベイビーステップ手法によるTDD実装の詳細ガイダンスを提供します。

## 1. Test-Driven Development (TDD) - t_wadastylen

**ゴール**: "動く綺麗なコード" - Ron Jeffries

### ベイビーステップ - TDDの基盤

**コア原則**: 各ステップで可能な限り小さな変更を行う

#### ベイビーステップが重要な理由

- **即座のエラー特定**: テスト失敗時、原因は最後の小さな変更にある
- **継続的な動作状態**: コードは常に数秒でグリーンに戻れる
- **迅速なフィードバック**: 各ステップは最大1-2分
- **信頼の構築**: 小さな成功が大きな機能に積み重なる

#### 実践でのベイビーステップ

```typescript
// ❌ 大きなステップ - 複数の変更を一度に
function calculateTotal(items, tax, discount) {
  const subtotal = items.reduce((sum, item) => sum + item.price, 0);
  const afterTax = subtotal * (1 + tax);
  const afterDiscount = afterTax * (1 - discount);
  return afterDiscount;
}

// ✅ ベイビーステップ - 一度に一つの変更
// ステップ1: ゼロを返す（最小限でテストを通す）
function calculateTotal(items) {
  return 0;
}

// ステップ2: 基本的な合計（次のテストがこれを駆動）
function calculateTotal(items) {
  return items.reduce((sum, item) => sum + item.price, 0);
}

// ステップ3: 税サポートを追加（テストが要求する場合のみ）
// ... 小刻みに続ける
```

#### ベイビーステップのリズム

1. **最小の失敗テストを書く**（30秒）
2. **最小限のコードで通す**（1分）
3. **テストを実行**（10秒）
4. **必要なら小さくリファクタリング**（30秒）
5. **グリーンならコミット**（20秒）

合計サイクル: 約2分

### 計画からのテスト生成（Pre-Red）

RGRCサイクルに入る前に、SOWからテストを自動生成：

```bash
# 1. テスト計画付きSOWを確認
.claude/workspace/planning/[feature-name]/sow.md

# 2. テスト計画が存在する場合、test-generatorを呼び出し
Task(
  subagent_type="test-generator",
  description="SOWからテストを生成",
  prompt="SOWテスト計画からテストを生成"
)

# 3. 生成されたテストを検証
npm test -- --listTests | grep -E "\.test\.|\.spec\."
```

**生成タイミング**：

- SOWに「テスト計画」セクションが含まれる
- 計画された機能の既存テストがない
- ユーザーがテスト生成を要求

**スキップ条件**：

- テストが既に存在
- SOWテスト計画が未定義
- クイック修正モード

### リアルタイムフィードバック付き拡張RGRCサイクル

1. **Redフェーズ**（信頼度目標: 0.9）

   ```bash
   npm test -- --testNamePattern="[current test]" | grep -E "FAIL|PASS"
   ```

   - 明確な意図で失敗テストを書く（または生成されたテストを使用）
   - 失敗理由が期待と一致することを検証
   - テストアサーションで理解を文書化
   - **終了基準**: 期待された理由でテスト失敗

2. **Greenフェーズ**（信頼度目標: 0.7）

   ```bash
   npm test -- --watch --testNamePattern="[current test]"
   ```

   - 通すための最小限の実装
   - 素早い解決策が許容される
   - 形式より機能に焦点
   - **終了基準**: テストが一貫して合格

3. **Refactorフェーズ**（信頼度目標: 0.95）

   ```bash
   npm test | tail -5 | grep -E "Passing|Failing"
   ```

   - SOLID原則を適用
   - 重複を削除（DRY）
   - 命名と構造を改善
   - 抽象化を抽出
   - **終了基準**: すべてのテストがグリーン、コードがクリーン

4. **Commitフェーズ**（信頼度目標: 1.0）
   - 品質チェック合格
   - カバレッジ維持/改善
   - 安定したコミット準備完了
   - ユーザーがgitコマンド実行
