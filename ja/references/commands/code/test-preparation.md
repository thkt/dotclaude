# テスト準備（Phase 0 - 対話型テスト有効化）

このモジュールは、仕様からのTDD前テスト生成を処理します。

## 目的

仕様から**スキップ状態**のテストケースを生成し、真のベイビーステップTDDのためにユーザー確認を経て一つずつ有効化します。

## 使用タイミング

spec.mdが存在し、テストシナリオが含まれている場合。

## ステップ1: スキップされたテストを生成

スキップモードでtest-generatorを使用してテスト足場を作成：

```typescript
Task({
  subagent_type: "test-generator",
  description: "仕様からスキップされたテストを生成",
  prompt: `
機能: "${featureDescription}"
仕様: ${specContent}

スキップモードでテストを生成:
1. FR-xxx要件 → スキップされたテストケース [✓]
2. Given-When-Thenシナリオ → スキップされた実行可能テスト [✓]
3. テストを順序付け: 単純 → 複雑（ベイビーステップ順） [→]
4. フレームワーク適切なスキップマーカーを使用:
   - Jest/Vitest: it.skip() + // TODO: [SKIP] コメント
   - 不明: コメントアウト + // TODO: [SKIP] マーカー

出力: すべてのテストがスキップ状態のテストファイル。
有効化順序の推奨を含める。
  `
})
```

次のテストを有効化しますか？

📝 テスト: "ゼロ入力を処理"
📁 ファイル: src/utils/discount.test.ts:15
📋 由来: FR-001（ゼロ購入処理）

```typescript
it('ゼロ入力を処理', () => {
  expect(calculateDiscount(0)).toBe(0.1)
})
```

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[S] 次のテストにスキップ
[Q] テスト生成を終了

```

## ステップ4: 有効化してRedフェーズに入る

ユーザー確認時（Y）：

1. **スキップマーカーを削除**
2. **テストを実行** → 失敗を確認（Redフェーズ）
3. **Greenフェーズに進む** → 最小限のコードを実装
4. **必要ならリファクタリング**
5. **ステップ3に戻る** 次のテストへ

```typescript
// 有効化前:
it.skip('ゼロ入力を処理', () => {
  // TODO: [SKIP] FR-001
  expect(calculateDiscount(0)).toBe(0.1)
})

// 有効化後:
it('ゼロ入力を処理', () => {
  expect(calculateDiscount(0)).toBe(0.1)
})
```

## メリット

- **真のベイビーステップ**: 一度に一つのテスト、ユーザー制御ペース
- **仕様駆動**: Given-When-Thenシナリオから派生したテスト
- **アクション前の確認**: サプライズなし、意図的な進捗
- **明確な進捗**: サイクル内の位置を常に把握

## TDDとの統合

```text
Phase 0: test-generatorがすべてのテストをスキップ状態で作成
  ↓
ループ:
  ├─ 次のスキップされたテストを表示
  ├─ 質問: "このテストを有効化？"（Y/S/Q）
  ├─ Yの場合:
  │   ├─ スキップマーカーを削除
  │   ├─ Red: テスト実行（失敗）
  │   ├─ Green: 実装
  │   ├─ Refactor: クリーンアップ
  │   └─ Commit: 状態を保存
  └─ 次のテスト
  ↓
すべてのテストが有効化され合格
```
