# Marketplace構造の維持

- Status: accepted
- Deciders: thkt
- Date: 2026-01-03

Technical Story: [SOW: Marketplace構造最適化](../../workspace/planning/20260103-marketplace-optimization/sow.md)

## Context and Problem Statement

~/.claude/ ディレクトリ配下のcommands, skills, agents, rulesを5つの独立プラグインに分割し、Git Submodule + 公式プラグインシステムで外部共有可能にする計画があった。

しかし、Claude Code プラグインシステムの技術調査により、プラグインキャッシュ制約で `shared/` ディレクトリ共有アプローチが実現不可能と判明した。

## Decision Drivers

- 既存ワークフロー（/think, /code, /audit, /commit）の安定動作維持
- 394件の相互参照を破壊しない
- 実現可能性と作業量のバランス
- 将来的な拡張性の確保

## Considered Options

- Option A: Marketplace維持（現状維持）
- Option B: Self-contained Plugins（完全自己完結型）
- Option C: Symlinks活用

## Decision Outcome

Chosen option: "Option A: Marketplace維持", because プラグインキャッシュ制約により shared/ 共有が不可能であり、自己完結型は作業量が大きく、現状維持が最もリスクが低い。

### Consequences

#### Positive Consequences

- 既存ワークフローが安定して動作し続ける
- 394件の相互参照を変更する必要がない
- 作業量が最小限で済む
- EN/JA同期の現状が維持される

#### Negative Consequences

- 個別プラグインとしての外部共有が不可能
- モノリシック構造のまま維持される

## Pros and Cons of the Options

### Option A: Marketplace維持

現在の marketplace.json 構造（9プラグイン、全て `"source": "./"`）を維持する。

- Good, because 既存の相互参照（394件）がそのまま動作する
- Good, because 作業量が最小限（変更なし）
- Good, because リスクが最低（破壊的変更なし）
- Bad, because 個別プラグインとしての外部共有が不可能
- Bad, because モノリシック構造のまま

### Option B: Self-contained Plugins

各プラグインに必要なファイルを全てコピーし、完全に自己完結させる。

- Good, because 個別プラグインとして外部共有可能
- Good, because 公式プラグインパターンに完全準拠
- Bad, because 394件の参照を各プラグイン内の相対パスに変換必要
- Bad, because ファイル重複（rules, skills等）が発生し、同期メンテナンス負荷が高い

### Option C: Symlinks活用

シンボリックリンクを使用して shared/ リソースを各プラグインから参照する。

- Good, because ファイル重複を避けられる
- Good, because 参照構造を一部維持できる
- Bad, because Windows環境での互換性に懸念がある
- Bad, because 環境依存性が高まる
